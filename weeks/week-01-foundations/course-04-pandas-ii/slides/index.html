Missing data & column ops

Load the canonical messy dataset

Titanic is the textbook dirty dataset — age and embarked have NaNs out of the box. Peek at the first rows before cleaning anything.

  


  

    
Find the holes

    
Always start cleaning by counting. isna().sum() is your microscope — a per-column tally of NaNs.

Three responses to a NaN

    
    
      
  • Drop — dropna(subset=['embarked']). Use when missingness is rare or non-informative.
    • 
      
  • Fill — fillna(df['age'].median()). Use when you can't afford to lose rows.
    • 
      
  • Interpolate — interpolate(axis=1). Use for ordered data with smooth structure.
    • 
    

  


  

    
Drop rows with NaN in a key column

    
When a column is required and only a few rows are missing it, drop those rows. dropna(subset=...) is targeted — safer than a bare dropna().


    

      
      
      
    

    

  


  

    
Fill with the column median

    
When you can't afford to lose rows, impute. assign + fillna keeps the operation pure — a new frame, no in-place mutation.


    

      
      
      
    

    

  


  

    
Interpolate across columns

    
When values are ordered with smooth structure (e.g. monthly bonuses), interpolate fills NaNs from neighbours. axis=1 walks left-to-right across each row.

Group, pivot, join

  


  

    
Split → Apply → Combine

    
    
Every groupby is these three steps. Pandas hides the loop; you just name the key and the aggregator.

  


  

    
GroupBy — the workhorse

    
Split → apply → combine. One key or many; one aggregation or a list.


    

      
      
      
    

    

  


  

    
GroupBy shrinks the row count

    
    
Result has one row per distinct group key. cat becomes the index; aggregated columns become the values.

Pivot ↔ Melt — long and wide are mirrors

    
    
Pivot widens (one row per id, one column per var); melt narrows (one row per id-var pair). Mirror operations.

  


  

    
Pivot — reshape from long to wide

    
Long-form data (one row per observation) becomes wide-form (a matrix) with one call. margins=True adds row/column totals — Geron's grade-book layout.


    

      
      
      
    

    

  


  

    
MultiIndex — stack vs unstack

    
    
Same data, two layouts. Pick whichever is easier to read or plot.

Merge — four join flavors

    
    
Same vocabulary as SQL. how= picks which side's keys survive when a match is missing.

  


  

    
Merge — SQL joins for DataFrames

    
Same vocabulary as SQL: inner, outer, left, right. how= picks the join type; on= picks the key.


    

      
      
      
    

    

  


  

    
Concat — stack rows or columns

    
concat glues frames together. axis=0 (default) stacks rows; axis=1 aligns by index and stacks columns.


    

      
      
      
    

    

  






  

    Part 3 · 1:00 – 1:30
    
Time series

  


  

    
A DatetimeIndex is the whole game

    
Once the index is datetimes, every time-aware method unlocks: .dt, .resample, partial-string slicing. pd.date_range generates regular timestamps.


    

      
      
      
    

    

  


  

    
Resampling — bucket time, then aggregate

    
    
Daily ticks above; month boundaries split them into buckets; one aggregated dot per bucket below. That is all resample does — a time-aware groupby.

  


  

    
Resample — change the frequency

    
Downsampling aggregates (many rows → one); upsampling interpolates (one row → many). Same method, opposite directions.


    

      
      
      
    

    

  


  

    
Rolling window — slide-and-aggregate

    
    
A window slides one step at a time; each position emits one aggregated value. First k-1 outputs are NaN until the window fills.
