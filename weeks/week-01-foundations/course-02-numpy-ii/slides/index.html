Part 1 · 0:00 – 0:30

Vectorize & ufuncs

The loop you stop writing

def slow(x):
    out = np.empty_like(x)
    for i in range(len(x)):
        out[i] = x[i]**2 + 3*x[i]
    return out

def fast(x):
    return x**2 + 3*x

On 1M elements, fast is typically 50-100x faster — and 5 characters shorter.

Element-wise arithmetic on paired arrays

a + b      a - b      a * b      a / b
np.maximum(a, b)        # cell-by-cell max
np.greater(a, b)        # cell-by-cell >

If shapes match, every operator applies element-by-element. If they don't — broadcasting (next part) makes them match.

Universal functions — math everywhere

np.sqrt(a)        np.exp(a)         np.log(a)
np.sin(a)         np.cos(a)
np.maximum(a, 0)  # ReLU in one line

A "ufunc" is a function that operates element-wise on any-shape array. Build your own with np.vectorize (slow) or just write the expression.

Reductions — and the axis mnemonic

a.mean()                  # collapse everything
a.mean(axis=0)            # collapse axis 0 -> one row
a.mean(axis=1)            # collapse axis 1 -> one column
a.sum(axis=(0, 2))        # collapse two axes

Mnemonic: axis=k means "the axis k disappears." Repeat that to yourself.

Part 2 · 0:30 – 1:00

Broadcasting

mixed shapes, no loops

The rule, in one sentence

Two shapes are compatible if, aligned right-to-left, every pair of dimensions is equal or one of them is 1.

A length-1 axis is stretched to match. A missing axis is treated as length 1.

Three live examples

# scalar
(2, 3) + ()          ->  (2, 3)

# row across rows
(2, 3) + (3,)        ->  (2, 3)

# column across cols
(2, 3) + (2, 1)      ->  (2, 3)

# outer-sum
(3, 1) + (1, 4)      ->  (3, 4)

Standardize a data matrix in one line

Z = (X - X.mean(axis=0)) / X.std(axis=0)
# X is (n, d). mean and std are (d,). They broadcast across rows.

This single line — column-wise standardization — is one of the most-used preprocessing steps in all of ML.

The "where did this shape come from" debug move

print(a.shape, b.shape, (a + b).shape)

When a broadcast surprises you, print the shapes. The bug is always in one of them.

Part 3 · 1:00 – 1:30

Linear algebra & stacking

Matrix multiplication — the @ operator

n1 = np.arange(10).reshape(2, 5)
n2 = np.arange(15).reshape(5, 3)
n1 @ n2                # shape (2, 3)

@ calls np.matmul. Use it; do not use * for matrix multiplication.

Solve a linear system

x = np.linalg.solve(A, b)     # Ax = b

Always prefer solve over inv(A) @ b — it is faster and numerically more stable.

Norms, inverse, determinant, eig

np.linalg.norm(m)        # Frobenius by default
np.linalg.inv(m)         # full inverse
np.linalg.det(m)         # determinant
w, V = np.linalg.eig(S)  # eigenpairs

All you need for PCA, basic optimization, and most "hand-rolled" ML math.

Stack & split — assemble bigger arrays

np.concatenate([a, b], axis=0)   # vstack
np.concatenate([a, b], axis=1)   # hstack
np.stack([a, b])                 # new leading axis
np.split(big, 2, axis=1)         # two halves
Recap

Three things to leave with

  1. If you wrote for i in range(len(arr)), you wrote it wrong. Vectorize.
  2. Broadcasting rule: aligned right-to-left, equal or one. Print shapes to debug.
  3. @ for matmul, np.linalg.solve for systems. Never form an inverse you don't need.
