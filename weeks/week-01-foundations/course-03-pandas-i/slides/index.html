A DataFrame is a dict of aligned Series. A Series is a labeled NumPy array. Hold those two sentences and the rest of the API falls into place.

Part 1 · 0:00 – 0:30

Series

A Series is a NumPy array with a label index

s = pd.Series([2, -1, 3, 5])
np.exp(s)                # ufuncs still work
s + 1000                 # scalar broadcasts
s < 0                    # element-wise compare

Everything you learned in NumPy still applies. Pandas just adds labels.

Label your rows

s2 = pd.Series([68, 83, 112, 68],
               index=['alice', 'bob', 'charles', 'darwin'])

s2['bob']      s2.loc['bob']     s2.iloc[1]

Gotcha — default-int labels are labels

s = pd.Series([1000, 1001, 1002, 1003])
sl = s[2:]            # labels 2 and 3 only
sl[0]                 # KeyError — there is no label 0!
sl.iloc[0]            # 1002 — positional

Use .iloc when you mean "position." Bare [k] means "label."

Automatic alignment

s2 + s3     # labels not in both -> NaN

This is the feature that makes Pandas Pandas. When you combine two Series, Pandas aligns by label first and only then arithmetic. Missing labels become NaN.

Part 2 · 0:30 – 1:00

DataFrames

Build from a dict of Series

people = pd.DataFrame({
    'weight':    pd.Series([68, 83, 112], index=['a', 'b', 'c']),
    'birthyear': pd.Series([1984, 1985, 1992],
                           index=['b', 'a', 'c']),
})

Pandas aligns the Series by their index. Different-length, different-order — no problem.

Or from values + columns + index

df = pd.DataFrame(
    [[1985, np.nan, 'Biking', 68],
     [1984, 3,      'Dancing', 83]],
    columns=['birthyear', 'children', 'hobby', 'weight'],
    index=['alice', 'bob'])

Or from a real CSV

from shared.data_utils import load_dataset
penguins = load_dataset('penguins')
penguins.head()

All Week 1 datasets come through load_dataset: penguins, titanic, tips, iris, stock-prices.

The first three things on any fresh dataset

Always inspect before you operate. Shape, types, summary — every single time.

Part 3 · 1:00 – 1:30

Indexing & filtering

Column access

df['species']             # one column -> Series
df[['species', 'mass']]   # several   -> DataFrame

Row access: .loc vs .iloc

df.loc[0]            # by label
df.iloc[2]           # by position
df.iloc[1:4]         # positional slice (endpoint exclusive)

Stick to .loc and .iloc. Bare df[k] is for columns; bare df[k:m] is for rows. The split confuses everyone.

Boolean masks — the main filter

df[df['mass'] > 6000]
mask = (df['species'] == 'Adelie') & (df['sex'] == 'Female')
df[mask]

Compose with & / | / ~ — never with and / or / not.

set_index, sort_values, sort_index

df.set_index(['day', 'time'])
df.sort_values('tip', ascending=False)
df.sort_index()

Once you understand the index, half of Pandas becomes "pick the right index, then everything is a one-liner."

Recap

Three things to leave with

  1. Series = labeled NumPy array; DataFrame = dict of aligned Series.
  2. .loc for labels, .iloc for positions. Use them instead of bare indexing.
  3. Boolean masks filter rows. Compose with &, |, ~.
