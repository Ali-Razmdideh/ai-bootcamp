The ND-Array

contiguous, typed, fast

NumPy is installed — confirm the version

Sanity-check that NumPy imports and report its version. Every notebook in this course starts the same way.

  


  

    
List vs. ndarray — same data, different machine

    
Python lists store pointers to boxed Python objects; ndarrays store raw C numbers back-to-back in one contiguous typed buffer. That difference is the entire speed story.


    

      
      
      
    

    

  


  

    
Anatomy of an ndarray

    
Every ndarray is a header (shape, dtype, strides) pointing at a flat, contiguous block of typed memory.

    
    
The header is tiny. The buffer is the data. Slicing & reshape mostly just rewrite the header.

  


  

    
NDArray header → memory block

    
A concrete (3, 3) float32 array: 4-byte elements, row stride 12, column stride 4.

    
    
The header carries shape + strides; the buffer carries bytes. Moving down a row in (3, 3) float32 = jump 12 bytes; moving across a column = jump 4 bytes.

  


  

    
Six recipes for creating arrays

    
99% of "make an array" reduces to one of these constructors: zeros, ones, full, arange, linspace, eye.


    

      
      
      
    

    

  


  

    
1-D, 2-D, 3-D — same ndarray, more axes

    

      

        
      

      

        
      

      

        
      

    

    
A 1-D row, a 2-D table, a 3-D stack of tables. ndim just counts the axes.

  


  

    
Random — use the modern API

    
np.random.seed is legacy. default_rng gives you a deterministic generator object — seed it once and pass it around.


    

      
      
      
    

    

  


  

    
dtypes and promotion

    
Adding two arrays promotes to the smallest dtype that holds both. Adding a float scalar always promotes to float64.


    

      
      
      
    

    

  


  

    
dtype memory footprint

    
Per-element cost in bytes. Multiply by size to get nbytes.

    
    
A million-element float64 array is 8 MB; the same data in float32 halves that — at the cost of precision.

  


  

    
astype — convert dtype, get a new array

    
astype returns a fresh array in the requested dtype. Going float→int truncates toward zero; no rounding.


    

      
      
      
    

    

  


  

    
Six attributes you always check

    
Every ndarray exposes shape, ndim, size, itemsize, nbytes, dtype. Inspecting these is the first move when a shape error bites.

    Part 2 · 0:30 – 1:00
    
Shape manipulation

    
reshape, transpose, axes

  


  

    
Reshape changes strides, not data

    
Reshape is essentially free — it makes a view over the same buffer. ravel returns a 1-D view when possible; flatten always copies.


    

      
      
      
    

    

  


  

    
Reshape — row-major fill order

    
The buffer is unchanged; the header relabels how to walk it. NumPy fills the new shape row-by-row (C-order).

    
    
Same 12 numbers, same buffer. Only the strides change.

  


  

    
Strides — from [i, j] to a flat offset

    
The 2-D index [i, j] maps to a byte offset in one contiguous block: offset = i · strides[0] + j · strides[1].

    
    
Strides are the bytes to skip per axis step. That's why reshape and transpose are cheap — they rewrite strides, never the buffer.

  


  

    
Transpose — also a stride trick

    
Transposing a 2-D array is the same as .T; for higher dims you list the axis permutation explicitly.


    

      
      
      
    

    

  


  

    
Add and remove dimensions

    
You will use these constantly for broadcasting (next course) and for matching tensor shapes in deep learning. np.newaxis inserts a length-1 axis; squeeze drops them.


    

      
      
      
    

    

  


  

    
Axis convention — what gets collapsed

    
The axis= argument names the axis that disappears after the reduction.

    
    
Mnemonic: axis=k removes the k-th entry of the shape tuple.

C-order vs F-order — a one-slide footnote

    
NumPy stores arrays row-by-row by default (C-order). Transposing flips to F-order without copying. You rarely care, until you do.

Indexing & slicing

    
basic, strided, fancy, boolean

  


  

    
Basic slicing — like Python, but per-axis

    
Slice each axis with start:stop:step, separated by commas. Returns a view over the same buffer.


    

      
      
      
    

    

  


  

    
Slicing visualized — a[1:3, :2]

    
Per-axis slicing carves a rectangular sub-grid out of the array.

    
  


  

    
Strided slicing — every other something

    
The third number in a slice is the step. ::-1 reverses; ::2 takes every other.

Fancy (integer-array) indexing

    
Pass lists of indices to pick any rows/columns/cells. Paired index arrays select one cell per pair. Always returns a copy.

Boolean masks — the most common filter

    
Mask + reduction is how you compute "how many rows satisfy X" in one line. You will use this every day.


    

      
      
      
    

    

  


  

    
np.where — pick from two arrays

    
Vectorized if/else. The shape of all three arguments must broadcast together.


    

      
      
      
    

    

  


  

    
Exercise — Masks & np.where

Gotcha — slices are views

    
Modifying a slice modifies the original. Use .copy() when you want independence.
