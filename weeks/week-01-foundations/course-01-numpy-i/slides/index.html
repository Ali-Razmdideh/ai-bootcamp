Part 1 · 0:00 – 0:30

The ND-Array

contiguous, typed, fast

List vs. ndarray — same data, different machine

py = [1, 2, 3, 4]                # heterogeneous, boxed
arr = np.array([1, 2, 3, 4])     # contiguous, typed (int64)

Python lists store pointers to Python objects; ndarrays store raw C numbers back-to-back. That difference is the entire speed story.

Live-time the loop sum vs np.sum — usually 50-100x.

Six recipes for creating arrays

np.zeros((3, 4))
np.ones((3, 4))
np.full((3, 4), np.pi)
np.arange(1, 5, 0.5)
np.linspace(0, 1, 6)
np.eye(3)

99% of "make an array" reduces to one of these.

Random — use the modern API

rng = np.random.default_rng(seed=42)
rng.random((3, 4))             # uniform [0, 1)
rng.standard_normal((3, 4))    # N(0, 1)

np.random.seed is legacy. default_rng gives you a deterministic generator object — pass it around.

dtypes and promotion

np.arange(0, 5, dtype=np.uint8) + 1.5
# -> float64  (the smallest dtype that holds both)

Adding two arrays promotes to the smallest type that holds both. Adding a float scalar always promotes to float64.

Six attributes you always check

  • shape — tuple of dimensions
  • ndim — number of axes
  • size — total elements
  • dtype — element type
  • itemsize — bytes per element
  • nbytes — total bytes
Part 2 · 0:30 – 1:00

Shape manipulation

reshape, transpose, axes

Reshape changes strides, not data

g  = np.arange(24)
g2 = g.reshape(6, 4)          # same buffer, new shape
g.ravel()                     # back to 1-D (view if possible)
g.flatten()                   # back to 1-D (always a copy)

Reshape is essentially free. Reshape-then-write may mutate the source — that bites everyone once.

Transpose — also a stride trick

m = np.arange(10).reshape(2, 5)
m.T                                   # (5, 2) view
t = np.arange(24).reshape(4, 2, 3)
t.transpose((1, 2, 0)).shape          # (2, 3, 4)

Transposing a 2-D array is the same as .T; for higher dims you list the axis permutation explicitly.

Add and remove dimensions

v = np.array([1, 2, 3])
v[np.newaxis, :].shape    # (1, 3) — row vector
v[:, np.newaxis].shape    # (3, 1) — column vector
np.squeeze(...)           # drop length-1 axes

You will use these constantly for broadcasting (next course) and for matching tensor shapes in deep learning.

C-order vs F-order — a one-slide footnote

    
NumPy stores arrays row-by-row by default (C-order). Transposing flips to F-order without copying. You rarely care, until you do.


a = np.arange(12).reshape(3, 4)
a.flags['C_CONTIGUOUS']        # True
a.T.flags['F_CONTIGUOUS']      # True

Indexing & slicing

    
basic, strided, fancy, boolean

  


  

    
Basic slicing — like Python, but per-axis


b = np.arange(48).reshape(4, 12)
b[1, 2]            # one cell
b[1, :]            # one row
b[:, 1]            # one column
b[1:3, 2:5]        # a sub-matrix
b[::-1]            # reverse rows

  


  

    
Strided slicing — every other something


a[1:3, ::2]    # rows 1-2, every other column

    
The third number in a slice is the step. ::-1 reverses; ::2 takes every other.

Fancy (integer-array) indexing


b[(0, 2), 2:5]                       # rows 0 and 2
b[:, (-1, 2, -1)]                    # cols last, 2, last
b[(-1, 2, -1, 2), (5, 9, 1, 9)]      # 4 paired cells

    
Pass lists of indices to pick any rows/columns/cells. Always returns a copy.

Boolean masks — the most common filter


m = np.array([20, -5, 30, 40])
m[m < 25]                  # [20, -5]
b[b % 3 == 1]              # every cell where x%3 == 1

    
Mask + reduction is how you compute "how many rows satisfy X" in one line. You will use this every day.

  


  

    
np.where — pick from two arrays


np.where(x % 2 == 0, x, -1)
# keep evens, replace odds with -1

    
Vectorized if/else. The shape of all three arguments must broadcast together.

Gotcha — slices are views


view = a[:2, :2]
view[0, 0] = -999
print(a)            # the source mutated!

    
Modifying a slice modifies the original. Use .copy() when you want independence.
