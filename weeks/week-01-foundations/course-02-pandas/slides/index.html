Part 1 — DataFrames & Loading

0:00 – 0:30

What is a DataFrame?

  • A 2-D table of columns. Each column is a typed Series.
  • Two axes: a row index and a columns label.
  • Underneath: NumPy arrays + a label-to-position lookup.

Loading data

import pandas as pd

df = pd.read_csv("penguins.csv")
df = pd.read_excel("file.xlsx")
df = pd.read_parquet("file.parquet")
df = pd.read_sql("SELECT * FROM t", conn)

In this bootcamp we use shared.data_utils.load_dataset so URLs aren't hard-coded in every notebook.

The first three things you always do

df.head()        # eyeball it
df.info()        # types + non-null counts
df.describe()    # 5-number summary of numeric columns

Skip these and you'll waste 30 minutes debugging a string column you thought was a float.

Index vs. columns

df.index        # row labels
df.columns      # column labels
df.set_index("penguin_id")
df.reset_index()

The index is not just row numbers — it's a real lookup. Time series almost always use a DatetimeIndex.

Part 2 — Selection & Filtering

0:30 – 1:00

Three ways to pick rows

toolusesexample
.loclabelsdf.loc[3:7, ['col']]
.ilocpositionsdf.iloc[3:7, 0:2]
booleanmasksdf[df['age'] > 25]

.loc end is inclusive; .iloc end is exclusive. Memorize this.

Boolean indexing — the workhorse

df[df['species'] == 'Adelie']
df[(df['species'] == 'Adelie') & (df['bill_length_mm'] > 40)]

# Same with .query
df.query('species == "Adelie" and bill_length_mm > 40')

Parens around each clause matter — & has higher precedence than ==.

Column access

df['col']            # always works
df.col               # works only if name is a valid attr
df[['a', 'b', 'c']]  # multiple columns -> new DataFrame

Part 3 — Cleaning & Grouping

1:00 – 1:30

Missing values

df.isna().sum()              # NaNs per column
df.dropna()                  # nuke any row with a NaN
df.fillna(0)                 # constant
df['x'].fillna(df['x'].median())   # impute with column median
df.ffill()                   # carry last valid value forward

Decide per column: drop vs. impute. There is no one correct strategy.

groupby + agg

df.groupby('species')['body_mass_g'].mean()

df.groupby(['species', 'sex']).agg(
    n=('body_mass_g', 'size'),
    mean_mass=('body_mass_g', 'mean'),
    std_mass =('body_mass_g', 'std'),
)

Pattern: split → apply → combine.

Feature engineering — derive new columns

df = df.assign(
    mass_kg = df['body_mass_g'] / 1000,
    is_heavy = df['body_mass_g'] > 4500,
    bill_ratio = df['bill_length_mm'] / df['bill_depth_mm'],
)

.assign returns a new DataFrame — method-chain friendly.

Leave with these reflexes

  1. head() → info() → isna().sum() on any fresh dataset.
  2. .loc for labels, .iloc for positions, masks for filters.
  3. Group, aggregate, derive. Pipelines, not for-loops.
