Context

Why pandas — and where Dask / Polars come in

We teach pandas because it is the lingua franca of Python data work: every tutorial, Stack Overflow answer, and ML library expects a DataFrame. But it is not the only option — and once your data or latency budget grows, two siblings are worth knowing.

pandas

Default for < ~few GB, single machine

Richest API, biggest ecosystem (scikit-learn, statsmodels, plotting). In-memory, single-threaded. The right tool for EDA, notebooks, modeling features.

Polars

When pandas is too slow or too big for RAM

Rust + Arrow, multi-threaded, lazy query optimizer, streaming for larger-than-RAM. Use for heavy ETL on a single beefy box, sub-second group-bys, log/event crunching.

Dask

When the data won't fit on one machine

Partitioned, pandas-like API that scales to a cluster (or a laptop's disk). Use for TB-scale CSV/Parquet pipelines, parallel feature engineering, distributed ML preprocessing.

Rule of thumb: prototype in pandas → rewrite the hot path in Polars when one machine is enough → switch to Dask when it isn't. The DataFrame mental model we build here transfers to both.

Part 1 · 0:00 – 0:30

Series

A Series is a NumPy array with a label index

A Series is a 1-D labeled array. NumPy ufuncs, scalar broadcasting and element-wise comparison all still work — Pandas just adds labels on top.

  


  

    
Series anatomy

    
  


  

    
Label your rows

    
An explicit index gives each entry a meaningful label. Access by label with [k] or .loc[k]; access by integer position with .iloc[i].


    

      
      
      
    

    

  


  

    
Gotcha — default-int labels are labels

    
When a Series has the default 0..N-1 index, slicing keeps the original labels — so slice_[0] is a KeyError. Use .iloc whenever you mean "position."


    

      
      
      
    

    

  


  

    
Build a Series from a dict

    
Passing a dict to pd.Series uses the keys as the index — a quick way to build a labeled Series from name → value pairs.


    

      
      
      
    

    

  


  

    
Automatic alignment

    
When you combine two Series, Pandas aligns by label first and only then does arithmetic. Labels missing from either operand become NaN. This is the feature that makes Pandas Pandas.

Indexing & filtering

  


  

    
Column access

    
A single column name returns a Series; a list of names returns a DataFrame with just those columns — the standard projection idiom.


    

      
      
      
    

    

  


  

    
Row access: .loc vs .iloc

    
Stick to .loc (label) and .iloc (position). Bare df[k] is for columns; bare df[k:m] is for rows. The split confuses everyone — being explicit removes the ambiguity.

Bare [] is overloaded — prefer .loc / .iloc for any non-trivial selection.

  


  

    
Boolean masks — the main filter

    
A comparison produces a boolean Series aligned with the rows; indexing the DataFrame with it keeps only the True rows. Compose with & / | / ~ — never with and/or/not.


    

      
      
      
    

    

  


  

    
Compose with &, |, ~

    
The bitwise operators combine boolean Series element-wise. Here ~penguins['sex'].isna() keeps only rows where sex is not missing.

Three responses to a NaN

    
    
      
  • Drop — dropna(subset=['embarked']). Use when missingness is rare or non-informative.
    • 
      
  • Fill — fillna(df['age'].median()). Use when you can't afford to lose rows.
    • 
      
  • Interpolate — interpolate(axis=1). Use for ordered data with smooth structure.
    • 
    

  


  

    
Drop rows with NaN in a key column

    
When a column is required and only a few rows are missing it, drop those rows. dropna(subset=...) is targeted — safer than a bare dropna().


    

      
      
      
    

    

  


  

    
Fill with the column median

    
When you can't afford to lose rows, impute. assign + fillna keeps the operation pure — a new frame, no in-place mutation.


    

      
      
      
    

    

  


  

    
Interpolate across columns

    
When values are ordered with smooth structure (e.g. monthly bonuses), interpolate fills NaNs from neighbours. axis=1 walks left-to-right across each row.

Time series

  


  

    
A DatetimeIndex is the whole game

    
Once the index is datetimes, every time-aware method unlocks: .dt, .resample, partial-string slicing. pd.date_range generates regular timestamps.


    

      
      
      
    

    

  


  

    
Resampling — bucket time, then aggregate

    
    
Daily ticks above; month boundaries split them into buckets; one aggregated dot per bucket below. That is all resample does — a time-aware groupby.

  


  

    
Resample — change the frequency

    
Downsampling aggregates (many rows → one); upsampling interpolates (one row → many). Same method, opposite directions.


    

      
      
      
    

    

  


  

    
Rolling window — slide-and-aggregate

    
    
A window slides one step at a time; each position emits one aggregated value. First k-1 outputs are NaN until the window fills.

  Beyond pandas
  
When to reach for Dask or Polars

  
Pandas is our default because it is the most widely used DataFrame library and integrates with the entire Python data stack. Once you hit its limits — RAM, single-thread speed, cluster scale — two alternatives use the same mental model.

  

    
      

        Tool
        Sweet spot
        Use case
      

    
    
      

        pandas
        < ~few GB, single machine, in-memory
        EDA, notebooks, ML feature engineering, plotting; anywhere ecosystem matters more than raw speed.
      

      

        Polars
        One big machine, multi-core, Arrow-native
        Fast ETL, sub-second group-bys on tens of GB, log/event analytics, lazy query optimization, larger-than-RAM streaming.
      

      

        Dask
        Multi-machine cluster, or laptop with disk spill
        TB-scale Parquet/CSV pipelines, distributed feature engineering, parallel preprocessing for ML, scaling existing pandas code with minimal rewrite.
      

    
  

  
The DataFrame model is the same. The skills you build in pandas transfer directly — only the execution engine changes.
