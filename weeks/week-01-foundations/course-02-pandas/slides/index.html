Loading data

import pandas as pd

df = pd.read_csv("file.csv")
df = pd.read_excel("file.xlsx")

df = pd.read_sql("SELECT * FROM t", conn)

In this bootcamp we use shared.data_utils.load_dataset(name) so URLs aren't hard-coded in every notebook.
read_csv has 50+ parameters. The ones you'll actually use: sep, header, names, na_values, parse_dates, dtype, nrows.

The first three things you always do

df.head()        # eyeball it
df.info()        # types + non-null counts + memory
df.describe()    # 5-number summary of numeric columns
Skip these and you'll waste 30 minutes debugging a string column you assumed was a float, or a join on the wrong dtype.

Bonus: df.describe(include='all') covers categorical columns too.

Index vs. columns

df.index           # row labels (often a RangeIndex)
df.columns         # column labels

df.set_index("penguin_id")    # promote a column → index
df.reset_index()              # demote index → column
df.rename(columns={"old": "new"})
The index isn't just row numbers — it's a real lookup. Time series almost always use a DatetimeIndex; multi-key joins use a MultiIndex.

Series — the building block

s = df["bill_length_mm"]        # one column → Series
type(s)                          # pandas.Series
s.values                         # underlying NumPy array
s.mean(), s.median(), s.std()
s.value_counts()                 # frequency table

A DataFrame is a dict of Series sharing the same index. Internalize that and half of Pandas falls into place.

Selection & Filtering

labels, positions, masks

Three ways to pick rows

toolusesexampleend
.loclabelsdf.loc[3:7, ['col']]inclusive
.ilocpositionsdf.iloc[3:7, 0:2]exclusive
booleanmasksdf[df['age'] > 25]—
The most common bug: .loc[0:3] returns rows 0, 1, 2, and 3 (4 rows). .iloc[0:3] returns 3 rows. Memorize this.

.loc and .iloc — visual

Boolean indexing — the workhorse

df[df['species'] == 'Adelie']

# Multiple conditions: parentheses + bitwise operators
df[(df['species'] == 'Adelie') & (df['bill_length_mm'] > 40)]
df[(df['species'] == 'Adelie') | (df['species'] == 'Gentoo')]
df[~df['sex'].isna()]                  #  ~  is "not"
Don't write and / or / not with Series — they raise. Use &, |, ~ and wrap each comparison in parentheses.

Cleaner syntax: .query and .isin

df.query('species == "Adelie" and bill_length_mm > 40')
df.query('species in ["Adelie", "Gentoo"] and 35 < bill_length_mm < 50')

# .isin is the readable form of (col == a) | (col == b) | ...
df[df['island'].isin(['Biscoe', 'Dream'])]

# Reference a local variable inside query with @
threshold = 4500
df.query('body_mass_g > @threshold')

Column access — three flavors

df['col']             # always works
df.col                # works only if name is a valid identifier
                      # (no spaces, no leading digit, no shadowing methods)
df[['a', 'b', 'c']]   # list → new DataFrame with those columns

df.filter(regex='_id$')   # all columns ending in _id
Don't write df.count hoping to access a column named "count" — that's the DataFrame method. Prefer df['col'] in code you ship.

Mutating safely

# Chained assignment is the classic Pandas gotcha
df[df['x'] > 0]['y'] = 99      #  ✗  SettingWithCopyWarning — may or may not work

# Always use a single .loc
df.loc[df['x'] > 0, 'y'] = 99  #  ✓
If you see SettingWithCopyWarning: stop, re-read your assignment, and rewrite it with a single .loc.
Part 3 · 1:00 – 1:30

Cleaning & Grouping

NaNs, split-apply-combine, features

Detecting missing values

df.isna().sum()              # NaNs per column
df.isna().any(axis=1).sum()  # how many rows have at least one NaN
df.isna().mean() * 100       # NaN percentage per column
Visualize NaNs with sns.heatmap(df.isna(), cbar=False) — instant view of where the holes are.

Strategy: drop vs. impute

drop

df.dropna()              # any NaN
df.dropna(how='all')     # only rows that are all NaN
df.dropna(subset=['x'])  # only NaN in col x
df.dropna(axis=1)        # drop NaN columns

Use when missing data is rare and probably MCAR.

impute

df.fillna(0)
df['x'].fillna(df['x'].median())
df['x'].fillna(df.groupby('cat')['x'].transform('median'))
df.ffill()    # carry last valid value forward
df.bfill()    # use next valid value backward

Use when missing data is structural or correlated.

There is no universally correct strategy. Decide per column, ideally informed by why the value is missing.

split → apply → combine

Every groupby in Pandas is exactly this pattern.

groupby + agg in code

df.groupby('species')['body_mass_g'].mean()

df.groupby(['species', 'sex']).agg(
    n         =('body_mass_g', 'size'),
    mass_mean =('body_mass_g', 'mean'),
    mass_std  =('body_mass_g', 'std'),
)

# transform: aggregate but keep the original shape
df['mass_zscore_by_species'] = (
    df.groupby('species')['body_mass_g']
      .transform(lambda x: (x - x.mean()) / x.std())
)
agg collapses; transform broadcasts the result back to the original index. filter keeps or drops whole groups.

Feature engineering — derive new columns

df = df.assign(
    mass_kg     = df['body_mass_g'] / 1000,
    is_heavy    = df['body_mass_g'] > 4500,
    bill_ratio  = df['bill_length_mm'] / df['bill_depth_mm'],
    log_mass    = np.log(df['body_mass_g']),
)

.assign returns a new DataFrame, so it chains beautifully. Prefer it over in-place df['x'] = ... when building pipelines.

The method-chain pattern

result = (
    df
      .dropna(subset=['body_mass_g'])
      .assign(mass_kg=lambda d: d['body_mass_g'] / 1000)
      .query('mass_kg > 4.0')
      .groupby('species')['mass_kg']
      .agg(['mean', 'std', 'count'])
      .round(2)
      .sort_values('mean', ascending=False)
)
Every step is a transformation; nothing is hidden in mutated state. This is how production data pipelines read.

Joins — one slide

left.merge(right, on='id', how='left')      # inner / left / right / outer
left.merge(right, left_on='aid', right_on='bid')
left.join(right, how='left')                 # index-based join

pd.concat([df1, df2], axis=0)                # stack rows
pd.concat([df1, df2], axis=1)                # bolt on columns
Always check row counts before and after a merge. A mis-keyed left join can silently multiply rows.

Common mistakes catalog

Chained assignment. df[mask][col] = ... — use single .loc.
and / or instead of & / |. Raises on Series.
Forgetting .copy(). A slice is a view; mutating it pokes the original.
Index mismatch on join. Merge produces NaN; you'll think your data is broken.
Mixed dtype object columns. A column of "ints" with one stray string becomes object and quietly slow.
Recap

Leave with these reflexes

  1. head() → info() → isna().sum() on any fresh dataset.
  2. .loc for labels, .iloc for positions, masks for filtering. Parens around comparisons.
  3. Group, aggregate, derive — in chained pipelines, not mutated state.
