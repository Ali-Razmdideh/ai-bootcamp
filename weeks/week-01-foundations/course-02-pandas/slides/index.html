Part 1 · 0:00 – 0:30

Series & DataFrames

the building blocks

Series — your first one

import pandas as pd
s = pd.Series([2, -1, 3, 5])
s
# 0    2
# 1   -1
# 2    3
# 3    5
# dtype: int64

A Series is a 1-D NumPy array with labels — think one column of a spreadsheet.

NumPy plus labels

import numpy as np
np.exp(s)                  # ufuncs work on Series

s + [1000, 2000, 3000, 4000]
s + 1000                   # scalar broadcasts (same as NumPy)
s < 0                      # element-wise comparison → bool Series

Index labels — name your rows

s2 = pd.Series([68, 83, 112, 68],
               index=["alice", "bob", "charles", "darwin"])

s2["bob"]                  # dict-like access: 83
s2.loc["bob"]              # by label    → 83
s2.iloc[1]                 # by position → 83
s2.iloc[1:3]               # slice keeps the labels
Use .loc for labels and .iloc for positions. Never the bare s[...] in code you'll re-read in three months.

Gotcha: default-int labels look like positions

surprise = pd.Series([1000, 1001, 1002, 1003])
slice_ = surprise[2:]      # slice — labels 2 and 3 survive

try:
    slice_[0]              # KeyError! label 0 is gone
except KeyError as e:
    print("Key error:", e)

slice_.iloc[0]             # 1002  — positions still work
If your index is numeric, s[0] is ambiguous. Use iloc / loc explicitly.

Build from a dict

weights = {"alice": 68, "bob": 83, "colin": 86, "darwin": 68}
s3 = pd.Series(weights)
# alice      68
# bob        83
# colin      86
# darwin     68

s4 = pd.Series(weights, index=["colin", "alice"])  # pick & reorder

Automatic alignment

s2.keys()    # alice · bob · charles · darwin
s3.keys()    # alice · bob · colin   · darwin

s2 + s3
# alice      136.0
# bob        166.0
# charles      NaN     ← only in s2
# colin        NaN     ← only in s3
# darwin     136.0
Pandas aligns by label, not by position. Missing labels become NaN — that's the whole story of how it merges sources.

The anatomy of a DataFrame

Build a DataFrame — from Series

people_dict = {
    "weight":    pd.Series([68, 83, 112], index=["alice", "bob", "charles"]),
    "birthyear": pd.Series([1984, 1985, 1992],
                           index=["bob", "alice", "charles"], name="year"),
    "children":  pd.Series([0, 3], index=["charles", "bob"]),
    "hobby":     pd.Series(["Biking", "Dancing"], index=["alice", "bob"]),
}
people = pd.DataFrame(people_dict)
people

Series are aligned by index; missing combinations become NaN. Series names are dropped.

Build a DataFrame — from values + labels

values = [[1985, np.nan, "Biking",  68],
          [1984, 3,      "Dancing", 83],
          [1992, 0,       np.nan,  112]]

d3 = pd.DataFrame(values,
                  columns=["birthyear", "children", "hobby", "weight"],
                  index=["alice", "bob", "charles"])

Loading from anywhere

df = pd.read_csv("file.csv")
df = pd.read_excel("file.xlsx")
df = pd.read_parquet("file.parquet")
df = pd.read_sql("SELECT * FROM t", conn)
df = pd.read_json("file.json")

# Straight from the internet
url = "https://raw.githubusercontent.com/plotly/datasets/master/us-cities-top-1k.csv"
us_cities = pd.read_csv(url, index_col=0).head()
read_csv has 50+ parameters. The ones you'll actually use: sep, header, names, na_values, parse_dates, dtype, nrows.

The first three things you always do

df.head()        # eyeball it
df.info()        # types + non-null counts + memory
df.describe()    # 5-number summary of numeric columns
Skip these and you'll waste 30 minutes debugging a string column you assumed was a float, or a join on the wrong dtype.
Part 2 · 0:30 – 1:00

Select & Filter

labels · positions · masks · queries

Columns

people["birthyear"]              # one column → Series
people[["birthyear", "hobby"]]   # list → DataFrame

Rows — three ways

people.loc["charles"]            # by label → Series
people.iloc[2]                   # by position → Series
people.iloc[1:3]                 # slice → DataFrame

# Boolean mask
people[np.array([True, False, True])]

# Almost always: combine with a condition
people[people["birthyear"] < 1990]
.loc end is inclusive, .iloc end is exclusive. df.loc[0:3] = 4 rows, df.iloc[0:3] = 3 rows.

Boolean masks compose with & / | / ~

people[(people["birthyear"] < 1990) & (people["weight"] > 70)]
people[people["hobby"].isin(["Biking", "Dancing"])]
people[~people["hobby"].isna()]
Never use Python and / or / not with Series. They raise. Use &, |, ~ with parens around each comparison.

Cleaner: .query

people.query("age > 30 and pets == 0")

# Reference a local variable with @
threshold = 30
people.query("body_mass_index > @threshold")

Add & drop columns

people["age"] = 2018 - people["birthyear"]
people["over 30"] = people["age"] > 30
birthyears = people.pop("birthyear")
del people["children"]

# Insert at a specific position
people.insert(1, "height", [172, 181, 185])

.assign — chainable feature engineering

(people
   .assign(body_mass_index = lambda df: df["weight"] / (df["height"] / 100) ** 2)
   .assign(overweight       = lambda df: df["body_mass_index"] > 25)
)
Pass a lambda, not people["bmi"] — the lambda sees the freshly-assigned columns from earlier in the chain. Without it, you can't reference a column created in the same call.

Sorting

people.sort_index(ascending=False)        # by row label, descending
people.sort_index(axis=1, inplace=True)   # by column name
people.sort_values(by="age", inplace=True)

Mutate safely

# SettingWithCopyWarning waiting to happen:
df[df["x"] > 0]["y"] = 99

# Always: single .loc
df.loc[df["x"] > 0, "y"] = 99
If you see SettingWithCopyWarning: stop, re-read your assignment, rewrite as one .loc.
Part 3 · 1:00 – 1:30

Clean, Group, Join

NaNs · split-apply-combine · merge

Where are the NaNs?

grades_array = np.array([[ 8,  8,  9],
                         [10,  9,  9],
                         [ 4,  8,  2],
                         [ 9, 10, 10]])
grades = pd.DataFrame(grades_array,
                      columns=["sep", "oct", "nov"],
                      index=["alice", "bob", "charles", "darwin"])

bonus = pd.DataFrame(
    [[0, np.nan, 2], [np.nan, 1, 0], [0, 1, 0], [3, 3, 0]],
    columns=["oct", "nov", "dec"],
    index=["bob", "colin", "darwin", "charles"])

(grades + bonus).isna().sum()

Two frames, different rows and columns. Adding them aligns by label — and produces NaNs wherever a label is missing on one side.

Fill or drop?

fill

(grades + bonus).fillna(0)
df["x"].fillna(df["x"].median())
df.ffill()    # last valid forward
df.bfill()    # next valid backward

drop

df.dropna()                  # any NaN row
df.dropna(how="all")         # only all-NaN rows
df.dropna(subset=["x"])      # only NaN in col x
df.dropna(axis=1, how="all") # all-NaN columns
Decide per column, ideally informed by why the value is missing.

Interpolate — fill the gaps with a curve

bonus.interpolate(axis=1)      # along the time-like axis

# Bob: 0 in Oct, 2 in Dec → interpolated Nov = 1
# Colin: missing Sep, has Nov=1 → still NaN in Oct (can't extrapolate left)

split → apply → combine

groupby — the workhorse

final_grades["hobby"] = ["Biking", "Dancing", np.nan, "Dancing", "Biking"]

final_grades.groupby("hobby").mean()        # NaN groups are skipped

final_grades.groupby(["hobby", "month"]).agg(
    n       = ("grade", "size"),
    avg     = ("grade", "mean"),
    spread  = ("grade", "std"),
)

# transform: aggregate but keep original shape
final_grades["z"] = (
    final_grades.groupby("hobby")["grade"]
                .transform(lambda x: (x - x.mean()) / x.std())
)

Pivot tables — spreadsheet thinking

pd.pivot_table(more_grades[["name", "grade", "bonus"]], index="name")
# default: mean of every numeric column, grouped by name

pd.pivot_table(more_grades, index="name",
               values=["grade", "bonus"], aggfunc="max")

pd.pivot_table(more_grades, index="name", values="grade",
               columns="month", margins=True)
# rows × cols matrix of means, with row / column totals

SQL-style joins

city_loc = pd.DataFrame(
    [["CA", "San Francisco", 37.78, -122.42],
     ["NY", "New York",      40.71,  -74.01],
     ["FL", "Miami",         25.79,  -80.32]],
    columns=["state", "city", "lat", "lng"])

city_pop = pd.DataFrame(
    [[ 808976, "San Francisco", "California"],
     [8363710, "New York",      "New-York"],
     [2242193, "Houston",       "Texas"]],
    columns=["population", "city", "state"])

pd.merge(city_loc, city_pop, on="city")              # INNER  → 2 rows
pd.merge(city_loc, city_pop, on="city", how="outer") # OUTER  → 4 rows + NaNs
pd.merge(city_loc, city_pop, on="city", how="left")  # LEFT
Always check row counts before and after a merge. A mis-keyed join can silently multiply rows.

Concatenation — stack frames

pd.concat([city_loc, city_pop])                  # stack rows
pd.concat([city_loc, city_pop], ignore_index=True)
pd.concat([city_loc, city_pop], join="inner")    # only shared columns
pd.concat([city_loc.set_index("city"),
           city_pop.set_index("city")], axis=1)  # side-by-side, aligned

Time series — one minute tour

temps = [4.4, 5.1, 6.1, 6.2, 6.1, 6.1, 5.7, 5.2, 4.7, 4.1, 3.9, 3.5]
dates = pd.date_range('2016/10/29 5:30pm', periods=12, freq='h')

ts = pd.Series(temps, index=dates, name="Temperature")

ts.resample("2h").mean()         # downsample, mean per 2-hour bucket
ts.resample("15min").interpolate(method="cubic")  # upsample + interpolate

ts.tz_localize("America/New_York").tz_convert("Europe/Paris")

Recap

Leave with these reflexes

  1. head() → info() → isna().sum() on any fresh dataset.
  2. .loc for labels, .iloc for positions, masks & query for filters.
  3. Group, aggregate, derive — in chained pipelines, not mutated state.
