Part 1 · 0:00 – 0:45

Matplotlib & Seaborn

make the data visible

The anatomy of a plot

One Figure can hold many Axes; one Axes can hold many Lines. Always work with (fig, ax), not the stateful plt shortcuts.

The object-oriented API

fig, ax = plt.subplots(figsize=(6, 4))
ax.plot(epochs, train_loss, label='train')
ax.plot(epochs, val_loss,   label='val')
ax.set_xlabel('epoch')
ax.set_ylabel('loss')
ax.set_title('Loss curves')
ax.legend()
ax.grid(alpha=0.3)
plt.plot(...) works but uses an invisible "current axes." That breaks the moment you want subplots. Start with fig, ax = plt.subplots() and never look back.

Subplots

fig, axes = plt.subplots(2, 3, figsize=(12, 6), sharex=True)
# axes is shape (2, 3) — index like a NumPy array

axes[0, 0].plot(...)
axes[1, 2].scatter(...)
fig.suptitle('Diagnostics')
fig.tight_layout()

Reuse the NumPy axis vocabulary you already know.

The two charts you'll draw most

line — losses, time series

ax.plot(epochs, train, label='train')
ax.plot(epochs, val,   label='val')

scatter — clusters, preds-vs-truth

ax.scatter(X[:, 0], X[:, 1],
           c=labels, cmap='viridis',
           s=20, alpha=0.7)
For a "predicted vs. actual" diagnostic, always add ax.axline((0, 0), slope=1, ls='--') — the y=x reference line.

Seaborn — stats plots in one line

sns.set_theme(style='whitegrid')

sns.heatmap(df.corr(numeric_only=True), annot=True, cmap='coolwarm', center=0)

sns.kdeplot(data=df, x='bill_length_mm', hue='species', fill=True)

sns.boxplot(data=df, x='species', y='body_mass_g')
sns.violinplot(data=df, x='species', y='body_mass_g')

sns.pairplot(df, hue='species')                  # the "first look" tool
Correlation heatmap is the fastest way to spot redundant features. Run it on any new dataset before modeling.

Choosing the right chart

questionchart
How does X evolve over time?line
Do two variables correlate?scatter (+ regression line)
What does the distribution of X look like?hist · kde · boxplot · violin
How does X differ across groups?boxplot · violin · strip + swarm
What's the relationship between many features?heatmap · pairplot
Where are the missing values?sns.heatmap(df.isna())

Common viz mistakes

Forgetting axis labels. A plot without labels is decoration, not information.
Default colormap on categorical data. viridis is ordinal; use a categorical palette for unordered classes.
Overlapping markers in dense scatters. Use alpha or a hexbin.
Wrong axis scale. Losses often need log y-scale: ax.set_yscale('log').
Part 2 · 0:45 – 1:15

SciPy

probability and optimization

scipy.stats — every distribution speaks the same dialect

from scipy import stats

# Continuous
stats.norm.pdf(x, loc=0, scale=1)              # PDF
stats.norm.cdf(x)                              # cumulative
stats.norm.rvs(size=1000)                      # samples
stats.norm.fit(data)                           # MLE parameters

# Discrete
stats.binom.pmf(k=5, n=10, p=0.5)

# Hypothesis testing
stats.ttest_ind(group_a, group_b)
stats.pearsonr(x, y)

Every distribution: .pdf / .cdf / .rvs / .fit. Learn the verbs once, get 80+ distributions free.

A 1-D loss landscape

The optimizer takes downhill steps until the gradient is ~0. This is the whole story of supervised learning.

scipy.optimize — what models actually do

from scipy import optimize

f = lambda x: (x - 3)**2 + np.sin(5 * x)
res = optimize.minimize(f, x0=0.0)

print(res.x)     # ≈ argmin
print(res.fun)   # min value
print(res.nit)   # iterations
Every supervised model: define a loss L(parameters), then ask an optimizer to minimize it. fit in sklearn, backward() in PyTorch — all the same idea.

Local minima — why initialization matters

optimize.minimize(f, x0=0.0).x      # ≈ 2.95
optimize.minimize(f, x0=10.0).x     # ≈ 6.21   different local min!
Bumpy loss surfaces are everywhere in neural nets. This is why "different random seed → different model" is a real conversation, not folklore.

For multi-modal landscapes try global solvers: differential_evolution, basinhopping, dual_annealing.

The optimization API at a glance

needfunction
minimize a scalar functionoptimize.minimize_scalar
minimize over many varsoptimize.minimize
nonlinear least-squares fitoptimize.curve_fit
find rootsoptimize.root
global optimum (no gradient)optimize.differential_evolution
linear programmingoptimize.linprog

Worked example: fit a Gaussian

samples = rng.normal(loc=2.0, scale=1.5, size=1000)
mu_hat, sigma_hat = stats.norm.fit(samples)        # MLE

xs = np.linspace(samples.min(), samples.max(), 200)
fig, ax = plt.subplots()
ax.hist(samples, bins=30, density=True, alpha=0.5)
ax.plot(xs, stats.norm.pdf(xs, mu_hat, sigma_hat), lw=2, label='fitted')
ax.legend()

This is exactly what a one-parameter "model" does — fit, then plot fit vs. data. Now scale it up to millions of parameters and you have deep learning.
