Part 1 · 0:00 – 0:30

The ND-Array

contiguous, typed, fast

Python list vs. ndarray

list

xs = [1, 2, 3, 4]
# array of PyObject* pointers
# each int boxed on the heap

ndarray

arr = np.array([1, 2, 3, 4])
# one contiguous int64 buffer
# 32 bytes, end of story

A single mental picture explains every speed difference we'll see.

Draw the memory model live (or use the next slide). Stress: list cells point to objects scattered across the heap; the ndarray is one block of bytes the CPU can stream through.

The memory model

The picture is the lesson. Pause here. Ask: "where does the CPU spend its time when summing a list of ints?" Answer: chasing pointers. The ndarray lets it stream bytes — that's it.

Measure it — the speed story

%timeit sum(range(1_000_000))             # ~30 ms
%timeit np.arange(1_000_000).sum()        # ~1 ms
~30× faster, half the code, same answer. This is not a micro-optimization — it's the reason NumPy exists.
Don't quote these numbers as gospel; show students how to measure on their own laptop. %timeit is your friend.

dtypes — one float promotes the whole array

np.array([1, 2, 3]).dtype              # int64
np.array([1.0, 2, 3]).dtype             # float64  (one float wins)
np.array([1, 2, 3], dtype=np.float32)   # explicit
np.array([True, False, True]).dtype     # bool
Pick the smallest dtype that fits. A 224×224×3 image in uint8 is 150 kB; in float64 it's 1.2 MB. Multiply by batch size 32 and you're at 38 MB just for one mini-batch.

Creating arrays — five recipes

np.zeros((3, 4))            # 3x4 of 0.0
np.ones((2, 2), dtype=int)  # 2x2 of 1
np.full((2, 2), 7)          # 2x2 of 7
np.arange(0, 10, 2)         # [0 2 4 6 8]   like range()
np.linspace(0, 1, 5)        # [0. 0.25 0.5 0.75 1.]   inclusive end
np.eye(3)                   # identity matrix
rng = np.random.default_rng(0)
rng.normal(size=(2, 3))     # 2x3 standard normal
Always seed your RNG in notebooks (default_rng(seed)). Reproducibility is the difference between a tutorial and a bug report.
np.random.seed(...) is the legacy API. Prefer the default_rng Generator — it's stateless across libraries.

Part 2 · 0:30 – 1:00

Shape & Slicing

strides, views, and N-D indexing

Shape is meaning

a = np.arange(12).reshape(3, 4)
a.shape, a.ndim, a.size      # (3, 4)  2  12

Shape isn't a number; it's vocabulary:

  • A grayscale image: (H, W)
  • A color image: (H, W, 3)
  • A batch of color images (PyTorch): (B, 3, H, W)
  • A batch of token IDs: (B, T)
  • Hidden states in a transformer: (B, T, D)

When debugging, always print .shape before guessing.

reshape and the -1 trick

x = np.arange(24)
x.reshape(4, 6)
x.reshape(2, 3, 4)
x.reshape(2, -1)     # NumPy infers the missing dim → (2, 12)
x.reshape(-1)        # flatten (same as x.ravel())
reshape never copies if it can avoid it. The data buffer stays put; only the strides change.

What is a stride, really?

A stride is "how many bytes do I jump to move one step along this axis?" That's all reshape and transpose ever change.

Transpose — and why batch layout matters

m = np.arange(6).reshape(2, 3)
m.T                           # (3, 2)

img = rng.random((224, 224, 3))           # HWC (matplotlib)
img.transpose(2, 0, 1).shape              # CHW (PyTorch)  → (3, 224, 224)
Half of all PyTorch tensor-shape bugs are HWC ↔ CHW confusion. Print shapes generously.

Slicing in N dimensions

a = np.arange(20).reshape(4, 5)
a[1, 2]        # scalar at row 1, col 2
a[1]           # whole row 1                          → shape (5,)
a[:, 0]        # whole column 0                       → shape (4,)
a[1:3, 2:4]    # 2x2 sub-block
a[::-1]        # rows reversed
a[:, ::2]      # every other column
a[..., 0]      # ... = "as many ':' as needed"

Comma separates axes. Colon means "everything along this axis."

Gotcha: slices are views, not copies

a = np.arange(12).reshape(3, 4)
b = a[:2, :2]
b[0, 0] = -999
print(a)        # the source mutated too!
You will hit this once. After that you'll remember .copy() forever.

Want a copy? a[:2, :2].copy() or np.array(a[:2, :2], copy=True).

Boolean & fancy indexing

x = rng.normal(size=10)

x[x > 0]          # boolean mask — keep positives
x[x < 0] = 0      # clip in-place

ix = np.array([0, 3, 5])
x[ix]             # fancy index — pick specific positions
Boolean masks compose: (x > 0) & (x < 1). Note the parentheses — & binds tighter than comparison operators.

Pattern: None / np.newaxis

v = np.array([1, 2, 3])      # shape (3,)
v[:, None].shape             # (3, 1)   — column
v[None, :].shape             # (1, 3)   — row
v[None, :, None].shape       # (1, 3, 1)
None = "insert a length-1 axis here." You'll use it constantly to set up broadcasting.
Part 3 · 1:00 – 1:30

Vectorize & Broadcast

the loop you stop writing

The loop you should never write

DON'T

out = np.empty_like(x)
for i in range(len(x)):
    out[i] = x[i]**2 + 3*x[i]

DO

out = x**2 + 3*x

Same answer. ~50× faster. Half the code.

Element-wise & universal functions (ufuncs)

a + b, a - b, a * b, a / b, a ** 2

np.exp(a), np.log(a), np.sqrt(a), np.sin(a)
np.maximum(a, 0)            # ReLU in one line
np.clip(a, 0, 1)

All of these are C loops behind a Python API. They're as fast as it gets without leaving NumPy.

Reductions — the axis mnemonic

a = np.arange(12).reshape(3, 4)
a.sum()                # scalar
a.sum(axis=0)          # shape (4,) — column totals
a.sum(axis=1)          # shape (3,) — row totals
a.sum(axis=0, keepdims=True)   # shape (1, 4) — handy for broadcasting
The mnemonic: axis is the one that disappears in the result.

Same vocabulary for mean, std, max, min, argmax, any, all.

Broadcasting in one picture

stretches the row to every row of the matrix. No copy. No allocation. Just smart strides.

The broadcasting rule

Align shapes from the right. Each pair of dimensions is compatible if they are equal, or one of them is 1.

ABResultNotes
(3, 3)(3,)(3, 3)row broadcast
(3, 1)(1, 3)(3, 3)outer-product shape
(4, 1, 3)(2, 3)(4, 2, 3)middle dim stretched
(B, T, D)(D,)(B, T, D)per-feature bias
(3, 2)(3,)ERROR3 vs. 2 — no rescue

Worked example: pairwise distances

pts = rng.normal(size=(100, 2))             # 100 points in 2D

# Naive: two nested Python loops — slow & verbose
# Vectorized:
diff = pts[:, None, :] - pts[None, :, :]    # (100, 1, 2) - (1, 100, 2)
                                            #            → (100, 100, 2)
dist = np.sqrt((diff**2).sum(axis=-1))       # (100, 100)
This [:, None, :] - [None, :, :] pattern shows up in every attention-mechanism implementation. It's worth staring at until it clicks.

Common broadcasting mistakes

Shape mismatch on the wrong side. Want to center columns? x - x.mean(axis=0) ✓ (shape (n, d) - (d,)).
Center rows? x - x.mean(axis=1, keepdims=True) — without keepdims you get a shape error.
Silent broadcasting. x + y where both happen to be 1-D — NumPy obliges; you might have wanted a matrix.
Habit: when in doubt, print shapes on both sides of every arithmetic op.
