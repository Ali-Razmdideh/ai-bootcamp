Part 1 — The ND-Array

0:00 – 0:30

Python list vs. NumPy array

xs = [1, 2, 3, 4]              # list of PyObject pointers
import numpy as np
arr = np.array([1, 2, 3, 4])   # contiguous int64 buffer
  • A list stores pointers to boxed Python ints scattered in memory.
  • An ndarray is one contiguous typed buffer — like a C array.
Draw the memory model: list = array of pointers each hopping to a different heap location; ndarray = one block of bytes. This single picture explains every speed difference we will see.

Why it is fast

  1. Locality: the CPU prefetches contiguous bytes.
  2. Typed: no per-element type checks.
  3. SIMD/BLAS: vector instructions process multiple values per cycle.
%timeit sum(range(1_000_000))            # ~30 ms
%timeit np.arange(1_000_000).sum()       # ~1 ms

dtypes

np.array([1, 2, 3]).dtype          # int64
np.array([1.0, 2, 3]).dtype        # float64  (one float promotes the whole array)
np.array([1, 2, 3], dtype=np.float32).dtype  # float32

Pick the smallest dtype that fits your data — memory matters for large images and tensors.

Creating arrays

np.zeros((3, 4))            # 3x4 of 0.0
np.ones((2, 2), int)        # 2x2 of 1
np.arange(0, 10, 2)         # [0 2 4 6 8]
np.linspace(0, 1, 5)        # [0. 0.25 0.5 0.75 1.]
rng = np.random.default_rng(0)
rng.normal(size=(2, 3))     # 2x3 standard normal

Habit: always seed your RNG in notebooks. Reproducibility is the difference between a tutorial and a bug report.

Part 2 — Shape & Slicing

0:30 – 1:00

Shape, ndim, size

a = np.arange(12).reshape(3, 4)
a.shape   # (3, 4)
a.ndim    # 2
a.size    # 12

An image is just a shape. A 224×224 RGB photo is an array of shape (224, 224, 3); a batch of 32 of them is (32, 224, 224, 3).

reshape and -1

x = np.arange(24)
x.reshape(4, 6)
x.reshape(2, 3, 4)
x.reshape(2, -1)    # NumPy infers the missing dim → (2, 12)

Reshape never copies if it can avoid it. The data buffer stays put; only the strides change.

Transpose

m = np.arange(6).reshape(2, 3)
m.T                  # (3, 2)
img = rng.random((224, 224, 3))
img.transpose(2, 0, 1).shape   # (3, 224, 224) — PyTorch layout

Slicing in N dimensions

a = np.arange(20).reshape(4, 5)
a[1, 2]        # scalar at row 1, col 2
a[1]           # whole row 1
a[:, 0]        # whole column 0
a[1:3, 2:4]    # 2x2 sub-block
a[::-1]        # rows reversed
a[:, ::2]      # every other column

Slices are views, not copies — mutating them mutates the source.

Boolean & fancy indexing

x = rng.normal(size=10)
x[x > 0]                       # keep positives
x[x > 0] = 0                   # clip them in-place

ix = np.array([0, 2, 4])
x[ix]                          # pick specific rows

Part 3 — Vectorization & Broadcasting

1:00 – 1:30

The loop you should never write

out = np.empty_like(x)
for i in range(len(x)):
    out[i] = x[i] ** 2 + 3 * x[i]

vs.

out = x**2 + 3*x

Same answer. ~50× faster. Half the code.

Element-wise & reductions

a + b, a * b, np.exp(a), np.sqrt(a)

a.sum(),    a.sum(axis=0)
a.mean(),   a.mean(axis=1)
a.max(),    a.argmax()

The axis argument is the one you must internalize: "the axis that disappears in the result."

Broadcasting in one picture

(3,)        +     (3, 3)
   [a b c]    +   [[1 2 3]
                   [4 5 6]
                   [7 8 9]]

         ↓  (NumPy stretches the row to every row)

                  [[1+a 2+b 3+c]
                   [4+a 5+b 6+c]
                   [7+a 8+b 9+c]]

The broadcasting rule

Align shapes from the right. Compatible if each dimension is either equal or 1.

ABResult
(3, 3)(3,)(3, 3)
(3, 1)(1, 3)(3, 3)
(4, 1, 3)(2, 3)(4, 2, 3)
(3, 2)(3,)ERROR

Worked example: pairwise distances

pts = rng.normal(size=(100, 2))     # 100 points in 2D

# Naive: two nested loops — slow, verbose
# Vectorized:
diff = pts[:, None, :] - pts[None, :, :]    # (100, 100, 2)
dist = np.sqrt((diff**2).sum(axis=-1))      # (100, 100)

This pattern — insert a length-1 axis with None — appears everywhere in ML code.

What you should leave with

  • ndarray = contiguous typed buffer. That is the entire speed story.
  • Shape is meaning. Memorize the axis argument.
  • If you typed for on numeric data, pause and ask: can broadcasting do this?
