Part 1 · 0:00 – 0:30

The ND-Array

contiguous, typed, fast

List vs. ndarray — same data, different machine

list

xs = [1, 2, 3, 4]
# array of PyObject* pointers
# each int boxed on the heap

ndarray

import numpy as np
arr = np.array([1, 2, 3, 4])
# one contiguous int64 buffer
# 32 bytes, end of story

The memory model

Measure it

%timeit sum(range(1_000_000))             # ~30 ms
%timeit np.arange(1_000_000).sum()        # ~1 ms
~30× faster, half the code. The ratio matters, not the absolute milliseconds — run it on your own laptop.

Creating arrays — the recipes you'll use weekly

np.zeros(5)                # [0. 0. 0. 0. 0.]
np.zeros((3, 4))           # 3×4 of 0.0
np.ones((3, 4))
np.full((3, 4), np.pi)     # 3×4 filled with π
np.empty((2, 3))           # uninitialized — fast, garbage values

np.arange(1, 5)            # [1 2 3 4]
np.arange(1, 5, 0.5)       # [1. 1.5 2. 2.5 3. 3.5 4. 4.5]
np.linspace(0, 5/3, 6)     # 6 evenly-spaced points, end included

rng = np.random.default_rng(seed=42)
rng.random((3, 4))         # uniform [0, 1)
rng.standard_normal((3, 4))
Seed your RNG. Reproducible code is the difference between a tutorial and a bug report.

dtypes — one float promotes the whole array

c = np.arange(1, 5)
print(c.dtype, c)                       # int64 [1 2 3 4]

k1 = np.arange(0, 5, dtype=np.uint8)
k2 = k1 + np.array([5, 6, 7, 8, 9], dtype=np.int8)
print(k2.dtype, k2)                     # int16 [5 7 9 11 13]

k3 = k1 + 1.5
print(k3.dtype, k3)                     # float64 [1.5 2.5 3.5 4.5 5.5]
Smallest dtype that fits = lowest memory. A 224×224×3 image in uint8 is 150 kB; in float64 it's 1.2 MB.

Inspect any array in six attributes

a = np.zeros((3, 4))
a.shape, a.ndim, a.size, a.itemsize, a.nbytes, a.dtype
#  (3, 4)    2      12        8         96     float64

Print these on every fresh array. It catches 80% of bugs before they happen.

Part 2 · 0:30 – 1:00

Shape & Slicing

reshape · transpose · index

Shape is meaning

g = np.arange(24)
g.shape = (6, 4)            # reshape in place

g2 = g.reshape(4, 6)        # new view, same buffer
g.ravel()                   # back to 1-D

t = np.arange(24).reshape(4, 2, 3)
t.transpose((1, 2, 0)).shape    # (2, 3, 4) — axes permuted

m1 = np.arange(10).reshape(2, 5)
m1.T                         # quick transpose of a 2-D array
reshape and transpose change strides, not data. No copy.

What shape means in practice

  • Grayscale image: (H, W)
  • Color image: (H, W, 3)
  • Batch of color images (PyTorch): (B, 3, H, W)
  • Batch of token IDs: (B, T)
  • Hidden states in a transformer: (B, T, D)

When debugging, always print .shape before guessing.

2-D slicing — the main move

b = np.arange(48).reshape(4, 12)

b[1, 2]          # scalar at row 1, col 2  → 14
b[1, :]          # whole row 1             → shape (12,)
b[:, 1]          # whole column 1          → shape (4,)
b[1:3, 2:5]      # 2×3 sub-block
b[::-1]          # rows reversed

Comma separates axes; colon = "everything along this axis."

Fancy indexing — pick whatever rows/cols you want

b = np.arange(48).reshape(4, 12)

b[(0, 2), 2:5]              # rows 0 and 2, cols 2-4
b[:, (-1, 2, -1)]           # all rows; cols last, 2, last (repeats!)
b[(-1, 2, -1, 2), (5, 9, 1, 9)]   # paired indices — 4 specific cells
Two index arrays of equal length pair up element-wise — handy for gathering arbitrary cells.

Boolean indexing — filter by a condition

m = np.array([20, -5, 30, 40])
m[m < 25]                   # [20 -5]

# Mix with rows
rows_on = np.array([True, False, True, False])
b[rows_on, :]               # keep rows 0 and 2

# 2-D mask: pick all elements satisfying a condition
b[b % 3 == 1]               # 1-D array of every elem where x%3 == 1

Gotcha: slices are views

a = np.arange(12).reshape(3, 4)
b = a[:2, :2]
b[0, 0] = -999
print(a)        # the source mutated too!
You will hit this once. After that you'll remember .copy() forever.

Pattern: None / np.newaxis

v = np.array([1, 2, 3])         # shape (3,)
v[:, None].shape                # (3, 1)   — column
v[None, :].shape                # (1, 3)   — row
v[None, :, None].shape          # (1, 3, 1)
None = "insert a length-1 axis here." You'll use it constantly to set up broadcasting.
Part 3 · 1:00 – 1:30

Vectorize

arithmetic · broadcasting · matmul

The loop you stop writing

DON'T

out = np.empty_like(x)
for i in range(len(x)):
    out[i] = x[i]**2 + 3*x[i]

DO

out = x**2 + 3*x

Same answer. ~50× faster. Half the code.

Element-wise arithmetic — paired arrays

a = np.array([14, 23, 32, 41])
b = np.array([5, 4, 3, 2])

a + b           # [19 27 35 43]
a - b           # [ 9 19 29 39]
a * b           # [70 92 96 82]
a / b           # [2.8  5.75 10.67 20.5]
a ** b          # element-wise power

np.add(a, b)        # same as a + b
np.greater(a, b)    # element-wise comparison
np.maximum(a, b)    # element-wise max

Universal functions — math everywhere

a = np.array([14, 23, 32, 41])

np.sqrt(a)              # [3.74 4.80 5.66 6.40]
np.exp(a)
np.sin(a)
np.maximum(a, 0)        # ReLU in one line
np.clip(a, 0, 30)       # [14 23 30 30]

All ufuncs are C loops behind a Python API. As fast as it gets without leaving NumPy.

Reductions — and the axis mnemonic

a = np.array([[-2.5, 3.1, 7],
              [10,    11,  12]])
a.mean()                        # 6.7666...

c = np.arange(24).reshape(2, 3, 4)
c.sum(axis=0)                   # collapse axis 0 → shape (3, 4)
c.sum(axis=1)                   # collapse axis 1 → shape (2, 4)
c.sum(axis=(0, 2))              # collapse axes 0 and 2 → shape (3,)
The mnemonic: axis is the one that disappears.

Broadcasting in one picture

Broadcasting — three live examples

k = np.arange(6).reshape(2, 3)
#   [[0 1 2]
#    [3 4 5]]

k + [100, 200, 300]
# shape (2,3) + (3,) → broadcast row across both rows
#   [[100 201 302]
#    [103 204 305]]

k + [[100], [200]]
# shape (2,3) + (2,1) → broadcast column across cols
#   [[100 101 102]
#    [203 204 205]]

h = np.arange(5).reshape(1, 1, 5)
h + [10, 20, 30, 40, 50]      # (1,1,5) + (5,) → (1,1,5)

The broadcasting rule

Align shapes from the right. Each pair is compatible if equal, or one is 1.

ABResult
(2, 3)(3,)(2, 3)
(2, 1)(1, 3)(2, 3)
(4, 1, 3)(2, 3)(4, 2, 3)
(B, T, D)(D,)(B, T, D)
(3, 2)(3,)ERROR

Matrix multiplication

n1 = np.arange(10).reshape(2, 5)
n2 = np.arange(15).reshape(5, 3)

n1.dot(n2)          # 2×5 @ 5×3 → 2×3
n1 @ n2             # same thing, prettier syntax
@ is the dedicated matmul operator since Python 3.5. Use it.
n1 * n2 is element-wise — shapes must match. Different beast from @.

Linear algebra — the toolbox

import numpy.linalg as linalg

m = np.array([[1,  2,  3],
              [5,  7, 11],
              [21, 29, 31]])

linalg.inv(m)              # inverse
linalg.det(m)              # determinant
linalg.eig(m)              # eigenvalues & eigenvectors
linalg.qr(m)               # QR decomposition
linalg.svd(m)              # singular value decomposition

# Solve  Ax = b  for x
A = np.array([[2, 6], [5, 3]])
b = np.array([6, -9])
linalg.solve(A, b)         # [-3.  2.]

Worked example — standardize a data matrix

rng = np.random.default_rng(0)
X = rng.normal(loc=5, scale=2, size=(200, 5))

# Subtract column mean, divide by column std — in two lines
Z = (X - X.mean(axis=0)) / X.std(axis=0)

Z.mean(axis=0).round(4)     # ≈ [0. 0. 0. 0. 0.]
Z.std(axis=0).round(4)      # ≈ [1. 1. 1. 1. 1.]

Every preprocessing step in sklearn boils down to this kind of one-line broadcast.

Recap

Three things to leave with

  1. ndarray = contiguous typed buffer. That is the entire speed story.
  2. Shape is meaning. Memorize the axis mnemonic.
  3. If you typed for on numeric data, ask: can broadcasting do this?
