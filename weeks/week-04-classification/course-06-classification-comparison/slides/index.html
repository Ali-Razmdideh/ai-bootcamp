Part 2 · 0:20 – 0:40

Four Scenarios (ISLR Section 4.5)

which method wins and why

Scenario 1: Gaussian classes, similar covariance

  • True model: classes are Gaussian with the same Σ.
  • LDA = Bayes optimal — it matches the true generating model exactly.
  • LR works too but is less efficient (doesn't use the Gaussian structure).
  • QDA: wastes parameters estimating differences in Σ that don't exist.
  • NB: misses within-class feature correlations → worse boundary.

Winner: LDA. Runner-up: LR.

Scenario 2: Gaussian classes, different covariance

  • True model: classes have different Σ_k.
  • QDA = Bayes optimal — separate Σ_k matches the true model.
  • LDA: forces shared Σ → biased boundary. Gets worse as covariances diverge.
  • LR: linear boundary misses the quadratic structure.
  • Requirement: need enough n so Σ_k estimates are reliable (n > Kp).

Winner: QDA (when n is large). LDA wins if n is small.

Scenario 3: Non-Gaussian features

  • Features are skewed, bimodal, or heavy-tailed — not well-approximated by Gaussians.
  • LR: makes no distributional assumptions. Robust and consistent.
  • LDA/QDA: Gaussian assumption is violated → boundary is biased in an unpredictable way.
  • NB with KDE: non-parametric marginals can handle any shape.

Winner: LR (or NB with KDE marginals).

In practice, LDA is often surprisingly robust to non-Gaussian features — especially if the true boundary is near-linear. The key is whether the feature distribution is "bad enough" to warp the Gaussian-derived boundary substantially.

Scenario 4: High-dimensional, small n

  • p features, n samples with p comparable to or larger than n.
  • LDA: Σ is p×p — singular when p ≥ n. Cannot be inverted. Fails.
  • QDA: even worse — K separate Σ_k matrices.
  • LR with L1: works, selects important features automatically.
  • NB: always works — only estimates p·K marginal parameters. Scales linearly in p.

Winner: NB or L1-regularized LR. Classic LDA/QDA fail.

Benchmark & Decision Guide

AUC tables, decision flowchart, preview of Day 2

Benchmark: Default dataset

MethodAUC (5-fold CV)Notes
Logistic Regression0.950Baseline; robust
LDA0.948Very close; features ≈ Gaussian
QDA0.947Slight overfit; balanced, income not very different Σ
Naive Bayes0.932Slightly worse; balance–income correlation ignored

On Default, all four classifiers perform similarly because the features are approximately Gaussian and the boundary is approximately linear. This is the usual outcome in practice with well-behaved data.

This is a key pedagogical point: on well-behaved data, method choice matters less than one might think. The big differences emerge in the scenarios from Part 2 — extreme dimensionality, strong covariance differences, or heavily non-Gaussian features.

Decision flowchart

Is p >> n or p close to n?

→ YES: Use Naive Bayes or L1 Logistic Regression

→ NO: Are features approximately Gaussian?

→ YES: Do classes have the same covariance structure?

→ YES: Use LDA

→ NO, n is large: Use QDA

→ NO, n is small: Use LDA (variance too high for QDA)

→ NO (non-Gaussian): Use Logistic Regression

Is the boundary non-linear?

→ YES: Decision Trees, SVM, Neural Nets (Courses 7–9)

This flowchart is a guide, not a law. Always validate with cross-validation. The "winner" in a flow chart can underperform on specific datasets. The chart gives a starting point for model selection; CV gives the truth.

Preview: beyond these four

  • Decision Trees (Course 7): axis-aligned boundaries; interpretable; high variance alone.
  • Random Forests / Boosting (Course 8): ensemble of trees; flexible non-linear boundary; state of the art on tabular data.
  • SVMs (Course 9): maximum-margin linear boundary; kernel trick for non-linear; works in high-dimensional settings.

LR, LDA, QDA, and NB form the linear/generative baseline. Everything in Courses 7–9 should be compared against these baselines first.

Recap

Week 4 Classification — Complete

  1. KNN: non-parametric, local, sensitive to k and scale.
  2. Logistic Regression: linear log-odds, MLE, robust, no distributional assumptions.
  3. LDA: generative Gaussian with shared Σ → linear boundary, Fisher projection.
  4. QDA: separate Σ_k → quadratic boundary; needs large n.
  5. Naive Bayes: conditional independence → O(pK) parameters; wins at high p.
  6. Poisson regression: count data; log link; multiplicative effects; check overdispersion.
  7. No free lunch: no method wins everywhere. Use CV and the decision guide.
