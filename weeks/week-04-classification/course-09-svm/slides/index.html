Part 1 · 0:00 – 0:25

Hyperplanes & the Maximum-Margin Classifier

geometry, unique optima, and support vectors

What is a hyperplane?

  • In p dimensions a hyperplane is a flat affine subspace of dimension p − 1.
  • Equation: β₀ + β₁X₁ + β₂X₂ + … + βₚXₚ = 0
  • In 2-D this is a line. In 3-D it is a plane.
  • β = (β₁,…,βₚ) is the normal vector — perpendicular to the hyperplane surface.
  • Points with f(X) > 0 lie on one side; f(X) < 0 on the other. The signed distance from a point to the hyperplane is proportional to |f(X)|.
The signed value f(x) tells us how far a point is from the hyperplane and on which side. This is the key quantity SVM uses for classification.

Separating hyperplanes

  • Code classes as Y = +1 and Y = −1.
  • A separating hyperplane satisfies yᵢ · f(xᵢ) > 0 for all i.
  • When a separating hyperplane exists, there are infinitely many of them (left panel).
  • The maximal-margin hyperplane is the unique one with the widest gap (right panel, dashed boundary lines).

Any hyperplane satisfying yᵢf(xᵢ) > 0 is valid. We want the best one — furthest from both classes simultaneously.

The Maximal Margin Classifier

  • The margin M is the perpendicular distance from the boundary to the nearest point of each class.
  • MMC solves:
maximize M   subject to:
  ‖β‖ = 1
  yᵢ(β₀ + β·xᵢ) ≥ M   for all i
  • Equivalent to: minimize ½‖β‖² subject to yᵢ(β₀ + β·xᵢ) ≥ 1.
  • This is a convex quadratic program — one unique global solution.
  • The support vectors are the points that lie exactly on the margin; they define the boundary. All other points can move without changing the solution.
The constraint ‖β‖=1 pins the scale so that f(xᵢ) equals the signed distance to the boundary. The margin is 2/‖β‖ so minimizing ‖β‖² maximizes the margin.

Soft Margin — The Support Vector Classifier

    
allowing violations in exchange for robustness

  


  

    
When the hard margin fails

    
    
      
  • Real data is almost never perfectly separable — classes overlap.
    • 
      
  • Even when separable, a noisy outlier forces the margin very narrow, producing high variance.
    • 
      
  • The support vector classifier allows some points to violate the margin (or even the wrong side of the boundary) in exchange for a wider, more stable gap.
    • 
    

  


  

    
Soft margin: the optimization

    

      

        

          maximize M

          subject to: ‖β‖ = 1

            yᵢ(β₀ + β·xᵢ) ≥ M(1 − εᵢ)

            εᵢ ≥ 0,   Σεᵢ ≤ C
        

        
    
          
  • εᵢ (slack) measures how much observation i violates the margin.
    • 
          
  • εᵢ = 0: on the correct side of the margin.
    • 
          
  • 0 < εᵢ ≤ 1: inside the margin but on the correct side.
    • 
          
  • εᵢ > 1: on the wrong side of the boundary (misclassified).
    • 
        

      

      

        
    
          
  • C is the total budget for slack violations. It is a regularization parameter.
    • 
          
  • Small C: tight budget → narrow margin → few support vectors → high variance (overfits).
    • 
          
  • Large C: loose budget → wide margin → many support vectors → more bias (underfits).
    • 
          
  • Tune C with cross-validation.
    • 
        

      

    

    
sklearn's SVC uses the inverse of C from ISLR — sklearn's C is roughly 1/λ. So sklearn small C = more regularization = ISLR large C. This trips people up constantly.

  


  

    
C is a regularization parameter

    
Four values of C on the same data: as C decreases the margin widens and more points are allowed to violate it.

    
Small C (top-left): very wide margin, many violations. Large C (bottom-right): tight margin, fewer violations but higher variance. Tune with CV.

    Part 3 · 0:50 – 1:20
    
Kernels

    
non-linear boundaries without explicit feature expansion

  


  

    
When linear boundaries fail

    
    
      
  • Sometimes no value of C makes a linear boundary work — the classes are not linearly separable in any rotation.
    • 
      
  • Classic example: concentric circles (one class inside the other).
    • 
      
  • Two remedies:
    • 
    

    
    
      
  1. Feature expansion: explicitly add X₁², X₂², X₁X₂, … to the predictor set.
    2. 
      
  2. The kernel trick: implicitly compute the expanded inner products without ever constructing the features.
    3. 
    

  


  

    
Feature expansion

    
    
      
  • Go from p-dimensional feature space to M > p dimensions by adding transformations.
    • 
      
  • Example with p = 2 — add quadratic terms (X₁², X₂², X₁X₂):
    • 
    

    

      β₀ + β₁X₁ + β₂X₂ + β₃X₁² + β₄X₂² + β₅X₁X₂ = 0
    

    
    
      
  • A linear SVC in this 5-D space produces a conic section boundary (ellipse, parabola, …) in the original 2-D space.
    • 
      
  • Problem: high-degree expansions blow up in dimensionality. Degree d with p predictors → C(p+d, d) basis functions.
    • 
      
  • Kernel trick avoids building the expanded features explicitly.
    • 
    

  


  

    
The kernel trick — inner products

    

      

        
    
          
  • The SVC solution can be written using only inner products between training points:
    • 
        

        

          f(x) = β₀ + Σᵢ αᵢ ⟨x, xᵢ⟩
        

        
    
          
  • Most α̂ᵢ = 0. Only support vectors (i ∈ S) have α̂ᵢ > 0.
    • 
          
  • So: f(x) = β₀ + Σᵢ∈S α̂ᵢ ⟨x, xᵢ⟩
    • 
        

      

      

        
    
          
  • Replace the inner product ⟨xᵢ, xᵢ'⟩ with a kernel function K(xᵢ, xᵢ').
    • 
          
  • The kernel computes the inner product in a (possibly huge) expanded space — without constructing that space.
    • 
          
  • Solution: f(x) = β₀ + Σᵢ∈S α̂ᵢ K(x, xᵢ)
    • 
        

      

    

  


  

    
Three kernels

    
    
      
  • 
        Linear: K(xᵢ, xᵢ') = Σⱼ xᵢⱼ xᵢ'ⱼ — standard dot product; same as SVC without expansion.
      
    • 
      
  • 
        Polynomial: K(xᵢ, xᵢ') = (1 + Σⱼ xᵢⱼ xᵢ'ⱼ)^d — encodes all monomials up to degree d. C(p+d,d) implicit features. Rigid curves.
      
    • 
      
  • 
        Radial (RBF): K(xᵢ, xᵢ') = exp(−γ Σⱼ (xᵢⱼ − xᵢ'ⱼ)²) — infinite-dimensional implicit space. Very flexible. The default choice.
        
      
          
    • Small γ: wide kernel, smooth boundary (more bias).
      • 
          
    • Large γ: narrow kernel, wiggly boundary (more variance).
      • 
        
    
      
    • 
    

    
The RBF kernel is local — it gives high similarity only when two points are close together. Large γ makes "close" very strict, so the boundary hugs individual points.

    Part 4 · 1:20 – 1:40
    
ROC Curves, Multi-class & p ≫ n

    
evaluating SVMs, extending beyond two classes, the Khan lab

  


  

    
ROC curves

    
    
      
  • The SVC predicts by the sign of f̂(x). By varying a threshold t in f̂(x) > t we trade off true positive rate (sensitivity) and false positive rate (1 − specificity).
    • 
      
  • Plotting TPR vs FPR as t varies gives the ROC curve. Area under it (AUC) summarises performance: AUC = 1 is perfect, AUC = 0.5 is random.
    • 
      
  • SVM with larger γ fits the training ROC curve better but may overfit on test data.
    • 
    

  


  

    
SVMs: more than two classes

    

      

        
OVA — One vs All

        
    
          
  • Fit K separate 2-class SVMs, each pitting class k against the remaining K−1.
    • 
          
  • Classify x* to the class with the largest f̂ₖ(x*).
    • 
          
  • K models, each trained on all n samples.
    • 
        

      

      

        
OVO — One vs One

        
    
          
  • Fit C(K,2) = K(K−1)/2 binary classifiers, one for each pair of classes.
    • 
          
  • Classify x* to the class that wins the most pairwise competitions.
    • 
          
  • More classifiers, but each sees fewer samples.
    • 
        

      

    

    
When K is small, OVO is generally preferred — sklearn's SVC uses OVO by default.

  


  

    
Khan — 63 cases, 2308 genes

    
    
      
  • Four cancer subtypes. More features than samples (p ≫ n).
    • 
      
  • Unregularised logistic regression diverges — infinitely many solutions.
    • 
      
  • Linear SVM thrives: the margin objective acts as regularisation, picking the minimum-norm β out of all solutions.
    • 
      
  • Use a linear kernel — the data is already very high-dimensional, so we do not need additional non-linearity.
    • 
      
  • Tune C with 3-fold CV (n = 63 is too small for 5-fold).
    • 
    

  


  
  

    
Khan gene-expression SVM

    
Load the Khan training/test split, fit a linear SVC, and GridSearchCV for C. Notice the model perfectly separates 4 cancer subtypes with 2308 features and only 63 training samples.


    

      
      
      
    

    

  








  

    Part 5 · 1:40 – 2:00
    
SVM vs Logistic Regression

    
hinge loss, connections, and when to choose which

  


  

    
SVM as loss + penalty

    
The SVC objective can be rewritten as:

    

      minimize   Σᵢ max[0, 1 − yᵢf(xᵢ)]  +  λ‖β‖²
    

    
    
      
  • The term max[0, 1 − yᵢf(xᵢ)] is the hinge loss. It is zero when yᵢf(xᵢ) ≥ 1 (correct and confident).
    • 
      
  • Logistic regression uses log-loss: log(1 + e^{−yᵢf(xᵢ)}), which is always positive.
    • 
      
  • Both add an L2 penalty λ‖β‖². The only difference is the loss function shape.
    • 
      
  • Hinge loss is exactly zero for correctly classified, confident points — this produces sparse solutions (only SVs matter).
    • 
    

  


  

    
Which to use?

    
    
      
  • Nearly separable classes? SVM and LDA outperform logistic regression.
    • 
      
  • Classes overlap substantially? LR with ridge and SVM give very similar results.
    • 
      
  • Need class probabilities? Use logistic regression — SVMs do not produce calibrated probabilities natively.
    • 
      
  • Non-linear boundary? Kernel SVMs are a natural choice. Kernels work with LR too, but are computationally more expensive.
    • 
      
  • p ≫ n (text, genomics)? Linear SVM is the standard tool.
    • 
    

  


  
  

    
Hinge loss vs log-loss

    
Plot both loss functions against the margin yf(x). Note that hinge loss is exactly 0 for confident correct predictions, while log-loss is always positive.


    

      
      
      
    

    

  








  Recap
  
Five things to leave with

  
    
    
  1. Hyperplanes separate. The MMC finds the unique widest gap; support vectors define it.
    2. 
    
  2. Soft margin adds robustness. C (sklearn) trades off margin width vs violations. Large C = tight margin = high variance.
    3. 
    
  3. Kernels give non-linearity for free. Polynomial and RBF kernels implicitly work in huge feature spaces via inner products.
    4. 
    
  4. Multi-class: use OVO. Fit K(K−1)/2 binary SVMs; classify by majority vote.
    5. 
    
  5. SVM for p ≫ n. Linear SVM is the go-to when features massively outnumber samples. SVM ≈ LR when classes overlap.
