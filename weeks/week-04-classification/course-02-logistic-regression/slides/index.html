Part 1 · 0:00 – 0:20

Why not linear regression for classification?

the fundamental problem

The naive approach — just fit a line

Encode Y = 1 (default) / 0 (no default). Fit OLS: Ŷ = β₀ + β₁ × balance.

  • At balance = 0, prediction = −0.08 → negative probability.
  • At balance = 2700, prediction = 1.05 → probability above 1.
  • The 0/1 response forces OLS to fit a tilted line through a cloud of 0s and 1s.
Show Figure 4.1 from ISLR. The left panel shows the linear fit — clearly inadequate. The right panel shows the logistic fit. The key insight: probabilities must live in [0,1] by definition.

The fix: the sigmoid function

P(y=1|x) = σ(z) = 1 / (1 + e−z),   z = β₀ + β₁x

  • σ(z) → 0 as z → −∞, σ(z) → 1 as z → +∞.
  • Always in (0, 1) — a valid probability everywhere.
  • S-shaped: the model is most uncertain near p = 0.5, most confident far from 0.5.
The sigmoid is the inverse of the logit (log-odds) function. They are the same model viewed from two angles.

Summary: linear vs logistic on Default

MethodPredictions at extremesInterpretable?Valid probabilities?
Linear regressionNegative / above 1Yes (slopes)No
Logistic regressionAlways in (0, 1)Yes (odds ratios)Yes

For a binary outcome, always use logistic regression. Linear regression can still be used for multi-class in some edge cases but logistic is the right default.

Linear vs logistic: probability bounds

    Part 2 · 0:20 – 0:50
    
Binary Logistic Regression

    
model, MLE, coefficients, confounding

  


  

    
The model: log-odds formulation

    

      P(y=1|x) = eβ₀ + β₁x / (1 + eβ₀ + β₁x)
    

    

      log(p / (1−p)) = β₀ + β₁x     (log-odds = logit)
    

    
    
      
  • The logit is linear in x — same structure as linear regression, different scale.
    • 
      
  • p / (1−p) = the odds: how much more likely is success than failure.
    • 
      
  • Odds = 1 → p = 0.5. Odds = 4 → p = 0.8. Odds = 9 → p = 0.9.
    • 
    

  


  

    
Maximum likelihood estimation

    

      ℓ(β) = Σᵢ [yᵢ log p(xᵢ) + (1−yᵢ) log(1 − p(xᵢ))]
    

    
    
      
  • The log-likelihood is concave in β — any gradient method finds the unique global maximum.
    • 
      
  • No closed-form solution (unlike OLS), but Newton–Raphson / IRLS converges quickly.
    • 
      
  • sklearn uses LBFGS by default; result is numerically the same as statsmodels GLM.
    • 
    

    
Concavity is a huge practical gift. It means we never have to worry about local minima. Contrast with neural nets.

  


  

    
Interpreting the coefficients: odds ratios

    
eβ₁ is the multiplicative change in odds for a one-unit increase in x₁.

    

      
Predictorβ̂Std Errorz-statp-value

      
Intercept−10.6510.361−29.49<0.001

      
balance0.005500.00022824.09<0.001

    

    
    
      
  • e0.0055 ≈ 1.0055 → each $1 increase in balance multiplies odds by 1.0055.
    • 
      
  • e100 × 0.0055 ≈ 1.73 → each $100 increase multiplies odds by 1.73×.
    • 
      
  • z-statistic = β̂ / SE. Large |z| → strong evidence against β = 0.
    • 
    

    
Table 4.1 from ISLR. The z-statistic plays the role of t-statistic in linear regression. Under H₀: β=0, z ~ N(0,1).

Confounding: the student paradox

    

      

        
Simple regression (student only)

        
β_student = +0.405, p < 0.001

        
Raw data: students default more.

      

      

        
Multiple regression (+ balance)

        
β_student = −0.647, p = 0.001

        
Controlling for balance: students default less.

      

    

    
Students carry higher balances on average. Balance drives defaults. Once balance is controlled, being a student is protective. Confounding reverses the sign.

    
Figure 4.3 from ISLR. This is one of the most important statistical lessons in the course. Marginal associations can be completely reversed by adding a third variable. This is Simpson's paradox in action.

    Part 4 · 1:10 – 1:25
    
Case-Control Sampling

    
biased sampling and the intercept correction

  


  

    
What is case-control sampling?

    
    
      
  • In medicine, disease events (cases) are rare. Collecting natural data takes years.
    • 
      
  • Instead: deliberately oversample cases and undersample controls.
    • 
      
  • Example: disease prevalence π = 1%; study samples π̃ = 50% cases. Intercept is now biased.
    • 
    

  


  

    
The fix: intercept correction

    
β̂₀corrected = β̂₀ + log(π̃ / (1−π̃)) − log(π / (1−π))

    
    
      
  • Only the intercept changes. All other β̂ⱼ are unbiased — they are not affected by case-control sampling.
    • 
      
  • π = true population prevalence (known from external sources).
    • 
      
  • π̃ = sampling fraction of cases in the study.
    • 
    

    
This is one reason why logistic regression is the workhorse of epidemiology. It's robust to case-control designs.

    
This result is exact for logistic regression — not an approximation. It follows directly from the form of the log-likelihood. No other common classifier has this property.

  







  

    Part 5 · 1:25 – 1:50
    
Multinomial Logistic Regression

    
softmax for K > 2 classes

  


  

    
Softmax generalizes the sigmoid

    
P(y = k | x) = eβkTx / Σj=1K eβjTx

    
    
      
  • K weight vectors β₁, …, βK. One per class.
    • 
      
  • Outputs sum to 1 — a valid probability distribution over K classes.
    • 
      
  • Only K−1 are identifiable (one is the reference class). sklearn handles this automatically.
    • 
      
  • For K = 2: softmax reduces exactly to the binary sigmoid.
    • 
    

  


  

    
One-vs-rest (OvR) vs multinomial

    
    
      
  • OvR: fit K separate binary classifiers (class k vs not-k). Simpler but probabilities don't sum to 1.
    • 
      
  • Multinomial (softmax): fit all K classes jointly. Probabilities sum to 1 by construction.
    • 
      
  • Multinomial is almost always better — especially when classes overlap or are unbalanced.
    • 
    

  


  

    
Iris demo — 3-class decision regions


Run the live example below.

    
Decision regions: contourf on petal_length × petal_width. Three linear boundaries meet at a point — the characteristic signature of softmax.

    
The three classes in Iris are: setosa (easily separated), versicolor, virginica (overlap). Two features (petal dims) are enough for very high accuracy. With 4 features accuracy is ~97%.

  


  

    
Softmax decision boundary geometry

    
    
      
  • Between any two classes k and l: boundary is where P(y=k|x) = P(y=l|x).
    • 
      
  • This simplifies to: (βk − βl)ᵀx = 0 — a hyperplane.
    • 
      
  • With K classes, you get K(K−1)/2 pairwise boundaries, all linear.
    • 
      
  • The full decision space is always a Voronoi-like polytope — convex regions.
    •

    Part 6 · 1:50 – 2:15
    
ROC Curves, AUC & Threshold Tuning

    
predict_proba, not just predict

  


  

    
The confusion matrix

    

      
Predicted: NoPredicted: Yes

      
Actual: NoTrue Negative (TN)False Positive (FP)

      
Actual: YesFalse Negative (FN)True Positive (TP)

    

    
    
      
  • FPR = FP / (FP + TN) — fraction of negatives incorrectly flagged. (Type I error)
    • 
      
  • FNR = FN / (FN + TP) — fraction of positives missed. (Type II error)
    • 
      
  • TPR = Recall = TP / (TP + FN) = 1 − FNR
    • 
    

    
For the Default dataset, false negatives (missed defaulters) are usually more costly than false positives (unnecessary interventions). The right threshold depends on this cost ratio — not blindly on accuracy.

  


  

    
predict_proba vs predict


Use predict_proba(X_test)[:, 1] for scores, predict() for hard labels (threshold=0.5).

    
    
      
  • predict_proba returns the continuous score. predict applies a hard 0.5 cutoff.
    • 
      
  • On imbalanced data (few defaults), 0.5 is almost never the right threshold.
    • 
      
  • Lower threshold → classify more as positive → higher recall, lower precision.
    • 
    

  


  

    
ROC curve

    
    
      
  • Sweep threshold from 1 → 0. At each threshold compute (FPR, TPR).
    • 
      
  • Plot TPR vs FPR: the Receiver Operating Characteristic curve.
    • 
      
  • Diagonal = random classifier (AUC = 0.5). Perfect = top-left corner (AUC = 1.0).
    • 
      
  • AUC = area under the ROC curve = P(score(pos) > score(neg)). Model-selection metric.
    • 
    


  


  

    
Threshold tuning for F1


    
F1 = 2PR/(P+R) balances precision and recall. On Default, the optimal threshold is ≈0.15, not 0.5 — because defaults are rare and missing them is costly.

    
The right threshold depends on the cost ratio. In fraud detection, you want very high recall at the cost of more false positives. In cancer screening, same. In spam filtering, you'd prefer high precision. Always calibrate to business costs.

    Part 7 · 2:15 – 2:40
    
L1 and L2 Regularization

    
C = 1/λ — the Week 3 story revisited

  


  

    
The regularized objective

    
maximize   ℓ(β) − λ·penalty(β)

    
    
      
  • L2 (Ridge): penalty = ½ Σⱼ β²ⱼ — shrinks all coefficients toward zero smoothly.
    • 
      
  • L1 (Lasso): penalty = Σⱼ |βⱼ| — zeros some coefficients exactly (sparse solution).
    • 
      
  • sklearn uses C = 1/λ. Small C = large λ = strong regularization.
    • 
      
  • Default in sklearn: penalty='l2', C=1.0.
    • 
    

    
The same geometry from Week 3 applies. L2 = Gaussian prior on β. L1 = Laplace prior on β. C = 1 is not "no regularization" — it's moderate regularization. Use C=1e6 to approximate unregularized LR.

  


  

    
Coefficient paths — L2 vs L1

    
Vary log C (x-axis). Plot each βⱼ for class "setosa" (4 features).

    
    
      
  • L2 paths: all coefficients shrink smoothly toward zero as C → 0. None exactly zero.
    • 
      
  • L1 paths: coefficients drop to exactly zero one at a time as C decreases. Sparse.
    • 
      
  • Same Lasso geometry from Week 3 — the diamond constraint corner causes exact zeros.
    • 
    


    
Use 'saga' or 'liblinear' solver for L1. The default 'lbfgs' only supports L2. This is a common gotcha.

  


  

    
When to regularize and how to choose C

    
    
      
  • Always standardize features before regularizing — otherwise large-scale features are penalized less.
    • 
      
  • Use cross-validation to choose C: LogisticRegressionCV or GridSearchCV.
    • 
      
  • L1 for feature selection (want sparse β). L2 for pure regularization. Elastic-net for both.
    • 
      
  • L1 with Lasso: C = 0.05 on Iris reduces 4 features — some classes drop to 1–2 non-zero features.
    • 
    

  


  
  

    
L1 vs L2: coefficient paths

    
Generate data with 10 features (5 informative, 5 pure noise). Vary C from 0.001 to 100. See how L1 zeroes out noise features while L2 only shrinks them.


    

      
      
      
    

    


  







  

    Part 8 · 2:40 – 3:00
    
When Linear Fails

    
the wall that motivates what comes next

  


  

    
The make_moons dataset


    
    
      
  • Two interleaved half-moons — no straight line can separate them.
    • 
      
  • A linear decision boundary is geometrically limited: it cuts the space with a hyperplane.
    • 
      
  • Accuracy caps at ~85% no matter how much data you have.
    • 
    

  


  

    
Three exits from the linear wall

    
    
      
  • Non-linear features: add x₁², x₁x₂, polynomial terms → still LR, richer features. (Week 3: moving beyond linearity)
    • 
      
  • Kernel trick: SVM with RBF kernel maps to infinite-dimensional feature space implicitly. (Course 9)
    • 
      
  • Non-linear model: Decision trees, random forests, neural nets. (Courses 7–8)
    • 
    

    
Logistic regression is not the end — it is the baseline. Every non-linear model you meet next should be compared against it.

  


  

    
LDA as a partial fix — and motivation

    
    
      
  • LDA models the full class-conditional distribution — can naturally capture some curvature via Gaussian shapes.
    • 
      
  • QDA: separate covariance per class → curved (quadratic) boundaries.
    • 
      
  • This is exactly what Course 3 covers next.
    • 
    

  


  
  

    
The make_moons wall

    
Fit logistic regression on two interleaved half-moons. See the accuracy cap at ~85% regardless of iterations or regularisation — a linear boundary can't solve this.

  1. Linear regression can't model probabilities. Always use logistic for binary Y.
    2. 
    
  2. Log-odds = β₀ + β₁x. eβ is the odds ratio per unit of x.
    3. 
    
  3. MLE is concave. Any optimizer finds the unique global optimum.
    4. 
    
  4. Confounding reverses signs. Marginal and partial associations can disagree completely.
    5. 
    
  5. Case-control? Fix the intercept — all other β̂ are unbiased.
    6. 
    
  6. Softmax generalizes sigmoid to K classes. K−1 identifiable weight vectors.
    7. 
    
  7. Tune the threshold. 0.5 is almost never right on imbalanced data. Use AUC to compare models.
    8. 
    
  8. C = 1/λ. L1 zeroes coefficients. L2 shrinks them. Always standardize first.
