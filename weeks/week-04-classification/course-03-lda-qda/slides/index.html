Part 1 · 0:00 – 0:30

Why LDA?

Bayes theorem for classification

Two philosophies of classification

ApproachWhat is modelledMethod
DiscriminativeP(Y=k | X=x) directlyLogistic regression
GenerativeP(X=x | Y=k) + priorsLDA, QDA, Naive Bayes
  • Generative models: model each class separately, then apply Bayes theorem.
  • More assumptions → better performance when assumptions hold, worse when they don't.
  • LDA wins over LR: well-separated classes (LR coefficients blow up), small n, Gaussian data.
The instability of logistic regression with perfect or near-perfect separation is a real practical problem. LDA handles this gracefully because it doesn't try to push coefficients to ±∞.

Bayes theorem for classification

Pr(Y=k | X=x) = πk · fk(x) / Σl πl · fl(x)

  • πk = prior probability of class k = P(Y=k). Estimated as nk/n.
  • fk(x) = class-conditional density = P(X=x | Y=k). This is what LDA models.
  • Denominator = total probability P(X=x). It doesn't depend on k — only needed for normalization.

The Bayes classifier is optimal when the distributions are known exactly. LDA approximates them with Gaussians.

The Bayes error rate is the irreducible minimum — no classifier, however complex, can beat it if the features are fixed. LDA with correct Gaussian assumptions achieves it asymptotically.

Prior shifts the decision boundary

Equal priors (π₁=π₂=0.5) → boundary at midpoint. Unequal priors → boundary shifts toward the rarer class.

Equal priors → midpoint boundary

Unequal priors: boundary shifts toward rare class

Rare class prior (small πk) penalises that class — boundary moves toward it so you predict it only when very confident.

This is ISLR Figure 4.4. The log(π_k/π_l) term in the LDA decision boundary equation is exactly what causes this shift. If class 2 is 10× rarer, you need to be 10× more confident before predicting it.

1D LDA — two overlapping Gaussians

p=1: one feature, two classes. The cut point is where the two density curves cross (adjusted for priors).

Histogram of training data (bars) and true density curves. Purple overlap = irreducible Bayes error.

In p=1 the optimal cut point is (μ₁+μ₂)/2 when π₁=π₂ and σ₁=σ₂. Otherwise it shifts — exactly the log(π₁/π₂) / (μ₂−μ₁)·σ² correction.

ISLR Figure 4.4 (left panel). The overlap is the minimum achievable error — even the Bayes classifier is wrong for observations that fall in this region.

Multivariate Gaussian density

fk(x) = (2π)−p/2 |Σ|−½ exp(−½ (x−μk)TΣ−1(x−μk))

  • p = number of features (dimension of x).
  • |Σ| = determinant of covariance — normalisation constant.
  • (x−μ)TΣ−1(x−μ) = Mahalanobis distance — measures how far x is from μ, corrected for correlations.

When we take the log and compare across classes, the (2π)−p/2|Σ|−½ factor is constant under shared Σ and disappears. Only the Mahalanobis term and log πk remain.

The Mahalanobis distance is the key concept here. It generalises Euclidean distance to account for the shape and orientation of the Gaussian cloud. If Σ=I then it reduces to Euclidean distance.

Gaussian LDA: the assumption

fk(x) = N(μk, Σ)

  • Each class has its own mean vector μk (center of the Gaussian).
  • Σ is shared across all K classes — same covariance matrix.
  • This shared-Σ assumption is the key distinguishing feature of LDA vs QDA.
The shared Σ assumption is what makes the boundary linear (as we'll see in the next slide). QDA relaxes this assumption.

Discriminant score — deriving the linear boundary

Taking log of the Bayes posterior (log-monotone transform preserves argmax):

log Pr(Y=k|X=x) = log πk + log fk(x) − const

δk(x) = xTΣ−1μk − ½ μkTΣ−1μk + log πk

δk(x) is linear in x. The quadratic term xTΣ−1x cancels because Σ is shared across all k.

This cancellation is the entire algebra of LDA. Write out the full Gaussian log-density: -½log|Σ| - ½(x-μ)ᵀΣ⁻¹(x-μ) + log π. Expanding (x-μ)ᵀΣ⁻¹(x-μ) = xᵀΣ⁻¹x - 2μᵀΣ⁻¹x + μᵀΣ⁻¹μ. The xᵀΣ⁻¹x is the same for all k → it cancels in comparisons. Left with δ_k linear in x.

Why the boundary is linear

Assign x to class k = argmaxk δk(x).

The decision boundary between class k and class l is where δk(x) = δl(x):

xTΣ−1(μk − μl) = ½(μkTΣ−1μk − μlTΣ−1μl) − log(πk/πl)

  • Left side: linear in x → equation of a hyperplane.
  • The boundary is a straight line (2D), plane (3D), or hyperplane (p-D).
  • In p=1 dimensions: a single cut point on the real line.

p=2, K=3 — Bayes vs LDA boundaries

Schematic: K=3 classes, shared Σ (similar ellipses)

Bayes boundaries (dashed): optimal, known only if true distribution known.

LDA boundaries (solid): estimated from data using plug-in μ̂k, Σ̂. Converge to Bayes as n→∞.

  • With K=3 there are 3 pairwise boundaries meeting at a triple point.
  • The shared Σ assumption means all three ellipses have the same shape — the boundaries are straight.
  • With finite n, LDA boundaries deviate from Bayes — this is estimation error.
This is Figure 4.6 from ISLR. The dashed lines are the true Bayes boundaries computed from the known generating distributions. The solid lines are what LDA estimates from the data. With more data, the solid lines converge to the dashed ones.

From δk(x) to posterior probabilities

The posterior is a softmax over the discriminant scores:

Pr(Y=k | X=x) = exp(δk(x)) / Σl=1K exp(δl(x))

  • This is exactly the softmax function — the same one used in neural network output layers.
  • For K=2 it reduces to the logistic sigmoid: Pr(Y=1|x) = 1/(1 + exp(δ₁(x)−δ₂(x))).
  • Probabilities sum to 1 across all classes by construction.

LDA and logistic regression both produce linear boundaries and use the softmax form. The difference is how the coefficients are found: generative fit (LDA) vs maximum likelihood on the boundary directly (LR).

This connection to softmax is underemphasised in most textbooks. It shows that LDA is, in a sense, a special case of linear multi-class classification — just with a specific closed-form way of estimating the weights.

Fisher's linear discriminant projection

    
Find the direction w that maximises the Fisher criterion:

    

      J(w) = (wTSBw) / (wTSWw)
    

    
SB = between-class scatter matrix; SW = within-class scatter matrix.

    
    
      
  • Solution: w ∝ SW−1(μ₁ − μ₂) — the same direction as LDA.
    • 
      
  • Project all points onto w → 1D histogram; classes should separate well.
    • 
      
  • With K classes, p features: up to min(K−1, p) discriminant axes.
    • 
    


    
Figure 4.6 in ISLR shows the 1D projection for LDA on the Default dataset. The two histograms (default vs no-default) have significant overlap, which is why LDA doesn't achieve perfect separation here.

QDA — Quadratic Discriminant Analysis

    
separate covariance matrices → curved boundaries

  


  

    
Relaxing the shared-Σ assumption

    
    
      
  • LDA: fk(x) = N(μk, Σ) — shared covariance. Linear boundary.
    • 
      
  • QDA: fk(x) = N(μk, Σk) — separate covariance per class. Quadratic boundary.
    • 
    

    

      δk(x) = −½ log|Σk| − ½(x−μk)TΣk−1(x−μk) + log πk
    

    
The term xTΣk−1x no longer cancels (Σk differs by class). The discriminant score is quadratic in x → curved decision boundary.

  


  
  

    
LDA: linear boundary vs QDA: curved boundary

    

      

        
        
Same Σ → ellipses have same shape → straight line

      

      

        
        
Different Σₖ → different ellipse shapes → curved boundary

      

    

    
The boundary shape mirrors the difference in covariance — if one class is elongated and another is round, the boundary that separates them must curve.

    
This is the intuition behind Figure 4.9 in ISLR. When two Gaussians have the same shape (shared Σ), the equal-density contour between them is a straight line. When they have different shapes, it becomes a quadric surface.

  


  

    
Parameter count: LDA vs QDA

    

      
MethodParameters for ΣWhen to prefer

      
LDAp(p+1)/2 sharedSmall n, linear boundary, similar class shapes

      
QDAK · p(p+1)/2 totalLarge n, curved boundary, different class shapes

    

    
    
      
  • With p=4 features, K=3 classes: LDA has 10 Σ params; QDA has 30.
    • 
      
  • QDA is more flexible (lower bias) but needs more data to estimate reliably (higher variance).
    • 
      
  • Classic bias-variance tradeoff: more complex model needs more data to pay off.
    • 
    

    
When n is large, QDA consistently outperforms LDA if the true boundaries are curved. When n is small, the extra variance of QDA's estimate hurts more than the bias of LDA's wrong assumption.
