Part 1 · 0:00 – 0:30

Regression trees

stratifying the predictor space

The Hitters dataset

Baseball players, predict log(salary) from Years in MLB and Hits last season.

How would you split this space into rectangles to predict salary?

Eye-balled: young players earn low, regardless of hits. Veterans split by hits — the heavy hitters earn the most.

The fitted Hitters tree

  • Three regions, three predictions (mean of training salaries in each).
  • R₁ = rookies (Years < 4.5) → log(salary) ≈ 5.11 (~$165k).
  • R₂ = veterans, fewer hits → 6.00 (~$400k).
  • R₃ = veterans, big hitters → 6.74 (~$845k).

Years is the most important predictor; hits matter only once you're past the rookie cutoff.

The algorithm: recursive binary splitting

  1. Consider every feature and every threshold.
  2. Pick the (Xj, s) that most reduces RSS:

    RSS = Σm Σi∈Rm (yi − ŷRm)²

  3. Recurse on each child region.
  4. Stop when leaves contain < min_samples or pure.

Top-down, greedy. Best split now, not best subsequence overall — exhaustive search is NP-hard.

The induced partition

Recursive binary splits → axis-aligned rectangles.

Trees cannot make oblique cuts. A diagonal class boundary requires a staircase of axis-aligned splits — a weakness that ensembles partly fix.

Classification trees & pruning

    
Gini, entropy, cost-complexity α

  


  

    
Same algorithm, different criterion

    
For a classification leaf, we don't have an MSE. We have class proportions p̂mk. Three measures of "node purity":

    

      

        
Misclassification error

        
1 − maxk p̂mk

        
Intuitive, but not sensitive enough for growing.

      

      

        
Gini impurity

        
Σk p̂mk(1 − p̂mk)

        
Default in sklearn.

      

      

        
Cross-entropy

        
− Σk p̂mk log p̂mk

        
Numerically similar to Gini.

      

    

    
Gini and entropy are smooth and pick more informative splits than raw error rate. In practice they agree on almost everything.

Pre-pruning is too short-sighted

    
"Stop growing once the next split gains less than ε" — tempting but bad. A useless split now can enable a great split next.

    
Better: grow a very large tree, then prune it back.

  


  

    
Cost-complexity pruning

    

      Cα(T) = Σleaves m Σi∈Rm (yi − ŷm)² + α · |T|
    

    

      

        
α = 0

        
Keep every leaf. Massive overfit.

      

      

        
α moderate

        
Sweet spot. Cross-validate.

      

      

        
α → ∞

        
Collapse to root — predict the mean.

    4.

How trees carve the space

    
geometry, impurity, pruning, depth

  


  

    
A tree partitions 2D space

    
    
Every leaf corresponds to one axis-aligned rectangle. ISLR Fig 8.3.

  


  

    
Recursive binary splitting

    
    
Greedy: best single split, repeat — never undo an earlier choice.

  


  

    
Why staircase boundaries

    
    
Each split is parallel to an axis — diagonals require many splits.

  


  

    
Gini / Entropy / Misclassification

    
    
Gini and entropy are smooth and concave; both reward purer splits than misclassification. ISLR Fig 8.6.

  


  

    
Greedy split scoring

    
    
Try every threshold; keep the one that drops impurity the most. Repeat on each child.

  


  

    
Cost-complexity pruning path

    
    
Sweep α: tree shrinks monotonically; CV error dips, then rises. Pick the α at the dip.

  


  

    
Depth = under / just / over

    
    
Bias drops then variance explodes — depth is the knob.

    Part 3 · 1:00 – 1:30
    
Strengths, weaknesses, & ensembles

    
why one tree is rarely enough

  


  

    
Why teams love trees

    
    
      
  • Interpretable end-to-end. Every prediction is a flowchart. Mirrors how humans reason.
    • 
      
  • Mixed dtypes, no scaling. Numeric + categorical, missing values handled, no StandardScaler needed.
    • 
      
  • Native non-linearity & interactions. No polynomial features or kernels required.
    • 
      
  • Cheap inference. A trained tree is just a chain of ifs.
    • 
    

  


  

    
Trees vs linear models

    

      

        
      

      

        
      

    

    
Models match the geometry they assume. Pick to fit the data, not the textbook.

  


  

    
The weakness: variance

    
Refit a tree on a bootstrap sample of the same data.

    
    
      
  • The root split may change entirely.
    • 
      
  • Downstream splits change in cascade.
    • 
      
  • Borderline cases jump between leaves — predictions wobble.
    • 
    

    
A single tree is unstable. The greedy top-down search is sensitive to small changes in the data.

  


  

    
Bootstrap → four different trees

    
Refit a depth-3 classifier on four bootstrap resamples of the same Carseats-like data. Same data, four different first splits — that variance is what ensembles attack next course. The fix is in the next course

    
Fit many trees on perturbed copies of the data, then vote / average.
The bias barely changes; the variance collapses.

    
    
      
  • Random copies → bagging & random forests.
    • 
      
  • Sequential residual-fitting → gradient boosting.
    • 
      
  • Bayesian random perturbations → BART.
