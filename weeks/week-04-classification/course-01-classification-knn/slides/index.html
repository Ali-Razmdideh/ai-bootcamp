Part 1 · 0:00 – 0:30

What a classifier outputs

label, probability, decision boundary

Qualitative responses are everywhere

  • eye color ∈ {brown, blue, green}
  • email ∈ {spam, ham}
  • credit card ∈ {default, no default}
  • ER patient ∈ {stroke, overdose, seizure}

Given features X and a label Y in some set 𝒞, we want a function C(X) → 𝒞.

Often what we really want is the probability a sample belongs to each class — "this claim is 87% likely fraudulent" beats "fraud / not fraud" every time.

Why not just fit linear regression?

For a binary outcome, code Y = 0 / 1 and regress. Two failure modes:

Probabilities outside [0, 1]

Linear regression has no floor or ceiling. For low balances, the fitted "probability of default" goes negative.

No ordering of classes

Coding stroke=1, overdose=2, seizure=3 implies seizure is "twice as far" from stroke as overdose. Nonsense.

The fix: model log-odds (Course 2) or model the class-conditional density (Course 3).

Confusion matrix — the source of every metric

    

      

        
        predicted 0
        predicted 1
      

      
true 0
          TN
          FP

      
true 1
          FN
          TP

    

    

      
Accuracy = (TN+TP)/n  ·  Precision = TP/(TP+FP)

      
Recall = TP/(TP+FN)  ·  F1 = 2·P·R/(P+R)

    

  


  

    
Confusion matrix anatomy

    
    
Rows = ground truth; columns = predictions. Recall divides a row; precision divides a column.

  


  

    
Precision vs recall — the overlap view

    
    
Precision asks "of the alarms I raised, how many were real?"; recall asks "of the real positives, how many did I catch?"

  


  

    
Confusion matrix — the source of every metric

    
The simplest possible classifier: predict default if balance > 1700. Score it like any other model — confusion matrix plus precision / recall / F1.


    

      
      
      
    

    

  


  

    
Which metric for which job?

    
    
      
  • Accuracy — only on balanced classes. Lies on imbalanced data (see Part 3).
    • 
      
  • Precision — when false positives are costly. Spam filter: don't bury legit mail.
    • 
      
  • Recall — when false negatives are costly. Cancer screening: missing one is catastrophic.
    • 
      
  • F1 — when both matter equally.
    • 
      
  • ROC/AUC — threshold-free; ranks positives above negatives. The default report card.
    • 
    

  


  

    
The ROC curve

    

      

        
      

      

        
    
          
  • Sweep the decision threshold from 1 → 0.
    • 
          
  • At each threshold record (FPR, TPR) → trace the curve.
    • 
          
  • AUC = area under the curve = P(score(positive) > score(negative)).
    • 
          
  • Diagonal = random; top-left corner = perfect.
    • 
        

      

    

  


  

    
ROC = sliding the threshold

    

      

        
      

      

        
      

    

    
Each ROC point is one threshold: lowering t moves the bar left, catching more positives (TPR↑) but also more false alarms (FPR↑).

    Part 2 · 0:30 – 1:00
    
KNN

    
the no-training, Bayes-optimal-in-the-limit baseline

  


  

    
KNN in one slide

    

      

        
      

      

        
    
          
  1. Receive a new point x.
    2. 
          
  2. Find its k nearest training points.
    3. 
          
  3. Majority vote (classification) or mean (regression).
    4. 
        

        
No model fit. The training data is the model.

        
Bayes-optimal in the limit — as n → ∞ and k → ∞ slowly, KNN converges to the best possible classifier. The bar everything else must beat.

      

    

  


  

    
k controls the bias-variance dial

    

      

        
k = 1

        
Boundary wraps around every training point. Zero training error, jagged, high variance — memorizing noise.

      

      

        
k = 10

        
Smooth, recognizable boundary. Usually the sweet spot for noisy real data.

      

      

        
k = 100

        
Almost flat — predicts the global majority for nearly every query. High bias.

      

    

    
Pick k with cross-validation. Plot CV error vs k; choose the lowest, or the simplest within 1 SE.

  


  

    
kNN — the neighborhood vote

    

      

        
      

      

        

          
vote tally

          

            3 ○
             / 
            2 ✕
          

          
→ predict ○

        

        
Find the k closest training points; majority class wins. Ties broken by distance or by lowering k.

      

    

  


  

    
k = 1 vs k = 15 — two boundaries (ISLR 2.15)

    

      

        
      

      

        
      

    

    
Small k carves around every training point; large k averages large neighborhoods toward the global mix.

  


  

    
kNN bias-variance vs k (ISLR 2.17)

    
    
Training error rises monotonically with k; test error is U-shaped — the sweet spot k* sits between memorization (k=1) and over-smoothing.

Always scale before KNN

    

      

        
Two predictors on wildly different scales:

        
    
          
  • balance ∈ [0, 2 700]
    • 
          
  • income ∈ [0, 70 000]
    • 
        

        
Without scaling, Euclidean distance is dominated by income — balance contributes essentially zero. KNN ignores the variable that actually matters.

      

      

        
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.neighbors import KNeighborsClassifier

knn = Pipeline([
    ('scale', StandardScaler()),
    ('knn',   KNeighborsClassifier(n_neighbors=5)),
])
knn.fit(X_train, y_train)

        
Pipeline ensures the scaler is fit on training folds only — no leakage during CV.

      

    

  


  

    
Why scale before KNN

    
Income dwarfs balance numerically — without scaling, KNN's "nearest" is decided by income alone. Print the raw feature ranges to see why.


    

      
      
      
    

    

  


  

    
KNN with a scaling pipeline

    
Wrap StandardScaler and KNeighborsClassifier(k=5) in a Pipeline so the scaler is fit on training folds only — no leakage during CV. Then score on a held-out test set.

KNN's failure modes

    
    
      
  • Curse of dimensionality — in 50-D, "nearest" loses meaning. All points are roughly equidistant.
    • 
      
  • Memory + latency — prediction is O(n·p) per query; bad on 10⁷ rows.
    • 
      
  • No coefficients — you cannot say "balance matters most"; nothing to inspect.
    • 
    

    
Despite this, KNN is the right first model: zero hyperparameters except k, often surprisingly strong on tabular data with p < 20.

  







  

    Part 3 · 1:00 – 1:30
    
Class imbalance

    
why accuracy lies, and what to read instead

  


  

    
"Always say No" scores 97%

    

      

        
Default rate ≈ 3.3%. The constant classifier predict 0 scores:

        

          
accuracy96.7%

          
recall on defaulters0%

          
precisionundefined (0/0)

          
AUC0.50

        

      

      

        
Reporting 96.7% accuracy on this task is a lie. The model finds none of the people you actually care about.

      

    

  


  

    
Class imbalance — the accuracy mirage

    
    
When one class swamps the other, accuracy rewards the trivial "all-negative" model. Switch to precision, recall, F1, or AUC.

  


  

    
The majority-class trap

    
Score the trivial "always predict No" classifier on the held-out test set. Accuracy looks great — but recall on defaulters is zero. Useless model.


    

      
      
      
    

    

  


  

    
Two fixes that always help

    

      

        
stratify=y

        
train_test_split(X, y,
    stratify=y, test_size=0.25)

        
Keeps class proportions identical in train and test. Free, no downside.

      

      

        
class_weight='balanced'

        
LogisticRegression(
    class_weight='balanced')

        
Weights the loss inversely to class frequency. Trades precision for recall — usually the right trade when positives are rare and costly.

      

    

  


  

    
Fix: stratify + class_weight

    
Re-fit logistic regression with class_weight='balanced' on a stratified split. Accuracy drops but recall on the positive class jumps — almost always the right trade-off for fraud, default, disease, etc.


    

      
      
      
    

    

  


  

    
Move the threshold, not the model

    
Most classifiers expose predict_proba. Don't accept the default 0.5 — pick a threshold that matches the cost of FN vs FP.

    
proba = model.predict_proba(X_test)[:, 1]

import numpy as np
from sklearn.metrics import f1_score
thresholds = np.linspace(0.05, 0.6, 50)
f1s = [f1_score(y_test, (proba >= t)) for t in thresholds]
best_t = thresholds[int(np.argmax(f1s))]

    
On Default, the F1-optimal threshold is around 0.15, not 0.5.
