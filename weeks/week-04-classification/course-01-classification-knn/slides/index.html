Part 1 · 0:00 – 0:30

What a classifier outputs

label, probability, boundary

From regression to classification

  • Regression: ŷ is a number; minimize squared error.
  • Classification: ŷ is a label; minimize misclassification (or maximize log-likelihood).
  • Same X, different y. Same toolbox: pipelines, CV, regularization.

The confusion matrix

For binary classification every prediction falls into one of four cells:

Predicted 0Predicted 1
Actual 0TN
True Negative		FP
Type I error
Actual 1FN
Type II error		TP
True Positive

Every metric — accuracy, precision, recall, F1, MCC — is a function of these four numbers.

Reading a confusion matrix

Threshold classifier on Default: "predict default if balance > 1700"

Pred NoPred Yes
True No9 62542
True Yes25281
✓ 9 625 TN — correctly called safe
✗ 42 FP — flagged safe customers as risky
✗ 252 FN — missed defaulters ← costly!
✓ 81 TP — caught defaulters

Accuracy = (9625+81)/10000 = 97% — looks great. But recall = 81/333 = 24% — we miss 76% of defaulters.

Accuracy

Accuracy = (TP + TN) / (TP + TN + FP + FN)

Intuition: fraction of all predictions that are correct.

✓ Use when classes are roughly balanced and all errors cost the same.
✗ Misleading on imbalanced data: a model that always predicts the majority class scores 97% on Default while catching zero defaulters.

Example: classifying handwritten digits 0–9 (MNIST) with ~equal class sizes — accuracy is fine.

Precision

Precision = TP / (TP + FP)

Intuition: of every positive prediction, what fraction is actually positive?
"When I ring the alarm, how often am I right?"

✓ Use when false positives are costly.
  • Spam filter: a legitimate email landing in spam is worse than spam reaching the inbox.
  • Recommendation engine: irrelevant results erode user trust.
  • Drug approval: approving an ineffective drug harms patients.

Precision can be gamed: predict only the most certain positives and ignore borderline cases.

Recall (Sensitivity / TPR)

Recall = TP / (TP + FN)

Intuition: of every actual positive, what fraction did we catch?
"How many real alarms did I not miss?"

✓ Use when false negatives are costly.
  • Cancer screening: missing a tumour is worse than a false alarm.
  • Fraud detection: every fraudulent transaction you miss costs money.
  • Safety systems: failing to detect a fault can be catastrophic.

Recall can be gamed: predict everything as positive — then TP = all positives and FN = 0, giving recall = 1.

The precision–recall tradeoff

Moving the decision threshold changes the balance:

Threshold ↓Predictions flaggedPrecisionRecall
0.9Very few (only very confident)High ↑Low ↓
0.5ModerateMediumMedium
0.1Almost everythingLow ↓High ↑

You can't maximise both simultaneously. The right threshold depends on the relative cost of FP vs FN in your problem.

F1 Score

F1 = 2 · Precision · Recall / (Precision + Recall)

Intuition: harmonic mean of precision and recall. The harmonic mean penalises extreme imbalance — a model that achieves P=1 and R=0.01 gets F1 = 0.02, not 0.5.

✓ Use when both FP and FN have similar costs.

Example — spam filter
P = 0.90, R = 0.80
F1 = 2·0.90·0.80/(0.90+0.80) = 0.847

Pathological case
P = 1.0, R = 0.01
F1 = 2·1·0.01/(1+0.01) ≈ 0.020
Correctly punished.

Fβ Score — tuning the balance

Fβ = (1 + β²) · Precision · Recall / (β² · Precision + Recall)

β is the answer to: "How many times more important is recall than precision?"

βWeight on recallUse case
0.5Less (precision matters more)Spam filter — avoid good mail in spam
1.0Equal (= F1)General purpose
2.0More (recall matters more)Cancer screening — missing a case is worse

F0.5: Precision counts 4× more than Recall in the denominator.
F2: Recall counts 4× more than Precision in the denominator.

Multi-class averaging

With K > 2 classes, how do you aggregate per-class scores into one number?

StrategyFormulaWhen to use
MacroSimple mean over K classesAll classes equally important (rare disease)
WeightedMean weighted by class size nₖ/nClass frequencies reflect real-world distribution
MicroPool all TP/FP/FN, then compute onceLarge classes dominate; ≈ accuracy for single-label
SamplesPer-sample averageMulti-label classification only

Macro vs Weighted: if your rare class (1% of data) matters as much as the common class (99%), use macro. If class frequency reflects importance, use weighted.

Multi-class averaging — example

Three-class problem: Cat / Dog / Bird. Bird is rare (5 samples).

ClassPrecisionRecallF1Support
Cat0.900.880.89100
Dog0.850.870.8695
Bird0.400.600.485

Macro F1 = (0.89 + 0.86 + 0.48) / 3 = 0.74
Bird pulls it down — this is what we want if Bird matters.

Weighted F1 = (0.89·100 + 0.86·95 + 0.48·5) / 200 ≈ 0.88
Bird barely affects it — hides poor rare-class performance.

ROC curve and AUC

ROC (Receiver Operating Characteristic) plots TPR vs FPR as the decision threshold sweeps from 1 → 0.

AUC = P(score(positive) > score(negative))

The probability that a randomly chosen positive gets a higher predicted score than a randomly chosen negative.

AUCMeaning
0.50Random guessing — no signal
0.70Acceptable
0.85Good
1.00Perfect separation
✓ Threshold-free. Works well on moderately imbalanced data. Default go-to when you don't know the operating threshold yet.

Precision–Recall curve and AUC-PR

Plots Precision vs Recall as threshold sweeps. The area under it is AP (average precision).

AUC-ROC
FPR denominator = TN + FP (all negatives).
With many negatives (imbalanced data) FPR stays small even with many FP — ROC looks optimistic.

AUC-PR
Precision denominator = TP + FP (only predicted positives).
Directly measures quality of positive predictions — doesn't get inflated by abundant negatives.

✓ Use AUC-PR when the positive class is very rare (< 5%), e.g. fraud detection, medical screening.

Which metric for which job?

MetricBest whenWatch out for
AccuracyBalanced classes, equal error costsImbalanced data — majority-class trap
PrecisionFP cost high (spam, drug approval)Can be gamed by predicting few positives
RecallFN cost high (cancer, fraud)Can be gamed by predicting everything positive
F1FP ≈ FN costDoesn't account for TN

Part 2 · 0:30 – 1:00

KNN

the no-training baseline

KNN in one slide

  • To predict a new point, find its k nearest training points.
  • Take a majority vote (classification) or average (regression).
  • No model fit — the training data is the model.

k controls the bias-variance dial

  • k = 1 — perfectly fits training, jagged boundary, high variance.
  • k = 25 — smooth boundary, more bias, less variance.
  • Pick k with cross-validation.

Always scale before KNN

Income (0 to 70 000) versus balance (0 to 2700) — without scaling, KNN distance is dominated by income alone. Wrap in a Pipeline:

Pipeline([
    ('scale', StandardScaler()),
    ('knn', KNeighborsClassifier(k=5)),
])
Part 3 · 1:00 – 1:30

Class imbalance

the trap and the fixes

'Always say No' scores 97%

Default rate ≈ 3.3%. Constant classifier "predict No" achieves:

MetricValueVerdict
Accuracy96.7%Lies! We catch nothing.
Recall0%Useless for the real goal.
Precisionundefined (0/0)No positive predictions.
F10Correctly punishes this model.
AUC0.5Random guessing — exposed.

Always pair accuracy with recall and AUC on imbalanced problems.

Why imbalance hurts training

  • The model sees far more negatives → biased toward predicting negative.
  • Standard loss treats every sample equally → minority class contributes little to the gradient.
  • Cross-validation folds may have no positives if splits aren't stratified.
  • Decision threshold at 0.5 is often wrong — the right threshold may be 0.1 or lower.

Fix 1 — Stratified splits + class_weight

Stratified splits

train_test_split(X, y,
    stratify=y,   # ← key
    test_size=0.3,
    random_state=0)

Ensures both halves have the same class ratio as the full dataset.

class_weight

LogisticRegression(
    class_weight='balanced'
)
# weight = n / (K * n_k)
# minority gets a higher weight

Upweights the minority loss without changing the data.

Fix 2 — Threshold tuning

The default threshold of 0.5 assumes equal misclassification costs. Lower it to increase recall:

proba = model.predict_proba(X_test)[:, 1]
y_hat = (proba > 0.1).astype(int)   # threshold = 0.1
print(classification_report(y_test, y_hat))

Use the precision-recall curve or the ROC curve to find the threshold that meets your business constraint (e.g. FPR < 5%).

Fix 3 — Resampling

Oversampling — SMOTE

from imblearn.over_sampling import SMOTE
X_res, y_res = SMOTE(random_state=0).fit_resample(X_train, y_train)

Synthesises new minority samples by interpolating between existing ones. Use on very rare classes (<5%).

Undersampling

from imblearn.under_sampling import RandomUnderSampler
X_res, y_res = RandomUnderSampler(random_state=0).fit_resample(X_train, y_train)

Randomly drops majority samples. Fast but wastes data. Useful when n is very large.

Always resample inside the cross-validation loop — resampling before splitting leaks information.

Metrics for imbalanced data

MetricWhy it helpsTypical positive rate
Recall + Precision separatelyFocus on minority class directlyAny
F1 / FβHarmonic mean — zero positives → F1 = 0Any
AUC-ROCThreshold-free, unaffected by class ratio> 5%
AUC-PR (Average Precision)Not inflated by abundant negatives< 5%

Never use accuracy alone on imbalanced data. Report at least recall + AUC.

Recap

Three things to leave with

  1. A classifier outputs a label (often via a probability). Read the confusion matrix every time.
  2. KNN: no training, no fancy math, but it does need scaling.
  3. Accuracy lies on imbalanced data. Reach for recall/F1/AUC and class_weight.
