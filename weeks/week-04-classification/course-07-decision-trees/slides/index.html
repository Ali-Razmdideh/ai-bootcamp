Part 0 · 0:00 – 0:15

How Splits Are Chosen

recursive binary splitting · threshold scan · greedy search

The search problem

At each node we have N observations and P features. We want to find the single (feature j, threshold s) pair that produces the most homogeneous children.

RL(j, s) = { x | xj < s }    (left child)
RR(j, s) = { x | xj ≥ s }    (right child)
  • We try every feature and every possible threshold value.
  • We pick the (j, s) that scores best according to a split criterion.
  • Then we repeat independently on each child — recursively.

The split criterion differs for regression vs. classification — but the search loop is identical.

Regression splits: minimise RSS

For a candidate split (j, s), compute the mean in each child:

ȳL = mean{ yi : xij < s }    ȳR = mean{ yi : xij ≥ s }

Cost(j, s) = RSSL + RSSR = Σi∈RL(yi − ȳL)² + Σi∈RR(yi − ȳR)²

Worked example — 6 points, one feature, three candidate thresholds:

sLeft nRSSLRight nRSSRTotal
2.010.0054.804.80
4.030.6730.671.33 ✓
6.054.8010.004.80

Choose s = 4.0 — it produces the most equal, tight groups. Pick the (j, s) with the smallest total RSS across all features.

Scanning thresholds — regression

Repeat for every feature j. The global minimum across all (j, s) pairs gives the decision boundary.

Classification splits: maximise impurity gain

ΔG(j, s) = G(parent) − nL/n · G(RL) − nR/n · G(RR)

where G = Gini index (or entropy — same structure)

Worked example — 10 binary-labelled points, two candidate thresholds (Gini):

sLeft (n, +, −)GLRight (n, +, −)GRΔG
s = 34 · 3+, 1−0.3756 · 2+, 4−0.4440.083
s = 77 · 6+, 1−0.2453 · 0+, 3−0.0000.253 ✓

s = 7 wins — the right child is perfectly pure (all negatives). Pick the (j, s) with the largest ΔG across all features.

Scanning thresholds — classification

Feature 2 has a higher Gini gain peak — it wins the split at s*.

The full search loop

for each feature j in 1..P:
    sort observations by x_j          # O(n log n)
    for each midpoint s between adjacent values:
        compute Cost(j, s)            # RSS  or  weighted impurity
        if Cost(j, s) < best_cost:
            best_j, best_s = j, s

split node on (best_j, best_s)
recurse on left and right children
  • Complexity: O(n p log n) per node level.
  • Greedy: picks the best local split at each step — no lookahead.
  • Result: axis-aligned rectangular regions in feature space.

Greedy search is fast but not globally optimal — each split ignores future consequences. This is why deep trees overfit and need pruning.

Part 1 · 0:15 – 0:40

Regression Trees

recursive binary splitting · leaf means · RSS

The algorithm

  1. Scan every feature j and every cutpoint s.
  2. Pick the (j, s) that minimises total within-region RSS.
  3. Recurse on the two children independently.
  4. Stop when a node has fewer than min_samples_split rows (default 2).

Prediction = mean of training responses in the leaf.

Split criterion: RSS

At each node, minimise:

RSSL(s) + RSSR(s)

RSSL = Σi: xj < s (yi − ȳL)²   |   RSSR = Σi: xj ≥ s (yi − ȳR)²

  • Search is greedy — pick the best local split, no lookahead.
  • Scan all features × all thresholds at every node: O(n p log n) per level.
  • Result is a partition into rectangular regions in feature space.

Building the baseball tree — step by step

Each step picks the single split that maximises the RSS reduction across the entire remaining region.

Classification Trees

    
Gini · cross-entropy · classification error

  


  
  

    
Notation: node class proportions

    
At node m, let pmk = proportion of training observations in region m that belong to class k.

    

      Example — binary classification (k = Yes / No):
    

    

      Node A: 99 Yes, 1 No → pm,Yes = 0.99, pm,No = 0.01  (very pure)

      Node B: 51 Yes, 49 No → pm,Yes = 0.51, pm,No = 0.49  (almost random)
    

    

      We predict the majority class in each leaf. We need a measure that tells us how good a node is — its impurity.
    

  


  
  

    
Classification error rate — definition

    

      E = 1 − max(p, 1−p)
    

    

      "fraction of observations that are not the majority class in this node"
    

    

      
p(class 1)Classification Error

      
        
0.510.49

        
0.600.40

        
0.900.10

      
    

    

      Moving from p = 0.51 → 0.60 makes the node much purer — but classification error only changes 0.49 → 0.40. A small, nearly uniform step.
    

  


  
  

    
The kink problem

    
Classification error is piecewise linear with a kink at p = 0.5.

    
    
      
  • Many different splits receive nearly identical scores near p = 0.5 — the criterion cannot distinguish between them.
    • 
      
  • It only asks: "What fraction would I misclassify if I predicted the majority class?"
    • 
      
  • It ignores how confident the node is. A node with 51% / 49% and one with 90% / 10% are both just "predict class 1" — even though the second is far purer.
    • 
    

    

      Classification error is therefore used for pruning / evaluating a finished tree — not for growing it.
    

  


  
  

    
Why Gini and entropy are preferred

    
Both penalise uncertainty more strongly near p = 0.5, making them more sensitive to purity improvements.

    

      
pClassification ErrorGini = 2p(1−p)

      
        
0.50.500.50

        
0.60.400.48

        
0.90.100.18

      
    

    

      Gini and entropy show a larger relative change when purity improves — they are better at discriminating between candidate splits.
    

    

      Use Gini or entropy for growing; use classification error for pruning.
    

  


  

    
Three impurity measures

    

      
MeasureFormula (binary)Max at p=0.5

      
        
Classification error1 − max(p, 1−p)0.500

        
Gini index2p(1−p)0.500

        
Cross-entropy−p log p − (1−p)log(1−p)0.693

      
    

    
    
      
  • All three are 0 for pure nodes, maximum at 50/50.
    • 
      
  • Gini and entropy are differentiable → better gradients for growing.
    • 
      
  • Error rate is used for pruning (we care about the final misclassification rate, not impurity).
    • 
    

  


  
  

    
Gini vs entropy vs error rate — impurity comparison

    
Plot all three impurity measures as a function of p (fraction of class 1). See that Gini and entropy are smooth and peak at p=0.5 while error rate has a kink.


    

      
      
      
    

    


  

  

    
Gini index

    

      G = 1 − Σk p̂mk²
    

    
    
      
  • Interpretation: probability that a randomly chosen element is mis-classified if labelled by the class distribution.
    • 
      
  • Binary: G = 2p(1−p) — symmetric, peaks at p = 0.5.
    • 
      
  • sklearn default for DecisionTreeClassifier.
    • 
    

  


  

    
Cross-entropy (deviance)

    

      D = −Σk p̂mk log p̂mk
    

    
    
      
  • Information-theoretic measure: bits needed to encode the label distribution.
    • 
      
  • Also called deviance in the tree literature.
    • 
      
  • Numerically very similar to Gini — in practice they select almost identical splits.
    • 
      
  • Use criterion='entropy' or criterion='log_loss' in sklearn.
    •

    Part 3 · 1:05 – 1:25
    
Cost-Complexity Pruning

    
grow full · penalise size · CV to pick α

  


  

    
The problem: full trees overfit

    
    
      
  • A fully grown tree has one leaf per (nearly) unique training observation.
    • 
      
  • Train error → 0, but test error is terrible.
    • 
      
  • Naïve fix: set max_depth by hand — but how do you choose?
    • 
    

    

      Better: grow full, then prune by penalising tree complexity.
    

  


  

    
Penalised cost criterion

    

      Cα(T) = Σleaves MSE(leaf) + α · |T|
    

    
    
      
  • α = 0: no penalty → full tree T₀.
    • 
      
  • α → ∞: every split is too costly → root-only stump.
    • 
      
  • Weakest-link pruning finds the nested sequence T₀ ⊃ T₁ ⊃ … each minimising Cα for a range of α.
    • 
    

    

      sklearn: cost_complexity_pruning_path() returns all candidate α values.
    

  


  
  

    
Cost-complexity pruning — choose α with cross-validation

    
Grow a full tree then use sklearn's cost_complexity_pruning_path to find candidate α values. Use 5-fold CV to select the best one.


    

      
      
      
    

    


  

  

    
Selecting α with cross-validation

    
    
    
      
  1. Compute the pruning path: all candidate α values.
    2. 
      
  2. For each α, evaluate K-fold CV error.
    3. 
      
  3. Pick the α that minimises CV error (or 1-SE rule for parsimony).
    4. 
      
  4. Refit the full training set with the selected α.
    5. 
    

  


  

    
Effect of α on tree structure

    

      

        
α = 0

        
Full tree · 100% train acc · overfits

      

      

        
α = CV-optimal

        
Pruned tree · balances bias/variance

      

      

        
α → ∞

        
Root stump · predicts majority class everywhere

      

    

    

      The pruned tree is usually much more interpretable than the full tree — and generalises better.

    Part 5 · 1:35 – 1:45
    
Pros, Cons & What's Next

    
the variance problem → ensembles

  


  

    
Strengths of trees

    
    
      
  • Interpretable: every prediction is a set of if–then rules.
    • 
      
  • No feature scaling: splits depend only on ordering.
    • 
      
  • Mixed types: numeric and categorical in one model.
    • 
      
  • Interactions: nested splits capture X₁·X₂ interactions automatically.
    • 
      
  • Missing values: surrogate splits handle them gracefully.
    •

The variance problem

    
Fit the same depth-3 tree on four different bootstrap samples of the same training data:

    
    
      
  • Root split changes from ShelveLoc_Good to Price to …
    • 
      
  • Tree structure changes completely.
    • 
      
  • Each individual tree is unbiased but has huge variance.
    • 
    

    

      High variance is the single biggest weakness. Ensembles fix it.
    

  


  

    
The bridge to Course 8

    

      

        
Bagging

        
Average B independent trees on bootstrap samples → lower variance

      

      

        
Random Forests

        
Bagging + random feature subsets → decorrelated trees → even lower variance

      

      

        
Boosting

        
Sequential trees each fitting the residuals → reduce bias

      

    

    
Single tree: interpretable but variable. Ensemble: accurate but a black box. Course 8 teaches you to navigate that trade-off.

  


  

    
Recap

    
    
      
  • Trees split recursively to minimise RSS (regression) or Gini/entropy (classification).
    • 
      
  • Always grow full, then prune with ccp_alpha chosen by K-fold CV.
    • 
      
  • Use Gini/entropy for growing; classification error is fine for pruning.
    • 
      
  • Trees beat linear models on non-linear, axis-aligned boundaries; lose on smooth linear ones.
    • 
      
  • Variance is the Achilles' heel → ensembles to the rescue in Course 8.
