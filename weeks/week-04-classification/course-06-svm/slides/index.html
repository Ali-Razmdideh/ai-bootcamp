Part 1 · 0:00 – 0:30

Maximum margin

geometry, support vectors, the soft margin

A hyperplane separates feature space

In p dimensions a hyperplane is the (p−1)-dim flat surface

β₀ + β₁x₁ + ⋯ + βpxp = 0

Code labels y ∈ {−1, +1}. If yi·(β₀ + β·xi) > 0 for every i, that hyperplane separates the classes.

Infinitely many lines separate — pick the best

The maximum-margin hyperplane is the unique one furthest from both classes.

Points touching the margin are the support vectors (circled). Move any other point — the boundary doesn't budge.

Only support vectors define the model. That's the source of SVM's robustness and its name.

The optimization

maximize M s.t. yi(β₀ + β·xi) ≥ M,  ‖β‖ = 1

Convex quadratic program — one global optimum, solved in seconds for n in the thousands.

Equivalent dual form: minimize ½‖w‖² subject to yi(w·xi + b) ≥ 1.

Real data overlaps — soft margin

Hard margin breaks the moment one stray red sneaks into the blue cloud. The support vector classifier allows slack — let some points violate the margin (or even cross it), pay a penalty.

maximize M s.t. yi(β₀ + β·xi) ≥ M(1 − εi),   ε ≥ 0,   Σ εi ≤ C

The C parameter — tuning the slack

Small C

Wide margin, many support vectors. Tolerates violations. High bias, low variance.

Large C

Narrow margin, few support vectors. Punishes violations. Low bias, high variance — overfits.

Tune with CV

GridSearchCV(svc, {'C': np.logspace(-2, 3, 6)})

Heads up: sklearn's C is the same direction as the formula above — large C = strict. Opposite of LR's C.

Part 2 · 0:30 – 1:00

Kernels

non-linear boundaries, cheaply

Linear boundaries fail on curved truth

The fix from Course 3 was to add features: x₁², x₂², x₁x₂, … Works, but the feature space blows up fast.

Kernels do this implicitly — non-linear boundary, original feature space, no explicit expansion.

Inner products run the show

The SVM solution can be written entirely in terms of inner products:

f(x) = β₀ + Σi∈𝒮 α̂i·⟨x, xi⟩

  • 𝒮 is the set of support vectors — usually a small subset.
  • We never need xi on its own, only ⟨x, xi⟩.
  • Replace ⟨·,·⟩ with a more powerful similarity → kernel trick.

Three kernels in the wild

Linear

K(x, x') = x · x'

Same boundary shape as logistic regression. Best when n < p.

Polynomial (d)

(1 + x · x')d

Bends, but rigidly. Equivalent to fitting in a degree-d polynomial space without computing it.

RBF (γ)

exp(−γ‖x − x'‖²)

Local bumps. Implicit feature space is infinite-dimensional. The default.

The γ knob in the RBF kernel

  • Small γ — wide kernel; smooth boundary; more bias.
  • Large γ — narrow kernel; boundary curls around individual points; more variance.
  • Grid-search (C, γ) with CV. gamma='scale' is a sensible default starting point.

SVM in sklearn

from sklearn.svm import SVC
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import GridSearchCV

pipe = Pipeline([('sc', StandardScaler()),
                 ('svc', SVC(kernel='rbf'))])

grid = GridSearchCV(pipe, {
    'svc__C':     [0.1, 1, 10, 100],
    'svc__gamma': [1e-3, 1e-2, 1e-1, 1],
}, cv=5).fit(X_train, y_train)

Always scale before an SVM — kernels are based on distances. Wrap in a Pipeline so scaling is fit on training folds only.

SVM is hinge-loss + L2 regularization

minβ   Σi max(0, 1 − yi·f(xi))   +   λ‖β‖²

Both are "loss + L2"; the loss differs. Hinge loss is zero past the margin — that's why only support vectors matter.

Part 3 · 1:00 – 1:30

p ≫ n & the comparison map

the Khan gene-expression lab

Khan — 63 patients, 2 308 genes, 4 cancer subtypes

  • p (genes) ≫ n (patients): the classic genomics setup.
  • Unregularized LR diverges — perfect separation in 2 308-D.
  • Linear SVM thrives — the margin objective is itself a form of regularization.
  • RBF is overkill here: 2 308 raw dimensions already give plenty of flexibility.
from sklearn.svm import SVC
svc = SVC(kernel='linear', C=10).fit(X_train, y_train)
print(svc.score(X_test, y_test))   # often perfect

Rule of thumb: p ≫ n → linear kernel. Switch to RBF only when n is large and features are dense.

SVM vs the field — when to use what

situation first model
classes well-separated, n < pSVM (linear) or LDA
need calibrated probabilitiesLogistic regression
curved boundary, n moderateSVM (RBF) or gradient boosting
tabular, mixed dtypesRandom forest / gradient boosting
huge n (millions of rows)Linear models / GBM — not RBF-SVM
images, audio, textDeep nets (next week onward)

More than two classes

One-vs-rest (OVR)

Fit K binary SVMs (class k vs rest); predict the class with the highest score.

One-vs-one (OVO)

Fit K(K−1)/2 pairwise SVMs; predict by majority vote among them.

sklearn defaults to OVO for kernel SVMs. For K not too large, OVO is usually preferred — each sub-problem is balanced.

Caveats and gotchas

  • No calibrated probabilities. SVC(probability=True) wraps Platt scaling — slow and approximate.
  • Doesn't scale to n > 10⁵ with RBF. Training is O(n²) – O(n³). Switch to linear SVM (LinearSVC) or GBM.
  • Two hyperparameters to tune. Always do C × γ grid with CV.
  • Scale your features. Repeating because forgetting is the #1 SVM bug.
Recap

Three things to leave with

  1. SVM finds the maximum-margin boundary. Only support vectors define it; soft margin C controls slack.
  2. The kernel trick gives non-linear boundaries cheaply. RBF is the default; tune C and γ.
  3. SVM is the tool for p ≫ n. Use a linear kernel, scale features, grid-search C.
