Part 1 · 0:00 – 0:25

Bagging

bootstrap aggregating · variance reduction

The variance problem (recap)

  • A single deep tree has low bias but high variance.
  • Refit on a bootstrap sample → completely different tree (we saw this in Course 7).
  • Idea: if we could fit the model on many training sets and average, variance drops to zero while bias stays the same.
  • Bagging: simulate "many training sets" via bootstrap sampling.

Bagging algorithm

  1. Draw B bootstrap samples (sample n rows with replacement).
  2. Fit a deep, unpruned tree to each sample.
  3. Average predictions (regression) or majority vote (classification).

B = 100–500 is usually enough. Error stabilises quickly; more trees never hurt (just costs time).

Why averaging reduces variance

Even if ρ = 0.8, averaging 300 trees reduces variance by a factor of ~5. Random forests will reduce ρ further.

Bagging reduces variance — see it directly

Show that averaging multiple noisy trees produces a stable, lower-variance predictor. Compare single tree vs bagging as B increases.


  

  

    
OOB error — a free cross-validation

    
    
    
      
  • Each bootstrap sample uses ≈63% of training rows.
    • 
      
  • The remaining ≈37% are out-of-bag (OOB) — never seen by that tree.
    • 
      
  • For each observation i, average predictions from all trees that did NOT see i.
    • 
      
  • OOB MSE closely tracks test MSE — no separate validation set needed.
    • 
    

  


  
  

    
OOB error — free cross-validation

    
Use sklearn's BaggingClassifier with oob_score=True. Compare OOB error to held-out test error — they should be very close.


    

      
      
      
    

    


  

  

    
Bagging results on Boston

    
    
      
  • Single tree test MSE ≈ 25
    • 
      
  • Bagging 300 trees ≈ 12–14 (≈ 2× improvement)
    • 
    

    

      But bagging doesn't decorrelate the trees — if one feature dominates, every tree uses it. Random forests fix this.

Random Forests

    
decorrelation via random feature subsets

  


  

    
The decorrelation trick

    
    

      Bagging with m=p is just bagging. RF with m < p decorrelates the trees and shrinks the variance floor.
    

  


  

    
Choosing m

    

      

        
Classification

        
m = ⌊√p⌋
sklearn default: max_features='sqrt'

      

      

        
Regression

        
m = ⌊p/3⌋
sklearn default: max_features=1.0 (= p) — try 'sqrt' or p//3

      

      

        
Tuning

        
Vary m from 1 to p, plot test error.
Usually U-shaped:
too small = high bias,
too large = correlated trees.

      

    

  


  
  

    
Random Forest — effect of max_features on accuracy

    
Vary max_features from 1 to all p features. See the U-shaped accuracy curve: too few features → high bias; too many → correlated trees → high variance.


    

      
      
      
    

    


  

  

    
Variable importance

    
Two ways to measure importance of feature j:

    
    
      
  1. Gini importance: total impurity decrease attributable to j, averaged across all trees. Fast, built-in. Biased toward high-cardinality features.
    2. 
      
  2. Permutation importance: randomly shuffle column j in the test set; measure how much performance drops. Slower but more reliable.
    3. 
    

    

      Use permutation importance for final reporting; gini importance for quick screening.
    

  


  
  

    
Random Forest feature importance

    
Fit a Random Forest and compare feature importances. With 200 trees, the importance estimates are more stable and reliable than from a single tree.


    

      
      
      
    

    


  

  

    
Gene expression: Khan dataset

    
    
      
  • 4 tumour types · 2,308 genes · 63 training observations
    • 
      
  • p >> n — random forests handle this naturally by subsampling features
    • 
      
  • RF test accuracy ≈ 95%+ despite the extreme dimensionality
    • 
      
  • Variable importance reveals the handful of discriminative genes
    • 
    

    

      Random forests are one of the best out-of-the-box methods for high-dimensional gene expression problems.

Boosting

    
sequential residual fitting · shrinkage

  


  

    
Bagging vs Boosting

    

      
Bagging / RFBoosting

      
        
Tree trainingIndependent (parallel)Sequential

        
What each tree fitsFull yCurrent residuals

        
GoalReduce varianceReduce bias (+ variance)

        
OverfittingRare (more trees never hurt much)Possible if B too large or λ too large

      
    

  


  

    
The boosting algorithm

    
    
    
      
  1. Start: f̂(x) = 0, residuals rᵢ = yᵢ.
    2. 
      
  2. For b=1..B: fit tree T̂ᵇ to (xᵢ, rᵢ); update f̂ += λT̂ᵇ; update rᵢ -= λT̂ᵇ(xᵢ).
    3. 
      
  3. Final prediction: f̂(x) = Σ λT̂ᵇ(x).
    4. 
    

  


  

    
Three tuning parameters

    

      

        
B — # trees

        n_estimators
        
More trees → lower bias. But overfitting possible with large λ.

      

      

        
λ — shrinkage

        learning_rate
        
Smaller λ: slower but usually better. Typical: 0.01–0.1.

      

      

        
d — depth

        max_depth
        
d=1: stump (no interactions). d=2: one interaction. Often d=1..4 works.

      

    

    

      Small λ + large B almost always beats large λ + small B. Shrinkage prevents over-correction.
    

  


  
  

    
Gradient Boosting — three tuning knobs

    
Fit GradientBoostingClassifier while varying learning_rate, n_estimators, and max_depth. See how each affects train vs test accuracy.


    

      
      
      
    

    


  

  

    
Shrinkage: the analogy

    
    

      The practical rule: set λ = 0.01–0.05 and let B be determined by early stopping on a validation set.
    

  


  
  

    
Ensemble comparison: single tree vs bagging vs RF vs GBM

    
Run all four methods on the same dataset. Compare test accuracy and training time. See the progression from high-variance trees to tuned boosting.


    

      
      
      
    

    


  

  

    
Results on Hitters salary

    
    
      
  • Single pruned tree test MSE ≈ 0.50
    • 
      
  • Random forest (300, m=√p) ≈ 0.32
    •

      Boosting with careful tuning consistently outperforms bagging/RF on structured tabular data.
    

  







  

    Part 4 · 1:15 – 1:25
    
BART

    
Bayesian Additive Regression Trees

  


  

    
The BART model

    

      Y = T₁(X) + T₂(X) + ⋯ + TK(X) + ε
    

    
    
      
  • Sum of K small trees (typically K=200, depth 1–2).
    • 
      
  • Priors on tree structure and leaf values → regularisation.
    • 
      
  • Inference via MCMC → full posterior distribution over predictions.
    • 
      
  • This gives you uncertainty estimates — not just a point estimate.
    • 
    

  


  

    
The partial residual update

    
    

      At each iteration, update tree j by fitting it to y minus all other trees' predictions.
    

  


  

    
Burn-in and posterior mean

    
    
      
  • MCMC takes many iterations to converge: the first L iterations are burn-in and discarded.
    • 
      
  • After burn-in: collect M posterior draws of the tree ensemble.
    • 
      
  • Final prediction = mean over those M draws.
    • 
      
  • Uncertainty = standard deviation or credible interval over the M draws.
    • 
    

    

      BART is unique in providing calibrated uncertainty estimates while using an ensemble of trees.
    

  


  

    
BART vs Boosting

    

      
BoostingBART

      
        
TrainingGreedy forwardMCMC (joint posterior)

        
UncertaintyNoneFull posterior distribution

        
RegularisationTuning B, λ, dPriors on tree structure/leaves

        
SpeedFast (seconds)Slow (minutes–hours)

        
When to useProduction, predictionInference, uncertainty quantification

Summary

    
when to use which

  


  

    
Decision matrix

    

      
MethodInterpretableAccuracySpeedBest for

      
        
Single tree✅ HighLowVery fastExplanation, prototype

        
Bagging❌MediumFastBaseline ensemble

        
Random Forest❌HighFastRobust general tabular

        
Gradient Boosting❌Very highSlowCompetition, production

        
HistGBM / XGBoost❌Very highFastLarge data, production

        
BART❌HighVery slowUncertainty quantification

      
    

  


  

    
Practical starting point

    
    
      
  1. Quick baseline: RandomForestClassifier(n_estimators=300) — no tuning needed.
    2. 
      
  2. Best accuracy: HistGradientBoostingClassifier or XGBoost — tune learning_rate and n_estimators.
    3. 
      
  3. Explanation: fit a single depth-3 tree on the same problem and present to stakeholders.
    4. 
      
  4. Uncertainty: BART, or calibrate a GBM with CalibratedClassifierCV.
    5. 
    

  


  

    
Recap

    
    
      
  • Bagging = average B deep trees on bootstrap samples → lower variance.
    • 
      
  • Random forests = bagging + m random features per split → decorrelated, lower variance.
    • 
      
  • Boosting = sequential trees fitting residuals + shrinkage λ → lower bias & variance.
    • 
      
  • BART = Bayesian sum of trees via MCMC → uncertainty estimates.
    • 
      
  • For production tabular tasks: start with HistGBM or XGBoost. Use a single tree to explain.
