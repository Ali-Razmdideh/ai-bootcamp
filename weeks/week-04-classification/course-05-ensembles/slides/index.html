Part 1 · 0:00 – 0:30

Bagging + Random Forests

average the noise away

Why averaging helps — one slide of math

If Z₁, …, Zn are independent with variance σ², the mean Z̄ has variance σ² / n.

Averaging reduces variance — provided the things you average are independent.

But we only have one training set. The bootstrap is how we fake "independent" copies.

Bagging = Bootstrap aggregating

  1. Draw B bootstrap samples (with replacement) from the training set.
  2. Fit a deep, unpruned tree to each.
  3. Aggregate: average (regression) or majority vote (classification).

Each tree has high variance and low bias. The average has low variance and the same low bias.

Random forest = bagging + feature subsetting

Pure bagging has a problem: if one feature dominates, every tree picks it as the root split. The trees end up correlated → averaging gains little.

Fix: at every split, sample only m ≈ √p features as candidates. Forces trees to disagree → decorrelates them → averaging actually helps.

  • Classification default: m = √p.
  • Regression default: m = p / 3.
  • m = p recovers plain bagging.

OOB error — cross-validation for free

A bootstrap sample uses ~63% of rows; the other ~37% are out-of-bag for that tree. Averaging OOB predictions per row gives a near-LOO CV estimate for free. Here we compare a single tree to a 300-tree random forest on a Carseats-shaped synthetic dataset and read off oob_score_.

  


  

    
Heart data — bagging vs random forest

    
    
Both beat a single tree quickly. RF keeps improving where bagging plateaus.

  


  

    
One important caveat

    
Adding more trees cannot overfit a bagged forest — it only reduces variance. But adding more trees doesn't help past a certain point either. Pick B large enough that the OOB curve has flattened (often 200–500).

  


  

    
Why bagging reduces variance

    
    
averaging cancels noise — each tree's jagged splits disagree, the mean is smooth.

  


  

    
RF = bagging + feature subsampling

    
    
decorrelates the trees — one dominant feature can't hijack every root split.

  


  

    
Out-of-bag samples

    
    
average OOB predictions per row → near-leave-one-out estimate, no held-out split needed.

Feature importance (Gini / MDI)

    
For each feature, sum the decrease in node impurity from splits on that feature, averaged over all trees.

Gradient Boosting

    
fit residuals sequentially

  


  

    
The intuition

    
Random forests fit many trees in parallel and average them.
Boosting fits many small trees in sequence, each correcting the last.

  


  

    
The algorithm (regression)

    
    
      
  1. Initialize f̂(x) = 0, residuals ri = yi.
    2. 
      
  2. For b = 1, …, B:
        
      
          
    • Fit a small tree f̂b (depth ~3) to the residuals.
      • 
          
    • Update: f̂(x) ← f̂(x) + λ · f̂b(x).
      • 
          
    • Update residuals: ri ← ri − λ · f̂b(xi).
      • 
        
    
      
    3. 
      
  3. Return f̂(x) = Σb λ · f̂b(x).
    4. 
    

    
"Gradient" because for general losses the residual is replaced by the negative gradient — same idea, broader scope.

  


  

    
Three knobs that matter

    

      

        
n_estimators (B)

        
Unlike RF, boosting can overfit if B too large. Use early stopping on a validation set.

      

      

        
learning_rate (λ)

        
Small λ + large B → smooth, accurate, slow. Typical: 0.01 – 0.1.

      

      

        
max_depth (d)

        
d = 1 → additive model (stumps). d = 3-6 → captures interactions of that order.

      

    

    
Sweet spot: λ ≈ 0.05, depth 3-6, n_estimators chosen by early stopping. Most defaults beat hand-tuned RFs.

  


  

    
Three knobs that matter

    
Grid-search learning_rate × n_estimators on a Carseats-shaped binary target. Watch how small lr + many trees usually edges out big lr + few, at the cost of fit time.


    

      
      
      
    

    

  


  

    
The race in one figure

    
    
Boosting usually edges out RF after many iterations, at the cost of more tuning & risk of overfitting.

  


  

    
The production stack

    
    
      
  • HistGradientBoostingClassifier — sklearn's histogram-based GBM. Native categorical support; fast.
    • 
      
  • XGBoost — the Kaggle workhorse; mature regularization, GPU support.
    • 
      
  • LightGBM — Microsoft; leaf-wise growth, faster on large data.
    • 
      
  • CatBoost — Yandex; great with high-cardinality categoricals out of the box.
    • 
    

    
All four implement the same core idea with different speed / quality tweaks. Default to histogram-based GBMs; switch only when you have a reason.

Honorable mention: BART

    
Bayesian Additive Regression Trees — perturbs trees stochastically inside an MCMC loop. Lower tuning burden than boosting; gives uncertainty intervals natively. Worth knowing about for any statistical-genetics / causal-inference work.
