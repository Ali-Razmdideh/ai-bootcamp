Part 1 · 0:00 – 0:30

Multinomial LR

softmax = sigmoid for K classes

The ER example revisited

A patient arrives with chest pain. Three possible labels:

Y ∈ {stroke, drug overdose, epileptic seizure}

Coding these 1 / 2 / 3 and running linear regression invents an ordering. Overdose is not "halfway between stroke and seizure."

Softmax gives each class its own linear score; no ordering assumed.

The softmax extension

P(Y=k | X=x) = eβk0+βk1x1+⋯+βkpxp / Σℓ eβℓ0+⋯+βℓpxp

  • K weight vectors β₁, …, βK — one per class.
  • Outputs sum to 1 — a probability per class, by construction.
  • Equivalent to K − 1 binary log-odds with a chosen reference class.
  • Fit by maximizing the multinomial cross-entropy — the generalization of the binary log-likelihood. (You'll meet this loss again in every neural net.)

Sklearn in one line

from sklearn.linear_model import LogisticRegression

m = LogisticRegression(max_iter=2000).fit(X, y)
# auto-detects multinomial when y has > 2 classes
m.predict_proba(X_new)        # shape (n, K)
m.classes_                    # column ordering

In Week 8 (deep learning) the very last layer of a classification network is a softmax — same math, learned features.

Softmax = sigmoid for K classes

Exponentiate, normalize. The largest logit gets the largest share; outputs are positive and sum to one — exactly the sigmoid story, generalized to K outputs.

Probability simplex (K = 3)

Every softmax output lives on this triangle. Corners = certainty in one class; centroid = maximum uncertainty (⅓, ⅓, ⅓); distance to each edge = the probability of the opposite class.

L1, L2, and the generative cousins

    
stability when LR struggles

  


  

    
Same idea as Week 3 — applied to log-likelihood

    

      

        
L2 (ridge)

        
Shrinks every β toward zero. Default in sklearn. Numerically stable when p is large or features are correlated.

      

      

        
L1 (lasso)

        
penalty='l1', solver='saga'. Zeros some β out — sparse model, automatic feature selection.

      

      

        
C = 1 / λ

        
sklearn parameterizes by C = inverse penalty. Small C = strong regularization. Confusing at first, learn it once.

      

    

    
Regularization is one idea applied to many losses: OLS → ridge/lasso; MLE → L2/L1 logistic.

  


  

    
L1 vs L2 multinomial coefficients

    
    
Same data, same model, different penalty. L2 shrinks every coefficient toward zero; L1 picks a handful of survivors and kills the rest — built-in feature selection.

  


  

    
L1 vs L2 coefficient paths

    
Sweep C over five orders of magnitude and record each coefficient. L2 (left) shrinks every β smoothly toward zero; L1 (right) drives some coefficients to exactly zero — that's the sparsity story from Lasso, applied to the LR log-likelihood. When LR struggles

    
    
      
  • Perfectly separable classes → coefficients diverge to ±∞. The MLE doesn't exist.
    • 
      
  • n < p (e.g. genomics) → unregularized LR overfits wildly.
    • 
      
  • Tiny class → high-variance estimates.
    • 
    

    
Regularization fixes (1) and (2). For (3), the generative family below is often more stable.

  


  

    
Discriminative vs generative

    

      

        
Discriminative

        
Logistic regression models P(Y | X) directly.

        
Few assumptions. Wins when you have plenty of data.

      

      

        
Generative

        
Model P(X | Y), then flip with Bayes:

        
P(Y=k | X) = πk fk(x) / Σ πℓ fℓ(x)

        
Strong assumptions; pays off when n is small or classes are well-separated.

      

    

  


  

    
Generative vs Discriminative — cheat sheet

    
    
Generative models the joint p(x, y); discriminative skips straight to the decision rule. Same picture, two routes — which wins depends on whether the density assumptions hold.

  


  

    
LDA — Gaussian, shared covariance

    

      

        
      

      

        
Assume each class is Gaussian with the same covariance Σ.

        
The Bayes decision rule then reduces to a linear function of x — that's the "L" in LDA.

        
    
          
  • Decision boundary: linear, just like LR.
    • 
          
  • Coefficients estimated from class means and pooled Σ — closed form, no optimizer.
    • 
          
  • More stable than LR when classes are well-separated or n is small.
    • 
        

      

    

  


  

    
LDA boundary — equal covariance ⇒ line (ISLR Fig 4.6)

    
    
Two Gaussians with the same covariance ellipse. Bayes decision rule cancels the quadratic terms — what remains is linear in x, exactly like logistic regression.

  


  

    
QDA — Gaussian, per-class covariance

    
Drop the shared-Σ assumption. Each class gets its own Σk.

    
The decision boundary becomes quadratic — ellipses, hyperbolas, parabolas.

    

      

        
LDA

        
Few parameters; low variance; bias if class shapes differ.

      

      

        
QDA

        
More parameters; flexible; needs enough data per class to estimate Σk reliably.

      

    

  


  

    
QDA boundary — different covariances ⇒ curve

    
    
When the class covariances differ, the quadratic terms no longer cancel and the boundary curves. Pays off when the truth really is curved and you have enough data to estimate each Σk.

  


  

    
Naive Bayes — features independent within each class

    
fk(x) = fk1(x₁) · fk2(x₂) · ⋯ · fkp(xp)

    
    
      
  • Models each feature one-dimensionally — easy with histograms or per-feature Gaussians.
    • 
      
  • Handles mixed types: Gaussian for numeric, multinomial for word counts, Bernoulli for booleans.
    • 
      
  • Strong assumption ("naive") that's usually wrong — and the model still works astonishingly well, especially in text.
    • 
    

    
The classic spam filter is multinomial Naive Bayes on bag-of-words. Decades old, still a respectable baseline.

  


  

    
Naive Bayes — conditional independence

    
    
The joint density is replaced by a product of one-dimensional marginals — one histogram per feature, per class. Strong assumption; still a strong baseline, especially for text.

  


  

    
Which one when?

    

      

        situationreach for
      

      
moderate n, mixed features, want probabilitiesLogistic regression

      
small n, well-separated classes, Gaussian-ishLDA

      
enough data and classes really look differentQDA

      
p ≫ n, many sparse features (text)Naive Bayes

      
when in doubtstart with regularized LR

When linear fails

    
two moons + the road to non-linear

  


  

    
The make_moons dataset

    
Two interleaved half-moons — no straight line can separate them. Fit a plain logistic regression and watch the accuracy plateau around 85%: the model isn't broken, the hypothesis class is too narrow.

No straight line works — needs kernels (Course 6) or trees (Course 4) to carve a curved boundary.

Two ways forward

    

      

        
Add non-linear features

        
from sklearn.preprocessing import PolynomialFeatures
poly = PolynomialFeatures(degree=3)
LogisticRegression().fit(poly.fit_transform(X), y)

        
Same linear model, richer feature space. Preview of the SVM kernel trick (Course 6).

      

      

        
Switch model family

        
Decision trees (Course 4), random forests + gradient boosting (Course 5), kernel SVMs (Course 6) all carve curved boundaries natively.

      

    

  


  

    
The lesson

    
"Linear in log-odds" is a strong assumption. When the truth is curved, you must either reshape the inputs or change the model. Day 2 does both.
