Part 1 · 0:00 – 0:30

The GLM Framework

three components that generalize linear regression

Three components of a GLM

  1. Distribution: Y | X comes from the exponential family (Gaussian, Binomial, Poisson, …)
  2. Linear predictor: η = β₀ + β₁x₁ + … + βₚxₚ = Xβ
  3. Link function: g such that g(E[Y|X]) = η, i.e. E[Y|X] = g⁻¹(η)

The link function maps the mean of Y to the linear predictor. It ensures E[Y|X] stays in the valid range for the distribution.

The canonical link for each family is the function g such that the score equations take the simplest form. For Gaussian it's the identity; for Binomial it's the logit; for Poisson it's the log. These are the natural parameter links from exponential family theory.

The GLM family table

Distribution of YLink gE[Y|X] = g⁻¹(η)Method
GaussianIdentity: g(μ) = μXβLinear regression
BinomialLogit: g(μ) = log(μ/(1−μ))σ(Xβ)Logistic regression
PoissonLog: g(μ) = log(μ)exp(Xβ)Poisson regression
Negative BinomialLog: g(μ) = log(μ)exp(Xβ)Neg. Binomial regression
GammaLog or inverseexp(Xβ) or 1/XβGamma regression

All of these are fitted by IRLS (Iteratively Reweighted Least Squares) — a unified algorithm.

IRLS iteratively solves a weighted least squares problem where the weights and working response depend on the current parameter estimates. For Gaussian it converges in one step (ordinary OLS). For others it requires multiple iterations.

Mean-variance relationships

DistributionMeanVarianceImplication
Gaussianμσ² (constant)Homoscedastic — equal spread everywhere
Binomialnπnπ(1−π)Variance peaks at π=0.5
PoissonμμVariance = mean — spread grows with level
Neg. Binomialμμ + μ²/rOverdispersed Poisson

This mean-variance relationship is why Poisson regression naturally models count data: higher counts have more variability (heteroscedastic).

Gaussian regression with OLS assumes constant variance — this is almost always wrong for count data. Using OLS on counts ignores the heteroscedasticity, leading to poor inference (too-narrow confidence intervals for large counts, too-wide for small counts).

Why linear regression fails for counts

  • Negative predictions: OLS can predict negative counts — meaningless for data like bike rentals, accidents, or word frequencies.
  • Wrong variance structure: OLS assumes constant variance; count data has variance ∝ mean.
  • Inefficient: ignoring the Poisson structure wastes information and gives biased SEs.

Poisson regression solves all three: the log link guarantees positive predictions, and the Poisson log-likelihood correctly weights observations by their expected count.

Part 2 · 0:30 – 1:00

Poisson Regression

log link, multiplicative effects, MLE

The Poisson regression model

log(E[Y | X]) = β₀ + β₁x₁ + … + βₚxₚ

E[Y | X] = exp(β₀ + β₁x₁ + … + βₚxₚ) = exp(Xβ)

  • The log link ensures E[Y|X] > 0 for any β — no negative predictions possible.
  • The linear predictor is additive on the log-mean scale.
  • Y | X ~ Poisson(exp(Xβ)) — the mean and variance are both exp(Xβ).

Interpreting coefficients

A one-unit increase in x_j multiplies the expected count by eβ_j.

β_jeβ_jInterpretation
01.00No effect on count
0.101.11+11% increase in expected count per unit
0.692.00Count doubles per unit increase
−0.220.80−20% decrease in expected count per unit

Poisson regression has multiplicative (not additive) effects. This is appropriate: doubling counts when the mean is 100 is very different from when it is 2.

Compare to linear regression where β_j is the additive change. In Poisson, β_j is the additive change in log(mean) — which means a multiplicative change in the mean itself. This is the natural scale for counts and rates.

Maximum likelihood for Poisson

Log-likelihood (dropping constants):

ℓ(β) = Σᵢ [yᵢ (Xᵢβ) − exp(Xᵢβ)]

  • This is concave in β — same good news as logistic regression.
  • Score equations: Σᵢ xᵢ (yᵢ − exp(Xᵢβ)) = 0 — residuals (observed − predicted) are orthogonal to predictors.
  • Solved by Newton–Raphson / IRLS in O(p²n) per iteration.

Goodness of fit: Deviance

Deviance = 2 × (log-likelihood of saturated model − log-likelihood of fitted model)

  • Poisson deviance: D = 2 Σᵢ [yᵢ log(yᵢ/μ̂ᵢ) − (yᵢ − μ̂ᵢ)]
  • Under the correct model: D ~ χ²(n − p − 1).
  • Plays the role of RSS in linear regression. Smaller = better fit.
  • Null deviance: deviance of intercept-only model. Explained deviance = null − residual.
The deviance ratio (explained/null) is analogous to R² in linear regression. AIC = deviance + 2p (penalizes model complexity), used for model selection.

Bikeshare Demo & Overdispersion

statsmodels, predicted counts, quasi-Poisson

The Bikeshare dataset

  • Hourly bike rental counts in Washington DC, 2011–2012.
  • Target: bikers — number of rental bikes per hour (integer count, min=1, max=977).
  • Features: hr (hour of day 0–23), temp (scaled °C), weathersit (1=clear, 4=heavy rain), workingday, mnth (month).

This is the canonical Poisson regression example from ISLR Section 4.6.

Linear regression failure

import statsmodels.api as sm
X = pd.get_dummies(bikes[['hr','mnth','weathersit','workingday']])
ols = sm.OLS(bikes['bikers'], sm.add_constant(X)).fit()
preds_ols = ols.predict(sm.add_constant(X))
print(preds_ols[preds_ols < 0].shape)  # negative predictions at night!
  • OLS predicts negative bike rentals at 3am. Physically impossible.
  • OLS residuals are heteroscedastic: variance grows with the fitted mean.
  • The Poisson model addresses both issues with the log link.
This is Figure 4.15 in ISLR 2nd edition. The left panel shows OLS predictions with negative values at night. The right panel shows Poisson predictions which are always positive.

Predicted counts by hour of day

  • Poisson predictions: always positive; smooth rise and fall through the day.
  • Bimodal pattern: morning rush (~8am) and evening rush (~5–6pm) captured by hour dummies.
  • OLS predictions: go negative at night (3–5am), miss the count's variance structure.
bikes['fitted_poisson'] = pois.fittedvalues

by_hour.plot()  # Poisson closely tracks the bimodal pattern

Overdispersion: when Var(Y) >> E[Y]

  • Poisson assumption: Var(Y) = E[Y]. Real count data often has Var(Y) >> E[Y].
  • Causes: unobserved heterogeneity, clustering, zero-inflation.
  • Consequence: Poisson SEs are too small → p-values too small → false discoveries.

Diagnosis: Pearson χ² / df ≫ 1 → overdispersed. Rule of thumb: ratio > 2 is noteworthy.

Overdispersion is extremely common in real data. The bikeshare data is actually overdispersed — the variance of bike counts is much larger than the mean at each hour. This is because of day-to-day variation in weather, events, and other factors not in the model.

Fixes for overdispersion

MethodHow it worksUse when
Quasi-PoissonScale SEs by √(Pearson χ²/df)Quick fix; same β̂, wider intervals
Negative BinomialAdd dispersion parameter r: Var = μ + μ²/rTrue overdispersion; better fit
Zero-Inflated PoissonMix of Poisson + point mass at 0Excess zeros (insurance claims, etc.)
# Quasi-Poisson (scaled SEs)
qpois = sm.GLM(y, X, family=sm.families.Poisson(),
               var_weights=np.ones(n)).fit(scale='X2')
For the bikeshare data, the overdispersion ratio is approximately 3–5. This means all Poisson standard errors should be multiplied by √3≈1.7 to account for the extra variance. Coefficients don't change, but significance does.
Recap

Six ideas to leave with

  1. GLM = distribution + link + linear predictor. Unifies LR, logistic, Poisson.
  2. Log link guarantees positive predictions — no negative counts ever.
  3. eβ_j is the multiplicative effect on mean count per unit increase of x_j.
  4. Poisson MLE is concave — same guarantee as logistic regression.
  5. Deviance plays the role of RSS — model comparison via AIC/deviance ratio.
  6. Check overdispersion: Pearson χ²/df ≫ 1 → use quasi-Poisson or negative binomial.
