Part 1 · 0:00 – 0:30

From log-odds to sigmoid

a linear model that respects [0, 1]

Linear regression on a 0/1 outcome

Code Y = 0/1, fit OLS. For low balance, the line dips below 0. That cannot be a probability.

For multi-class outcomes (stroke / overdose / seizure) it gets worse — coding them 1/2/3 invents an ordering that does not exist.

The sigmoid squashes the line into [0, 1]

P(Y=1 | X) = σ(β₀ + β₁X) = eβ₀+β₁X / (1 + eβ₀+β₁X)

Equivalently, the model is linear in log-odds:

log[ P/(1−P) ] = β₀ + β₁X

An S-shape in probability is a straight line in log-odds. That's the whole trick.

Fit by maximum likelihood, not OLS

Choose β to make the observed labels as probable as possible:

ℓ(β) = ∏i: y=1 p(xi) · ∏i: y=0 (1 − p(xi))

  • No closed form — solved iteratively (Newton-Raphson / L-BFGS).
  • Loss is convex → any reasonable optimizer reaches the unique maximum.
  • In R: glm(..., family=binomial). In sklearn: LogisticRegression().

Interpreting coefficients — odds ratios

A unit change in Xj multiplies the odds by eβj.

On Default with βbalance ≈ 0.0055:

e100·0.0055 ≈ 1.74

Every extra $100 of balance multiplies the odds of default by 1.74 — a 74% relative bump.

This is what an executive will actually ask about. Practice converting β → odds ratios on the spot.

Confounding visualized

  • Marginally, students default more often (they carry higher balances on average).
  • At any fixed balance, students default less often than non-students.
  • The multiple-LR coefficient captures the second story, after controlling for balance.

Lesson: a univariate model can lie. Always check the multivariate version before drawing causal arrows.

Part 3 · 1:00 – 1:30

ROC & threshold tuning

probabilities are the real output

predict_proba vs predict

Logistic regression's native output is a probability. predict applies a 0.5 cutoff — rarely optimal on imbalanced data.

proba = model.predict_proba(X_test)[:, 1]   # column for class 1
pred  = (proba >= 0.5).astype(int)            # default threshold

Default at threshold 0.5 → confusion matrix that misses three-quarters of true defaulters.

How the threshold trades errors

Lower threshold → catch more defaulters (recall ↑) but flag more honest customers (precision ↓).

Tune the threshold for F1

import numpy as np
from sklearn.metrics import f1_score, confusion_matrix

ts = np.linspace(0.05, 0.6, 50)
f1s = [f1_score(y_test, (proba >= t)) for t in ts]
best_t = ts[int(np.argmax(f1s))]

print(f"F1-optimal threshold: {best_t:.2f}")
print(confusion_matrix(y_test, proba >= best_t))
On Default the F1-optimal threshold lands near 0.15. Recall jumps from ~30% (at 0.5) to ~70% (at 0.15), with precision trading down accordingly.

ROC summary

  • The whole curve is generated by sweeping the threshold.
  • AUC summarizes the curve in one number, independent of any threshold.
  • Compare models on AUC; pick the operating point afterward.

sklearn.metrics.roc_curve + roc_auc_score for both.

Recap

Three things to leave with

  1. Linear in log-odds, sigmoid in probability. Fit by MLE; loss is convex.
  2. eβ is the odds ratio per unit of X. The interpretation an executive will ask about — and the one that flips sign under confounding.
  3. 0.5 is rarely the right threshold. Tune it for F1, recall, or any cost-sensitive metric; report AUC for the model itself.
