Part 1 · 0:00 – 0:30

The Naive Bayes Assumption

conditional independence within each class

From LDA to Naive Bayes

  • LDA: fk(x) = N(μk, Σ) — a joint p-variate Gaussian. Requires estimating p(p+1)/2 covariance parameters.
  • Problem: when p is large, Σ has too many parameters to estimate reliably from small n.
  • Naive Bayes fix: assume features are conditionally independent given the class:

    fk(x) = fk1(x1) · fk2(x2) · … · fkp(xp) = ∏j fkj(xj)

  • Now only p·K marginal densities to estimate — one per feature per class.
The name "naive" comes from this assumption being almost never literally true — features are rarely truly independent within a class. But the resulting classifier is often surprisingly good, because we're not trying to estimate the joint density exactly, just the decision boundary.

The posterior with Naive Bayes

Pr(Y=k | X=x) ∝ πk · ∏j=1p fkj(xj)

In log form: log Pr(Y=k|X=x) = log πk + Σj log fkj(xj) + const

  • The decision becomes: which class has the highest sum of log-likelihoods?
  • Each feature contributes independently and additively to the log-posterior.
  • Parameter count: p·K marginal parameters vs p²K/2 for full covariance (LDA).
The additive log-posterior structure is what links Naive Bayes to GAMs — see the connection slide later.

Three choices for the marginal density fkj

Feature typeModel for fkjParameters
QuantitativeGaussian: N(μkj, σ²kj)mean + variance per feature per class
Quantitative, non-normalKernel density estimate (KDE)non-parametric; bandwidth h
Qualitative / categoricalMultinomial table: P(xj = l | Y=k)count table, one proportion per level

Mixed features? No problem — use Gaussian for numeric, multinomial for categorical. LDA can't handle this naturally.

This is one of Naive Bayes' key practical advantages over LDA/QDA: it handles mixed feature types trivially. Each feature is modelled with the appropriate marginal.

Mixed features: Default dataset example

FeatureTypeNB density choice
balanceContinuous numericGaussian: N(μk,balance, σ²k,balance)
incomeContinuous numericGaussian: N(μk,income, σ²k,income)
studentBinary categoricalBernoulli: P(student=Yes | class k)

Each feature is modelled separately. No need for a joint distribution across all features — the independence assumption handles the combination automatically.

Part 2 · 0:30 – 1:00

Toy Example & Connections

decision regions, GAM link, LDA/QDA relationship

NB decision regions (Figure 4.14 style)

2-class, 2-feature Gaussian NB: each class has independent Gaussians for x₁ and x₂.

  • NB decision regions: axis-aligned ellipses — covariance ellipses have no off-diagonal tilt.
  • LDA decision regions: tilted ellipses — the shared Σ captures feature correlations.
  • When features truly are independent within class, NB = optimal Bayes classifier.
from sklearn.naive_bayes import GaussianNB
gnb = GaussianNB().fit(X_train, y_train)
# Decision boundary plot: compare with LDA boundary
This is similar to Figure 4.14 in ISLR 2nd edition. The key visual: NB has axis-aligned decision contours; LDA contours can be rotated. When features are correlated within a class, LDA captures this; NB misses it and loses accuracy.

Connection to LDA

  • If all fkj are Gaussian with the same variance σ² across classes:
  • NB = LDA with a diagonal covariance Σ = diag(σ²₁, …, σ²ₚ).
  • NB ignores within-class feature correlations (off-diagonal Σ terms).
  • If features are truly uncorrelated within each class: NB ≈ LDA in performance.

Naive Bayes is a special case of QDA: QDA with diagonal Σk.

Connection to GAMs

The log-posterior ratio between class k and class l:

log(Pr(k|x) / Pr(l|x)) = log(πk/πl) + Σj log(fkj(xj) / flj(xj))

  • This is a sum of functions of individual features — exactly the structure of a Generalized Additive Model (GAM).
  • Naive Bayes is a GAM where each gj(xj) = log(fkj(xj) / flj(xj)).
  • With Gaussian marginals: the gj become quadratic functions → NB gives a quadratic decision boundary.
This connection is noted in ISLR Section 4.4.4. The GAM structure is why NB is interpretable: each feature's contribution to the decision can be examined separately. This is analogous to why linear models are interpretable.

Why NB works despite wrong assumptions

  • Bias-variance tradeoff: the independence assumption biases the decision boundary, but the variance is much lower (fewer parameters).
  • When p is large and n is small, variance dominates — NB's low variance wins.
  • Even when features ARE correlated, the Bayes-optimal classification regions may still be well-approximated by NB's regions.
  • The key: NB needs the ranking of posteriors correct, not the exact probability values.

NB is one of the oldest and most practical classifiers. Text classification and spam detection are NB's home turf.

GaussianNB on Default

from sklearn.naive_bayes import GaussianNB
gnb = GaussianNB()
gnb.fit(X_train, y_train)
print('AUC:', roc_auc_score(y_test, gnb.predict_proba(X_test)[:,1]))

# Learned parameters per class:
gnb.theta_          # class means (K × p)
gnb.var_            # class variances (K × p)
  • NB with Gaussian marginals fits μkj and σ²kj independently per feature per class.
  • No covariance between features is estimated — the independence assumption is baked in.

MultinomialNB — the spam filter

from sklearn.naive_bayes import MultinomialNB
from sklearn.feature_extraction.text import CountVectorizer

vec  = CountVectorizer()
X_counts = vec.fit_transform(text_data)   # sparse word-count matrix
mnb = MultinomialNB(alpha=1.0)            # alpha = Laplace smoothing
mnb.fit(X_counts, y_text)
  • Each feature = word count. P(word w | class k) estimated from frequency tables.
  • Laplace smoothing (alpha=1.0): add 1 to every count to avoid P(unseen word) = 0.
  • Scales effortlessly to vocabulary size p = 100,000+. Training is one pass through the data.
Spam filtering was one of the first major ML success stories in production. NB was the dominant approach through the 2000s and is still competitive in text classification. The key advantage: training is a single pass over the data (counting word frequencies), and inference is O(p) per document.

When Naive Bayes wins

  • Very high p, small n (text, genomics): LDA can't estimate Σ with p > n; NB always works.
  • Text classification: words are approximately independent given topic/sentiment.
  • Fast training: one pass over data, closed-form, O(p·K·n) — scales to billions of features.
  • Mixed feature types: combine Gaussian + Bernoulli marginals naturally.
  • Online learning: update parameters incrementally as new data arrives (partial_fit).

When Naive Bayes loses

  • Strong feature correlations within classes: the independence assumption is violated; boundary is biased.
  • Well-calibrated probabilities needed: NB posteriors are often overconfident (push toward 0/1).
  • Large n, moderate p: LDA, QDA, or LR can estimate correlations — and benefit from the signal.

AUC benchmark on Default (balance + income): LR ≈ LDA ≈ QDA ≈ 0.95; GaussianNB ≈ 0.93. NB is competitive but slightly worse due to the ignored balance–income correlation.

Benchmark: all four classifiers

from sklearn.discriminant_analysis import LinearDiscriminantAnalysis
from sklearn.discriminant_analysis import QuadraticDiscriminantAnalysis
from sklearn.linear_model import LogisticRegression
from sklearn.naive_bayes import GaussianNB

models = [('LR', LogisticRegression(max_iter=2000)),
          ('LDA', LinearDiscriminantAnalysis()),
          ('QDA', QuadraticDiscriminantAnalysis()),
          ('GNB', GaussianNB())]
for name, m in models:
    m.fit(X_train, y_train)
    print(f'{name}: AUC = {roc_auc_score(y_test, m.predict_proba(X_test)[:,1]):.4f}')
Recap

Six ideas to leave with

  1. Conditional independence: f_k(x) = ∏_j f_kj(x_j). One marginal per feature per class.
  2. Posterior: Pr(Y=k|X) ∝ π_k · ∏_j f_kj(x_j). Additive in log-space.
  3. Three density choices: Gaussian (numeric), KDE (non-normal), multinomial (categorical).
  4. NB ↔ GAMs: log-posterior ratio is additive in feature functions — same structure.
  5. NB = QDA with diagonal Σ_k; NB = LDA when additionally same σ² per feature.
  6. NB wins when p large, n small; loses when features are strongly correlated within classes.
