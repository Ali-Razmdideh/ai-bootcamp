Part 1 · 0:00 – 0:30

Subset Selection

Best subset, forward stepwise, backward stepwise

Why replace least squares?

  • Prediction accuracy: OLS has low bias but can have high variance, especially when p is large relative to n.
  • Model interpretability: setting irrelevant coefficients to zero yields a simpler model.
  • Two families of fix: subset selection (keep only some predictors) and shrinkage (shrink all toward zero).
  • A third approach — dimension reduction — projects predictors into a lower-dimensional subspace.

Best Subset Selection

  • Fit all 2p possible models (one per predictor subset).
  • For each size k, keep the model with smallest RSS — call it Mk.
  • Choose the single best Mk using CV, AIC, or BIC.
  • 240 ≈ 1012 models for p=40 — intractable!

Forward & Backward Stepwise

Forward

  • Start with null model (intercept only).
  • Add one predictor at a time — the one that most improves fit.
  • Considers 1 + p(p+1)/2 models.
  • Works even when n < p.

Backward

  • Start with full model (all p predictors).
  • Remove one at a time — the one with the largest p-value.
  • Requires n > p so the full model can be fit.

    Part 2 · 0:30 – 1:00
    
Choosing the Optimal Model

    
Cp, AIC, BIC, Adj-R², validation, and cross-validation

  


  

    
Training error is a bad guide

    
    
      
  • The full model always has the smallest RSS and largest R² on training data.
    • 
      
  • Training error underestimates test error — it doesn't account for overfitting.
    • 
      
  • We need criteria that penalise complexity.
    • 
    

    

      
CriterionFormulaGoal

      
Mallow's Cₚ(RSS + 2dσ̂²) / nminimise

      
AIC−2 log L + 2dminimise

      
BIC−2 log L̂ + log(n)·dminimise

      
Adjusted R²1 − RSS/(n−d−1) / TSS/(n−1)maximise

    

  


  

    
BIC vs AIC — penalty strength

    
    
      
  • AIC/Cₚ penalty per parameter: 2
    • 
      
  • BIC penalty per parameter: log(n) — for n=400: log(400) ≈ 6.0
    • 
      
  • BIC penalises ~3× more than AIC for n=400 → tends to select smaller, more parsimonious models.
    • 
      
  • Rule of thumb: use AIC/Cₚ when predictive accuracy is the goal; use BIC when you want to identify the true model (if one exists).
    • 
      
  • Both are equivalent to CV asymptotically under different conditions — in practice, check both and report when they disagree.
    • 
    

  


  

    
Validation set and CV

    
    
      
  • Validation set: reserve 20–30% of data; fit on rest; pick model with smallest held-out MSE.
    • 
      
  • K-fold CV: rotate the hold-out K times; average MSEs. More stable, no σ̂² needed.
    • 
      
  • One-SE rule: pick the simplest model within 1 SE of the CV minimum — prefers parsimony.
    •

    Part 3 · 1:00 – 1:20
    
Ridge Regression

    
L2 shrinkage, coefficient paths, and the bias–variance trade-off

  


  

    
Ridge regression — the objective

    

      
OLS: minimise RSS = Σ(yᵢ − β₀ − Σβⱼxᵢⱼ)²

      

      
Ridge: minimise RSS + λΣβⱼ²

      
β̂₀ is NOT penalised. Must standardise X first.

    
    
      
  • λ = 0 → OLS. λ → ∞ → all βⱼ → 0.
    • 
      
  • Shrinks toward zero but never exactly zero.
    • 
      
  • All p predictors stay in the model.
    • 
      
  • Select λ with cross-validation.
    • 
    

  


  

    
Why ridge improves over OLS: bias–variance

    

      
    
    
      
  • OLS: zero bias, but high variance when p is large or predictors are correlated — small changes in data produce large changes in coefficients.
    • 
      
  • As λ grows: bias ↑ a little, variance ↓ sharply — a favourable trade when n is small relative to p.
    • 
      
  • Test MSE = Bias² + Variance + σ² has a U-shape as a function of λ — there is an optimal λ > 0.
    • 
      
  • Ridge wins (over OLS) when the variance reduction exceeds the bias cost, which is typical when predictors are correlated or p approaches n.
    •

    Part 4 · 1:20 – 1:40
    
The Lasso

    
L1 sparsity, variable selection, and the geometric picture

  


  

    
The Lasso objective

    

      
Lasso: minimise RSS + λΣ|βⱼ|

      
The L1 kink at zero forces some coefficients to be exactly zero → variable selection.

    
    
      
  • λ = 0 → OLS. λ → ∞ → null model.
    • 
      
  • Intermediate λ → sparse model.
    • 
      
  • Also called LASSO: Least Absolute Shrinkage and Selection Operator.
    • 
    

  


  

    
Why lasso produces exact zeros — geometry

    

      
    
    
      
  • Lasso: constraint region is a diamond with corners on axes.
    • 
      
  • RSS ellipse contacts a corner → β = exactly 0.
    • 
      
  • Ridge: constraint is a sphere — smooth surface, no corners → never exactly 0.
    • 
    

  


  

    
Lasso vs Ridge — when to choose which

    

      
Lasso

      
    
        
  • Sparse models (exact zeros).
    • 
        
  • Automatic variable selection.
    • 
        
  • Best when few predictors truly matter.
    • 
      

    

      
Ridge

      
    
        
  • Keeps all predictors (shrinks, never zeros).
    • 
        
  • Best when many predictors all contribute a little.
    • 
        
  • Handles multicollinearity well.
    • 
      

    

    
Use cross-validation to decide which is better for your data. Elastic Net interpolates between the two.

    Part 5 · 1:40 – 2:00
    
Dimension Reduction

    
Principal Components Regression and Partial Least Squares

  


  

    
Dimension reduction — the idea

    
    
      
  • Transform p predictors into M < p new features Z₁, …, ZM (linear combinations).
    • 
      
  • Fit OLS of Y on Z₁, …, ZM. M < p → constrains coefficient space → variance reduction.
    • 
      
  • Key question: how do we choose the linear combinations?
    • 
    

    

      
PCR
Unsupervised
Directions of max variance in X (PCA). Ignores Y when finding components.

      
PLS
Supervised
Directions that maximise covariance with Y. Uses the response to guide extraction.

    

  


  

    
PCR vs PLS — choosing M by CV

    

      
    
    
      
  • PLS often needs fewer components than PCR.
    • 
      
  • PLS directions are guided by Y → more relevant.
    • 
      
  • Both use K-fold CV to pick M.
    • 
      
  • Both typically outperform OLS when p is large relative to n.
    • 
      
  • sklearn: Pipeline(PCA + LinearRegression) for PCR; PLSRegression for PLS.
    • 
    

  


  

    
Comparing all methods — practical guide

    

      
MethodVariable selection?Best when…Key parameter

      
Best SubsetYes (exact)p ≤ 30, computation OKk

      
Forward/BackwardYes (greedy)Large p, fast neededk

      
RidgeNo (shrinks all)Many small effects, multicollinearityλ

      
LassoYes (exact zeros)Few true predictors, interpretabilityλ

      
PCRNoCorrelated X, p > nM

      
PLSNop > n, Y-relevant structureM

    

    
Always use cross-validation to select the key parameter. No method universally dominates.

  Recap
  
Three things to leave with

  
    
    
  1. Subset selection (best subset, forward, backward) identifies which predictors to include. CV or BIC chooses the model size.
    2. 
    
  2. Ridge (L2) and Lasso (L1) shrink coefficients toward zero. Lasso creates exact zeros — variable selection. CV picks λ.
    3. 
    
  3. PCR and PLS reduce dimension first, then regress. PCR is unsupervised; PLS uses the response to guide directions.
    4. 
  

  
No single method wins everywhere. Use cross-validation to compare them on your data. Lasso is usually the first choice when you want sparsity; Ridge when you expect many small effects.
