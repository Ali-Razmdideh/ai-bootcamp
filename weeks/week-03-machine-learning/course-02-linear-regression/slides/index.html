Part 1

Simple Linear Regression

one predictor, one line, one loss

The linear regression model

  • We assume Y = β₀ + β₁X + ε
  • β₀ (intercept) and β₁ (slope) are unknown constants — the coefficients.
  • ε is the error term: everything the model doesn't explain.
  • True regression functions are never linear — but linear is often a useful approximation.
Slide 1 of ISLR Ch3. Stress that the model is an approximation. George Box: 'All models are wrong, but some are useful.'

Advertising data — the running example

200 markets · TV, radio, newspaper budgets ($thousands) · sales (thousands of units). Positive trend — a straight line looks reasonable.

Least-squares estimation

  • Prediction: ŷᵢ = β̂₀ + β̂₁xᵢ  ·   Residual: eᵢ = yᵢ − ŷᵢ
  • RSS = Σeᵢ² = Σ(yᵢ − β̂₀ − β̂₁xᵢ)²
  • Closed form: β̂₁ = Σ(xᵢ−x̄)(yᵢ−ȳ) / Σ(xᵢ−x̄)²,   β̂₀ = ȳ − β̂₁x̄

Confidence Interval — what it really means

    

      
Definition: A 95 % CI is a range built from sample data such that, if we repeated the experiment many times, ~95 % of the intervals would contain the true β₁.

      
⚠️ Not: "95 % chance β₁ is in this interval." The true β₁ is fixed — the randomness is in the interval itself.

    

    

      

        
🎣 Fishing net analogy

        
    
          
  • The fish = true β₁ (doesn't move).
    • 
          
  • Each sample = one net cast at a random spot.
    • 
          
  • A 95 % CI means: your net-casting procedure catches the fish 95 % of the time.
    • 
        

      

      

        
Formula

        
$$\hat{\beta}_1 \pm t^*_{\alpha/2,\,n-2} \cdot \text{SE}(\hat{\beta}_1)$$

        
where   $\text{SE}(\hat{\beta}_1) = \hat{\sigma}/\sqrt{\sum(x_i-\bar{x})^2}$

        
and   $t^*_{0.025,\,n-2} \approx 1.96$ for large $n$.

        
Sampling distribution:   $\displaystyle\frac{\hat{\beta}_1 - \beta_1}{\text{SE}(\hat{\beta}_1)} \sim t_{n-2}$

      

    

  


  

    
Standard errors and confidence intervals

    
    
    
      
  • SE(β̂₁)² = σ² / Σ(xᵢ − x̄)²   — wider X spread → smaller SE.
    • 
      
  • 95 % CI ≈ β̂₁ ± 2·SE(β̂₁). Advertising TV: [0.042, 0.053] — excludes zero.
    • 
    

  


  
  

    
Student's t-Test — signal vs noise

    

      
Question: Is this coefficient large enough relative to its uncertainty to be considered real?

      
H₀: βj = 0  (predictor has no effect)   vs   Hₐ: βj ≠ 0

    

    

      

        
📻 Noisy room analogy

        
    
          
  • Signal = estimated effect β̂ (your voice).
    • 
          
  • Noise = standard error SE (background noise).
    • 
          
  • t = signal / noise. Large |t| → your voice cuts through → likely real.
    • 
          
  • |t| < 1: effect could easily be noise.   |t| > 2: strong evidence.
    • 
        

      

      

        
Formula

        
$$t = \frac{\hat{\beta}_j}{\text{SE}(\hat{\beta}_j)}, \quad \text{SE}(\hat{\beta}_j) = \hat{\sigma}\sqrt{(X^\top X)^{-1}_{jj}}$$

        
where   $\hat{\sigma}^2 = \text{RSS}/(n-p-1)$

        
Distribution under H₀:

        
$t \sim t_{n-p-1}$

        
Heavy tails (vs Normal) account for estimating σ. As n → ∞ it converges to N(0,1).

      

    

  


  

    
Hypothesis test: is β₁ = 0?

    
    
    
      
  • H₀: β₁ = 0.  t = β̂₁ / SE(β̂₁) = 17.67 for TV — far in the tail.
    • 
      
  • p = P(|T| ≥ 17.67 | H₀) ≪ 0.001. We reject H₀.
    • 
    

  


  
  

    
P-Value — the probability of surprise

    

      
Definition: Probability of observing a test statistic at least as extreme as the one we got, if H₀ were true.

      
⚠️ NOT the probability H₀ is true. NOT the probability the result is "due to chance."

    

    

      

        
⚖️ Juror analogy

        
    
          
  • H₀ = "defendant is innocent."
    • 
          
  • Small p-value: evidence is very improbable under innocence → lean toward rejecting H₀.
    • 
          
  • Large p-value: evidence is consistent with innocence → not enough to convict.
    • 
          
  • p does not say how likely the alternative is — only how inconsistent the data is with H₀.
    • 
        

      

      

        
Formula (two-sided)

        
$$p = 2 \cdot P(T_{n-p-1} > |t_{\text{obs}}|)$$

        
Thresholds

        

          
p < 0.001Very strong evidence

          
0.001–0.01Strong evidence

          
0.01–0.05Moderate (conventional cutoff)

          
> 0.10Little to no evidence

        

        
p-value = shaded tail area beyond |t_obs| under the t-distribution.

Chi-Squared Test — are categories independent?

    

      
Use when: your variables are categorical (counts / frequencies).

      
H₀: the two variables are independent  |  Hₐ: they are associated.

    

    

      

        
🎲 Loaded die analogy

        
    
          
  • Roll a die 600×. Expected under fairness: 100 per face.
    • 
          
  • χ² measures total "surprise": how far observed counts deviate from expected.
    • 
          
  • Fair die → small χ², large p → can't reject H₀.
    • 
          
  • Loaded die → large χ², small p → reject H₀.
    • 
        

        
vs t-test: t is for continuous outcomes; χ² is for categorical counts. χ² is always right-skewed and one-sided.

      

      

        
Formula

        
$$\chi^2 = \sum_{i,j} \frac{(O_{ij} - E_{ij})^2}{E_{ij}}$$

        
Expected cell count:   $E_{ij} = \dfrac{(\text{row}_i \text{ total}) \times (\text{col}_j \text{ total})}{n}$

        
Distribution under H₀:

        
$\chi^2 \sim \chi^2_k$   where   $k = (r-1)(c-1)$

        
Right-skewed, non-negative. Only the right tail counts (large χ² → evidence against H₀).

    Part 3
    
Assessing Model Accuracy

    
RSE and R²

  


  
  

    
TSS — Total Sum of Squares

    

      

        
TSS measures how much Y varies before we fit any model — it is the baseline variance:

        
$$\text{TSS} = \sum_{i=1}^{n}(y_i - \bar{y})^2$$

        
Each squared deviation asks: how far is this observation from the mean? Sum them up → total spread in Y.

      

      

        
Three sums that tell the whole story:

        

          
NameFormulaMeaning

          
TSSΣ(yᵢ−ȳ)²Total variance in Y

          
RSSΣ(yᵢ−ŷᵢ)²Variance left unexplained

          
ESSTSS − RSSVariance explained by model

        

        
R² = ESS / TSS = 1 − RSS/TSS  ∈ [0, 1] — the fraction of variance the model captures.

      

    

  


  
  

    
RSE — why divide by n − p − 1?

    

      

        
RSE = $\sqrt{\text{RSS}/(n-p-1)}$

        
Dividing by n − p − 1 (not n) makes RSE an unbiased estimator of the true σ.

        

          
Degrees of freedom intuition

          
    
            
  • We have n observations.
    • 
            
  • We spend p + 1 of them estimating β₀, β₁, …, βₚ.
    • 
            
  • Only n − p − 1 remain "free" to estimate noise.
    • 
          

        

        
If you divided by n instead, RSS would systematically undercount the true variance — you'd be overconfident about your fit.

      

      

        
Concrete example (Advertising TV, n=200, p=1):

        

          
FormulaValue

          
√(RSS / n)3.25 (biased)

          
√(RSS / (n−2))3.26 (unbiased RSE)

        

        
Difference is tiny here (n=200 is large), but matters for small datasets.

        
sklearn divides by n (biased). statsmodels uses n−p−1 (correct RSE). Always check which you're using.

      

    

  


  
  

    
RSE vs R² — when to use which

    

      

        
RSE   √(RSS / (n−p−1))

        
✓ Pros

        
    
          
  • Same units as Y — directly interpretable ("predictions are off by ±3.26 thousand $").
    • 
          
  • Not bounded — useful for comparing absolute error across models.
    • 
          
  • Tells you if the error is practically acceptable.
    • 
        

        
✗ Cons

        
    
          
  • Scale-dependent — can't compare RSE across datasets with different Y units.
    • 
        

      

      

        
R²   1 − RSS/TSS

        
✓ Pros

        
    
          
  • Unitless, always in [0, 1] — comparable across any dataset or Y scale.
    • 
          
  • Intuitive: "the model explains 61% of variance in sales."
    • 
          
  • Benchmark is clear: 0 = no better than predicting ȳ.
    • 
        

        
✗ Cons

        
    
          
  • Always increases when you add predictors — use adjusted R² for MLR.
    • 
          
  • High R² doesn't mean small errors in absolute terms.
    • 
        

      

    

    

      
Advertising TVRSER²

      
TV → Sales3.260.612

Multiple Linear Regression

    
more predictors, one model

  


  

    
The MLR model

    
    
      
  • Y = β₀ + β₁X₁ + β₂X₂ + … + βₚXₚ + ε
    • 
      
  • Each βⱼ = expected change in Y per unit ↑ in Xⱼ, holding all others fixed.
    • 
      
  • The fit is a hyperplane in p+1 dimensions.
    • 
    

  


  

    
The woes of interpreting MLR coefficients

    

      
    
        
  • Predictors usually move together — ceteris-paribus is theoretical.
    • 
        
  • Example: newspaper looks significant alone, but collapses once radio is in the model.
    •

    Part 5
    
Variable Selection

    
F-statistic, forward/backward, AIC/BIC

  


  

    
Is at least one predictor useful? — the F-test

    
    
      
  • H₀: β₁ = β₂ = … = βₚ = 0. F = [(TSS−RSS)/p] / [RSS/(n−p−1)]
    • 
      
  • Under H₀: F ~ F(p, n−p−1). Large F → reject H₀.
    • 
      
  • Advertising MLR: F = 570, p ≈ 0 → at least one predictor helps.
    • 
      
  • Don't use individual t-tests when p is large — you'll accumulate false positives.
    • 
    

  


  

    
Selection strategies and model-size criteria

    

      

        
    
          
  • Forward: greedily add predictor that most reduces AIC.
    • 
          
  • Backward: drop predictor with largest p-value.
    • 
          
  • Both are heuristics — no global guarantee.
    • 
        

      

      

        
      

    

    
    
      
  • Adj R² = 1 − [RSS/(n−p−1)] / [TSS/(n−1)]
    • 
      
  • AIC = −2 log L̂ + 2p;   BIC = −2 log L̂ + p·log(n) — BIC penalises extra predictors more heavily than AIC (log(n) > 2 when n > 7).
    •

    Part 7
    
Interactions & Non-linearity

    
synergy effects, hierarchy principle, polynomial regression

  


  

    
Interaction terms

    
    
      
  • sales = β₀ + β₁·TV + β₂·radio + β₃·(TV×radio) + ε
    • 
      
  • Effective slope of TV = β̂₁ + β̂₃·radio → slope depends on radio.
    • 
      
  • R² jumps from 89.7% (additive) to 96.8% — interaction explains 69% of remaining variance.
    • 
      
  • Hierarchy principle: if interaction is included, always include main effects too.
    • 
    

  


  

    
Qualitative × quantitative & polynomial

    

      

        
        
With interaction — different slopes per group

      

      

        
        
Polynomial: same framework, engineered features

      

    

    
Polynomial regression is still linear in the coefficients — just MLR with x, x², x³ as features.
