Part 1 · 0:00 – 0:30

Best subset

the combinatorial cost

Running example — the Credit data

Predict Balance from 10 predictors: Income, Limit, Rating, Cards, Age, Education, plus dummies for Gender, Student, Married, Ethnicity.

Some of these are highly correlated (Limit and Rating in particular — credit limits are set from ratings). Several are likely irrelevant once others are in the model. The question this whole course answers: which subset of these 10 predictors should I keep?

We will see best subset, forward stepwise, and Cp/AIC/BIC/CV all applied to the same Credit data so students can compare answers directly.

Why best-subset is intractable

  • p = 10 → 1024 subsets (Credit). Fine on a laptop.
  • p = 20 → about 1 million.
  • p = 40 → over 1,000,000,000,000 (a trillion). One billion+ models.
  • Exponential — completely infeasible past ~30 predictors.

Genomics, NLP, recommender systems routinely have p in the thousands. Best subset is a non-starter; we need a greedy or shrinkage alternative.

RSS always drops with more predictors

Adding a predictor can never increase training RSS. So we cannot pick model size by raw RSS.

We need a criterion that penalizes size: AIC, BIC, Cp, adjusted R², or — best — held-out CV error.

Best-subset on 5 predictors

Enumerate every non-empty subset of the first 5 predictors and keep the one with smallest training RSS at each size k. With only 5 predictors that's 25−1 = 31 fits — fine. At p = 19 it would be 524,288. RSS strictly decreases as k grows, which is precisely why raw RSS can't pick a size.

Forward stepwise

    
greedy but cheap

  

  

    
Forward vs Backward stepwise

    

      

        
Forward

        
Start empty. Add the predictor that most lowers RSS. Repeat until all p are in.

        
Cost: 1 + p + (p−1) + … = 1 + p(p+1)/2 model fits. O(p²).

      

      

        
Backward

        
Start with all p. Drop the predictor whose removal lowers RSS the least. Repeat until empty.

        
Same O(p²) cost. Requires n > p (otherwise the full model can't be fit).

      

    

    
For p = 1000, best subset = 2^1000 models. Stepwise = ~500,000. The difference between "impossible" and "lunchtime."

At size 4 the two methods disagree. Forward stepwise is locked into the rating it picked at step 1 — it can't swap rating for cards even if the swap improves the 4-variable fit. Best subset can. Greedy choices have consequences.

  

  

    
Forward stepwise algorithm


    
  
  • Start with the empty model.
    • 
  
  • At each step, add the single predictor that most reduces CV error.
    • 
  
  • Stop when CV error stops improving (or fit all p, then pick the best size).
    • 


  


  

    
From scratch in 12 lines

    
Forward stepwise: at every step, try adding each remaining predictor and keep the one with smallest 5-fold CV-MSE. O(p²) fits instead of best-subset's 2p. The trace shows which predictor was added at each step and the CV-MSE after it joined.

Model-size criteria

    
AIC, BIC, Cp, adj-R²

  

  

    
Four formulas, four answers


AIC = n log(RSS/n) + 2p
BIC = n log(RSS/n) + p log n
Cp  = (RSS + 2 p σ²)/n
adj-R² = 1 − (1−R²)(n−1)/(n−p−1)

Credit data — four criteria, four curves

    
Run best subset for sizes k = 1, 2, …, 10. Plot each criterion against k:

    
    
      
  • C_p minimum at k = 6.
    • 
      
  • AIC minimum at k = 6 (proportional to C_p for Gaussian errors).
    • 
      
  • BIC minimum at k = 4 — its log(n) penalty is heavier, so it prefers a smaller model.
    • 
      
  • Adjusted R² maximum at k = 7 — the loosest, picks the biggest model.
    • 
    

    
Same data, four reasonable criteria, three different answers. There is no single right size — pick a criterion based on whether you want prediction (AIC/C_p), parsimony (BIC), or just descriptive fit (adj-R²).

  

  

    
The one-standard-error rule

    
CV error curves are noisy. The minimum is often part of a flat region. Rule of thumb:

    
    
      
  1. Find the smallest CV error, MSE_min, and its standard error SE_min.
    2. 
      
  2. Among models with CV error ≤ MSE_min + SE_min, pick the simplest.
    3. 
    

    
A larger model that is only marginally better is not actually better — its apparent edge is within the noise. Always prefer the simpler model when they tie.

  

  

    
Which to trust?


    
  
  • BIC — most conservative; for large n it picks fewer predictors.
    • 
  
  • AIC / Cp — pick slightly larger models, focused on prediction.
    • 
  
  • adj-R² — loosest, easiest to fool with collinear predictors.
    • 
  
  • Pick a criterion before you look at the answers, and consider CV the gold standard.
    •
