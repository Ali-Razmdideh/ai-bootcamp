The validation-set trap

one split lies

The U-shaped curve, again

The fundamental picture of supervised learning:

Training error monotonically falls. Test error falls, hits a minimum, then climbs. Our whole job is to find that minimum without using the test set.

Why one split is not enough

  • Pick polynomial degree by held-out MSE → answer depends on the split.
  • Ten random splits give ten different 'best' degrees.
  • That spread is noise, not signal.

K-fold and LOOCV

    
average over many splits

  

  

    
Picture: ten splits, ten answers

    
Take Auto mpg ~ horsepower. Loop ten random 50/50 splits. For each, fit polynomials of degree 1–10 and record validation MSE.

    
The result is ten wiggly U-shaped curves. They all roughly bottom out between degree 2 and 4 — but the exact minimum jumps around: split 1 picks d=2, split 4 picks d=7, split 7 picks d=3.

    
A single train/test split is a high-variance estimate of test error. Trusting one split to pick a hyperparameter is like trusting one coin flip to call the season.

    
CV averages over many splits and gives a stable curve.

  

  

    
K-fold cross-validation


Split into K folds. For k = 1…K, train on the other K−1 folds and score on fold k. Average the K scores.
    
  
  • K = 5 or K = 10 is the standard. K = n is leave-one-out (LOOCV).
    • 
  
  • cross_val_score(model, X, y, cv=KFold(10)) is one line.
    •

LOOCV — the magic shortcut for linear models

    
K = n looks expensive — n model fits. For linear regression, there's an exact one-pass formula:

    
CV(n) = (1/n) Σᵢ [(yᵢ − ŷᵢ) / (1 − hᵢ)]²

    
where h_i is the i-th diagonal of the hat matrix H = X(XᵀX)⁻¹Xᵀ. Fit the model once, divide each residual by (1 − h_i), square, average. Done.

    
For arbitrary models (random forests, neural nets), there is no shortcut — LOOCV genuinely costs n fits, which is why we use 5- or 10-fold by default.

  

  

    
LOOCV — k equals n

    
LeaveOneOut is just K-fold with K = n. Cheap here because the model is a degree-2 polynomial linear regression. Reports the mean MSE across all n single-row holdouts.


    

      
      
      
    

    

  

  

    
K = 5 or 10 is the sweet spot

    
    
      
  • LOOCV (K = n): near-zero bias (each fold trains on n − 1), but folds are almost identical → high variance in the estimate.
    • 
      
  • K = 2: each fold trains on only n/2 → pessimistic bias.
    • 
      
  • K = 5 or 10: trains on 80–90% of data, folds are diverse enough. Empirically the right default.
    • 
    

    
ISLR's recommendation, repeated word-for-word: "K = 5 or K = 10 has been shown empirically to yield test error rate estimates that suffer neither from excessively high bias nor from very high variance."

The bootstrap

    
resample with replacement

  

  

    
The right way vs the wrong way

    
Setup from ISLR §5.3: n = 50 samples, p = 5000 features, y is a coin flip, the predictors are pure noise. True test error = 50%.

    

      

        
Wrong way

        
    
          
  1. Screen 5000 features → keep the 100 most correlated with y.
    2. 
          
  2. Run 10-fold CV on those 100 features.
    3. 
        

        
Reported CV error: ~3%. Reality: 50%.

      

      

        
Right way

        
    
          
  1. For each of the 10 folds:
    2. 
          
  2. screen features using only that fold's training data,
    3. 
          
  3. then score on its held-out fold.
    4. 
        

        
CV error: ~50%. Honest.

      

    

    
Every preprocessing step that uses y — feature screening, target encoding, supervised PCA — must live inside the CV loop. Otherwise the held-out fold has already leaked.

Bootstrap = sampling-distribution emulator

    
Estimate the standard error and a 95% CI of a regression slope by resampling the data 1000 times with replacement and refitting. Works for any statistic — slope, median, ratio, correlation — without distributional assumptions.

The portfolio example — α

    
Two financial assets with returns X and Y. Split your money: fraction α in X, (1 − α) in Y. The variance-minimizing α is:

    
α = (σ²_Y − σ_XY) / (σ²_X + σ²_Y − 2σ_XY)

    
We have 100 historical return pairs. We compute α̂ ≈ 0.576. Question: how uncertain is that 0.576?

    
Bootstrap recipe: resample (X, Y) pairs with replacement 1000 times, recompute α̂ each time, take the standard deviation. That's SE(α̂). Slap ±1.96·SE around 0.576 for a 95% CI.

    
No formula for SE(α̂) — the closed-form would require knowing σ²_X, σ²_Y, σ_XY exactly. The bootstrap sidesteps the math entirely.

  

  

    
"Pull yourself up by your bootstraps"

    
Baron Munchausen claimed to have escaped a swamp by pulling himself out by his own bootstraps. Efron borrowed the metaphor in 1979 for the technique that estimates uncertainty using only the sample itself.

    
Don't use the bootstrap to estimate test error. Each bootstrap sample reuses about ⅔ of the original data, so "training set" and "test set" overlap — the test error estimate is biased down. For test error: use CV. For SE/CI of a statistic: use the bootstrap.
